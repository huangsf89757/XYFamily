## 三、项目概览

### 3.1 项目定位

XYFamily 是一个通用多功能工具集平台，当前阶段构建**多租户账号权限底座**，为后续业务工具提供统一身份认证、权限管理和数据隔离能力。

### 3.2 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Go 1.22+ |
| Web 框架 | Gin |
| 数据库 | PostgreSQL 17 |
| 缓存 | Redis 7.x |
| 认证 | JWT (HS256) |
| 部署 | Docker + K8s |

### 3.3 核心架构

```
客户端 → Nginx/Ingress → xyfamily-1/2/3 (无状态多实例)
                              ↓
                    PostgreSQL (主从) + Redis (Sentinel)
```

### 3.4 组织架构

```
Organization (组织)
  └── Team (团队)
        └── Group (小组)
              └── Member (成员)
```

层级继承：organization_core_admin 可访问本组织 + 下属所有团队/小组。

## 五、开发规范速览

### 5.1 工程目录

```
cmd/xyfamily/          # 入口
internal/
  handler/           # API 入口（按模块分包）
  service/           # 业务逻辑
  repository/        # 数据访问
  model/             # 领域模型
  middleware/        # 横切中间件
pkg/
  jwt/               # JWT 封装
  bcrypt/            # 密码哈希
  response/          # 统一响应
  errors/            # 业务错误码
configs/
migrations/
```

### 5.2 分层规则

```
Handler → Service → Repository → Model → 数据库/Redis
```

**禁止跨层调用**（Handler 不能直接调用 Repository）。

### 5.3 多租户隔离（必考）

所有组织相关查询必须通过 `WithOrgScope`：

```go
// ✅ 正确
teams := WithOrgScope(db, req.OrgID).Find(&teams)

// ❌ 禁止
teams := db.Find(&teams)  // 无 org 过滤
```

### 5.4 Git 规范

```
<type>(scope): <description>

feat(auth): 新增验证码发送接口
fix(perm): 修复权限缓存未失效问题
docs(readme): 更新项目说明
```

详见：[Git 版本管控规范](../04-开发规范与编码手册/04.03-Git版本管控规范/Git版本管控规范-V1.0.0.md)

## 六、核心业务逻辑

### 6.1 认证流程

```
注册 → 发送验证码 → 校验验证码 → 创建账号(bcrypt) → 返回 account_id
登录 → 查询账号 → 校验密码 → 签发 JWT → 写入 sessions → 返回 Token
```

### 6.2 权限校验流程

```
请求 → JWT 校验 → Token 黑名单 → Membership 校验(层级回溯)
       → PermissionChecker(Redis 缓存 → DB 兜底) → 业务 Handler
```

### 6.3 成员邀请流程

```
邀请方 → POST /org/:id/members/invitations {identifier, role}
       → 创建 invitations 记录
被邀请方 → GET /auth/invitations（查看待确认）
       → POST /auth/invitations/:id/accept（确认加入）
```
