## 二、接口设计模板

```markdown
### {接口名称}

| 项目 | 内容 |
|------|------|
| 方法 | `{GET/POST/PUT/DELETE}` |
| 路径 | `/path/to/resource` |
| 认证 | ✅ / ❌ |
| 权限点 | `perm.code` |
| HTTP Status | 200 / 201 |

**请求体：**

```json
{
  "field": "value"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| field | string | ✅ | 说明 |

**响应：**

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

**约束：**
- 约束说明
```

## 三、ADR 模板

```markdown
## ADR-{序号}：{决策标题}

**状态**：已接受 / 废弃 / 替代
**日期**：2026-07-12

**背景**：
为什么需要做这个决策？当前的问题是什么？

**决策**：
- 决策内容
- 具体方案

**理由**：
为什么选择这个方案？替代方案及不选的原因。

**后果**：
- 正面影响
- 负面影响
- 需要做什么来缓解负面影响
```

## 四、测试用例模板

```markdown
### TC-{模块缩写}-{序号}：{用例标题}

**所属模块**：{模块名}
**用例类型**：功能测试 / 边界测试 / 异常测试 / 安全测试 / 性能测试
**优先级**：P0 / P1 / P2

**前置条件**：
1. 条件 1
2. 条件 2

**测试步骤**：
1. 步骤 1
2. 步骤 2

**预期结果**：
- HTTP {status}
- 响应内容

**实际结果**：
（执行后填写）

**通过/失败**：（最终判定）
```
