## 标签体系

所有 Wiki 页面强制绑定 1~3 个标签，统一项目标签池，禁止自定义零散标签。标签分六大维度：

| 维度 | 标签示例 |
|------|----------|
| **业务维度** | 认证模块、账号模块、组织模块、团队模块、小组模块、权限模块、审计模块、系统基础 |
| **研发阶段** | 立项阶段、需求阶段、设计阶段、开发阶段、测试阶段、灰度发布、交付阶段、运维阶段、迭代优化、复盘沉淀 |
| **文档类型** | 立项文档、需求PRD、技术方案、ADR决策、接口文档、数据库设计、部署手册、测试用例、故障复盘、规范标准、合规报告、培训课件 |
| **状态/优先级** | 核心文档、普通文档、临时草稿、待更新、正式生效、已废弃、机密文档、对外交付版 |
| **职能角色** | 产品、后端、安卓开发、iOS开发、H5前端、小程序开发、Web前端、测试、运维、项目管理、安全 |
| **合规&技术专项** | 数据脱敏、等保合规、隐私治理、漏洞安全、分布式、微服务、中间件、容灾多活、第三方集成 |

## 文档命名规范

统一格式：`文档名称-V版本.md`

- 密级、日期、模块、文档类型等信息由**目录结构**和**文档头部元数据**承载，文件名不重复
- 示例：`RBAC权限继承优化-V2.1.0.md`

---

## 文档头部必填元数据

所有正式文档必须包含：

| 字段 | 说明 |
|------|------|
| 文档密级 | 公开 / 内部 / 机密 |
| 文档版本 | 语义化版本号 |
| 编写人 | 文档编写者（规则见下方变更人规范） |
| 审核人 | 文档审核者 |
| 生效时间 | 文档生效日期 |
| 废弃时间 | 文档废弃日期（未废弃填 - ） |
| 关联标签 | 1~3 个标签 |
| 关联目录 | 所属目录路径 |

## 变更人填写规范

文档的「编写人」和「变更记录-变更人」字段，统一使用**当前实际操作所使用的 AI 编程工具名称**，便于追溯每条变更的来源：

| 使用工具 | 变更人填写 |
|----------|-----------|
| Claude Code | `ClaudeCode` |
| CatPaw | `CatPaw` |
| 人工编辑 | 编辑者真实姓名或工号 |

**规则：**
- 谁创建/修改了文档，就填谁的名称
- 同一文档的不同版本记录可出现不同变更人（如 V1.0.0 由 CatPaw 创建，V1.1.0 由 Trae 修改）
- 纯人工编辑时填写真实姓名或工号

## 文档更新机制

| 机制 | 规则 |
|------|------|
| 迭代更新 | 版本上线 3 个工作日内同步更新对应接口、方案、部署文档 |
| 变更同步 | 需求、架构、接口、线上配置变更，**当日同步更新** Wiki |
| 季度盘点 | 每季度末统一清理已废弃文档、更新过期资料、归档历史版本 |
| 机密文档管控 | 密钥、数据库账号、客户隐私方案禁止上传明文，需脱敏处理 |

## 分级权限管控

| 权限级别 | 范围 | 说明 |
|----------|------|------|
| 只读权限 | 全员 | 可查看公开、内部文档，检索学习；机密文档无查看权限 |
| 编辑权限 | 模块负责人 | 对应模块产品、开发、测试、运维，仅可编辑自身业务文档 |
| 管理权限 | 项目/技术/安全负责人 | 目录调整、文档审核、废弃清理、密级变更 |
| 对外输出权限 | 项目负责人 + 安全负责人 | 双审批后，可导出对外交付文档 |
