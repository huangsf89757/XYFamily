## 二、JWT 鉴权测试用例

### TC-JWT-001: 生成并解析 Access Token

- **描述**：验证 GenerateAccessToken + ParseAccessToken 正确签发和解析双 Token
- **输入**：account_id="account-1", org_ids=["org-1","org-2"], roles={"org-1":"organization_core_admin"}
- **预期**：token 非空、jti 非空、解析后 account_id/jti/org_ids/roles 正确
- **结果**：✅ 通过

### TC-JWT-002: 解析无效 Token

- **描述**：传入格式错误的 Token 字符串
- **预期**：返回错误
- **结果**：✅ 通过

### TC-JWT-003: 使用错误密钥解析 Token

- **描述**：用 secret-1 签发，用 secret-2 解析
- **预期**：返回错误（签名不匹配）
- **结果**：✅ 通过

### TC-JWT-004: 生成 Refresh Token

- **描述**：验证 GenerateRefreshToken 返回非空 token 和 hash
- **预期**：token 和 hash 均非空且不同
- **结果**：✅ 通过

### TC-JWT-005: TTL 配置正确

- **描述**：验证 AccessTTL 和 RefreshTTL 返回正确值
- **预期**：AccessTTL=1800s, RefreshTTL=604800s
- **结果**：✅ 通过

## 三、密码哈希测试用例

### TC-BCRYPT-001: 哈希并比对密码

- **描述**：验证 Hash + Compare 正确哈希和比对
- **输入**：password="MyStrong@Pass123", cost=10
- **预期**：hash 非空、hash != password、Compare 成功
- **结果**：✅ 通过

### TC-BCRYPT-002: 错误密码比对失败

- **描述**：哈希 "correct-password" 但比对 "wrong-password"
- **预期**：Compare 返回错误
- **结果**：✅ 通过

### TC-BCRYPT-003: 哈希唯一性

- **描述**：对相同密码生成两次哈希
- **预期**：两次哈希结果不同（随机盐）
- **结果**：✅ 通过
