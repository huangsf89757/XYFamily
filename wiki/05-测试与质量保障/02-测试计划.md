## 1. 文档定位

本文件承接《测试策略 - 测试与质量保障》，将测试活动映射到里程碑 Phase / Milestone 与门禁节点，明确"何时测、测什么、进出条件"。质量阈值以《里程碑 - 项目总览》§3 为准。

## 2. 阶段测试计划（对齐 Phase / Gate）

| Phase | 里程碑 | 关联门禁 | 主要测试活动 | 进出条件 |
|-------|--------|----------|--------------|----------|
| Phase 1 规划与设计冻结 | M0 | G0 | 需求可测性评审、验收标准基线化 | PRD 评审通过、验收标准基线化 |
| Phase 2 基础设施与后端核心 | M1–M6 | G1 / G2 | 单元测试、接口契约测试、SAST/依赖扫描、后端联调冒烟 | G2 准出：契约通过、单测 ≥ 80%、扫描零高危、冒烟通过 |
| Phase 3 多端研发与联调 | M7–M12 | - | 多端功能测试、跨端一致性回归、Mock/契约前置联调 | 各端功能自测通过、契约一致 |
| Phase 4 集成测试与安全合规 | M13 | G3 | 集成测试、多租户隔离专项、性能压测、安全渗透 | G3 准出：集成通过、渗透无高危、隔离验证通过、性能达标 |
| Phase 5 灰度发布与上线运营 | M14–M15 | G4 | 灰度发布验证、回滚预案验证、可观测性核验 | G4 准出：灰度/回滚就绪、监控告警就绪 |
| Phase 6 延后功能与持续迭代 | M16 | - | P2 功能测试（待排期） | 待排期 |

> 门禁准出指标详见《里程碑 - 项目总览》§3 质量门禁矩阵。
