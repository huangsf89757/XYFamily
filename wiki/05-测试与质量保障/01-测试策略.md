## 1. 文档定位

本文档是「测试与质量保障」板块的总纲，向上对齐里程碑 Stage-Gate 质量门禁（G2 后端转测门禁、G3 测试 / 安全合规评审），向下指导测试计划、用例库、专项测试与自动化子文档的编写。所有测试活动的**准出标准**以《里程碑 - 项目总览》§3 质量门禁矩阵为准，本文档不另行定义阈值。

> 门禁 / 基线来源说明：单测覆盖率、接口契约一致性、SAST/依赖扫描、性能基线、多租户隔离、安全渗透等阈值均引自《里程碑 - 项目总览》§3，本文档仅做测试落地映射。

## 2. 分级测试模型

采用经典四层测试金字塔 + 验收层，逐层向上集成度与执行成本递增：

| 层级 | 范围 | 主要目标 | 执行时机 | 关联门禁 |
|------|------|----------|----------|----------|
| 单元测试（Unit） | 函数 / 方法 / 组件 | 逻辑正确性，覆盖率达标 | 每次提交（CI 卡点） | G2 / G3 |
| 集成测试（Integration） | 模块间 / 服务间 / DB | 接口契约一致、数据流正确 | G2 后端转测、G3 全链路 | G2 / G3 |
| 系统测试（System） | 端到端业务流 | 多租户业务闭环、跨端一致性 | G3 联调完成后 | G3 |
| 验收测试（Acceptance） | 用户故事 / 需求项 | 需求与验收标准基线对齐 | 各 Sprint 收尾 / 发布前 | G0 / G4 |

- **单元 / 集成测试**以自动化为主，纳入 CI 卡点，不达标自动阻断合并。
- **系统 / 验收测试**结合自动化 + 人工探索，输出测试报告作为门禁评审材料。

## 3. 测试类型与覆盖范围

| 测试类型 | 覆盖重点 | 检测方式 | 卡点门禁 |
|----------|----------|----------|----------|
| 功能测试 | 9 大功能模块（注册认证 / 租户隔离 / 权限 / 数据 / ...）业务正确性 | 用例执行 + 自动化回归 | G2 / G3 |
| 接口契约测试 | OpenAPI 文档与实现 100% 一致 | Dredd / Swagger Diff 等契约测试 | G2 |
| 性能压测 | API 95% < 100ms；登录 95% < 200ms；吞吐 ≥ 1000 req/s | k6 / JMeter | G3 |
| 安全渗透 | 无高危及以上风险 | 渗透测试 / 漏洞扫描 | G3 |
| 多租户隔离专项 | 跨组织数据零泄露 | 隔离专项用例 + 数据校验 | G3 |
| 兼容 / 多端测试 | 四端（网页 / 桌面 / 移动 / 小程序）+ iOS / 鸿蒙 表现一致 | 多端回归矩阵 | G3 |
| 依赖 / 合规 | 零 Critical/High 开源漏洞；审计日志保留 1 年、注销后匿名化 | SCA + 合规检查脚本 | G2 / G3 |

> 9 大功能模块范围与验收标准基线见《产品 PRD - 多租户底座》；接口契约以 04-接口文档的 OpenAPI 为准。

## 4. 质量门禁卡点映射

| 门禁 | 准入条件 | 测试活动 | 准出质量指标（引自里程碑 §3） |
|------|----------|----------|-------------------------------|
| G2 后端转测门禁 | G1 通过、后端核心开发完成 | 接口契约测试、单测、SAST/依赖扫描、后端联调冒烟 | 接口契约测试通过、单测覆盖率 ≥ 80%、SAST/依赖扫描零高危、后端联调冒烟通过 |
| G3 测试 / 安全合规评审 | G2 通过、前后端联调完成 | 集成测试、渗透测试、多租户隔离验证、性能压测 | 集成测试通过、渗透测试无高危、多租户隔离验证通过、性能基线达标 |

> 前端单测覆盖率阈值 ≥ 70%（见里程碑 §3 质量门禁矩阵）。

## 5. 多端兼容性策略

XYFamily 交付四端（网页端 / 桌面端 / 移动端 / 小程序）并扩展 iOS 原生 App、鸿蒙 HarmonyOS，测试需覆盖：

- **逻辑一致性**：同一业务在多渠道表现、数据、权限一致（以 PRD 验收标准为基准）。
- **原型对齐**：UI 表现对齐 02-原型与 UI 设计的高保真原型（待完善：逐端走查清单）。
- **端侧专项**（待完善）：iOS（Swift）/ 鸿蒙（ArkTS）客户端稳定性、推送、离线态等端侧特性用例。

## 6. 测试环境与数据

- **多租户隔离测试数据**：必须构造≥2 个组织（Organization）的数据集，验证跨组织查询 / 写入零泄露，对应 RISK-003（跨组织数据泄露）。
- **数据脱敏 / 审计**：测试数据不得包含生产个人信息；涉及审计日志的检查项（保留 1 年、注销匿名化）纳入合规测试。
- **Mock / 契约**：接口未就绪时使用 Mock Server + OpenAPI 契约前置（对应 RISK-007 多端联调接口不一致）。

## 7. 自动化与 CI 卡点

- **单测覆盖率**：后端 ≥ 80%、前端 ≥ 70%，CI 接入覆盖率工具（JaCoCo / Vitest）卡点。
- **契约测试**：OpenAPI 契约与实现一致性 100%，作为 G2 准入硬性卡点。
- **静态扫描**：SAST（SonarQube / CodeQL）零高危、依赖漏洞扫描（SCA：Trivy / Dependabot）零 Critical/High。
- **流水线编排**（待完善：具体 CI 工具链与阶段划分）。
