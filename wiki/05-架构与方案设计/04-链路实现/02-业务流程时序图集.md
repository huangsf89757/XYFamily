## 一、注册流程（手机号 + 验证码）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API(Gin)
    participant MW as 中间件链
    participant SVC as Account/Auth SVC
    participant RD as Redis(验证码/黑名单)
    participant PG as PostgreSQL

    C->>API: POST /auth/register {phone, code, password}
    API->>MW: ①RateLimiter + ②JWT跳过(公开)
    MW->>RD: 校验验证码 vc:{phone}:register
    alt 验证码无效
        RD-->>C: 400 110001/110004
    else 通过
        MW->>SVC: 创建账号
        SVC->>PG: INSERT accounts (默认 active)
        SVC->>PG: INSERT sessions (Refresh)
        SVC->>RD: 写黑名单? (无) 生成 Token 对
        SVC->>RD: XADD audit_logs Stream (register)
        SVC-->>C: 201 {access, refresh}
    end
```

## 二、登录流程（密码）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as 中间件链
    participant SVC as Auth SVC
    participant RD as Redis
    participant PG as PG

    C->>API: POST /auth/login {identifier, password}
    API->>MW: ①RateLimiter(IP/账号维度)
    MW->>SVC: 识别标识 + bcrypt 校验
    SVC->>PG: 查 accounts(按 phone/email/username/id)
    alt 失败/锁定
        SVC->>RD: 失败计数 + 锁定
        SVC-->>C: 401 101008 / 429 104290
    else 成功
        SVC->>PG: 更新 last_login_at
        SVC->>PG: INSERT sessions
        SVC->>RD: XADD audit (login success)
        SVC-->>C: 200 {access, refresh}
    end
```

## 三、登出流程（幂等）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as ②③④ 中间件
    participant RD as Redis
    participant PG as PG

    C->>API: POST /auth/logout (Bearer Access + refresh?)
    MW->>RD: ②校验 Access 签名 → ③查 bl:{jti}
    alt 已失效
        MW-->>C: 200 幂等成功
    else 有效
        MW->>RD: SET bl:{jti} (TTL=剩余)
        MW->>PG: UPDATE sessions SET revoked_at (按 refresh)
        MW->>RD: XADD audit (logout)
        MW-->>C: 200 成功
    end
```

## 四、刷新 Token（Rotation）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant PG as PG(sessions)
    participant RD as Redis(黑名单)

    C->>API: POST /auth/refresh {refresh_token}
    API->>PG: 校验会话存在/未撤销/未过期
    alt 已被使用(疑似被盗)
        API->>PG: 该账号全部会话 revoked
        API->>RD: 相关 jti 入黑名单
        API-->>C: 401 101010 全员下线
    else 正常
        API->>PG: 旧会话 revoked + 新会话 INSERT
        API-->>C: 200 {新 access, 新 refresh}
    end
```

## 五、邀请 → 接受（统一邀请流）

```mermaid
sequenceDiagram
    participant I as 邀请人
    participant API as API
    participant MW as ④Membership ⑤Permission
    participant PG as PG(invitations/members)
    participant A as 被邀请人

    I->>API: POST /organizations/{id}/members/invitations
    MW->>MW: ④上下文 + ⑤ org.member.invite 校验
    API->>PG: INSERT invitations (status=pending, token)
    A->>API: POST /invitations/{token}/accept (或登录后确认)
    API->>PG: 校验 token 一次性 + 未过期
    API->>PG: INSERT org_members (role=指定)
    API->>PG: UPDATE invitations status=accepted
    API->>PG: 失效 perm 缓存 perm:account:{id}:org:{org_id}
    API-->>A: 200 加入成功
```

## 六、注销 → 恢复（宽限期）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant SVC as Account SVC
    participant PG as PG(accounts)

    C->>API: POST /account/deactivate
    API->>SVC: 置 status=deactivating, deactivated_at=now+30d
    SVC-->>C: 200 进入宽限期
    Note over C,PG: 宽限期内不可业务操作；定时任务扫描到期→匿名化
    C->>API: POST /account/undeactivate {code}
    API->>SVC: 校验验证码 + 未超宽限
    SVC->>PG: status=active (角色权限自动恢复)
    SVC-->>C: 200 恢复成功
```

## 七、组织创建（接棒机制）

```mermaid
sequenceDiagram
    participant S as SuperAdmin
    participant API as API
    participant MW as ④⑤ (org.create)
    participant PG as PG(organizations/members)
    participant RD as Redis(perm 缓存)

    S->>API: POST /organizations {name, initial_admin?}
    MW->>MW: ⑤ org.create 仅 SuperAdmin
    API->>PG: INSERT organizations (owner=创建者)
    API->>PG: INSERT org_members (创建者=organization_core_admin)
    alt 指定 initial_admin
        API->>PG: INSERT invitations (role=organization_core_admin, pending)
    end
    API->>RD: XADD audit (org.create)
    API-->>S: 201 {org_id}
```

## 八、典型权限校验（访问团队接口）

```mermaid
sequenceDiagram
    participant C as Client
    participant MW as 中间件链
    participant RD as Redis(perm)
    participant PG as PG(role_permissions)

    C->>MW: GET /teams/{id} + X-Org/Team-ID
    MW->>MW: ① ② ③ 通过
    MW->>MW: ④ 解析 scope + 层级回溯定角色
    MW->>RD: GET perm:account:{id}:org:{org_id}
    alt 命中
        RD-->>MW: 权限点集合
    else 未命中
        MW->>PG: SELECT permission_key FROM role_permissions WHERE role_key=生效角色
        PG-->>MW: 集合
        MW->>RD: SET 缓存 TTL=24h
    end
    MW->>MW: ⑤ 含 team.read? 是→放行 否→403
    MW->>C: 业务 Handler 处理
```

## 九、验证码登录流程（ARCH-012 修复）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as 中间件链
    participant SVC as Auth SVC
    participant RD as Redis
    participant PG as PG

    C->>API: POST /auth/verification-codes {target, type=login}
    API->>MW: ①RateLimiter
    MW->>RD: 检查频控 (target 1次/60s + IP 5次/5min + 账号级 20次/天)
    alt 频控超限
        MW-->>C: 429 114290/114291/114293
    else 通过
        MW->>RD: 写入验证码 vc:{target}:login (TTL=5min)
        MW-->>C: 200 {expires_in: 300}
    end
    C->>API: POST /auth/login {identifier, identifier_type=phone, credential_type=code, code}
    API->>MW: ①RateLimiter
    MW->>RD: 校验验证码 vc:{target}:login
    alt 验证码无效/过期/超限
        MW-->>C: 400 110001/110002/110004
    else 验证通过
        MW->>SVC: 查找账号
        SVC->>PG: SELECT accounts WHERE phone_hash=?
        alt 账号不存在
            SVC-->>C: 401 101007 账号不存在
        else 账号存在
            SVC->>SVC: 校验账号状态
            alt deactivated
                SVC->>SVC: 检查是否超期 → 兜底匿名化 (ARCH-014)
                SVC-->>C: 401 101009 账号已注销
            else deactivating
                SVC->>PG: INSERT sessions
                SVC->>RD: XADD audit (login success, deactivating)
                SVC-->>C: 200 {access, refresh, status=deactivating, deadline}
            else active
                SVC->>PG: INSERT sessions
                SVC->>RD: XADD audit (login success)
                SVC-->>C: 200 {access, refresh}
            end
        end
    end
```

## 十、密码重置流程（ARCH-012 修复）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as 中间件链
    participant SVC as Auth SVC
    participant PG as PG
    participant RD as Redis

    C->>API: POST /auth/verification-codes {target, type=reset_password}
    API->>MW: ①RateLimiter
    MW->>RD: 检查频控 + 写入验证码
    MW-->>C: 200 {expires_in: 300}
    C->>API: POST /auth/reset-password {type, target, code, new_password}
    API->>MW: ①RateLimiter (公开接口)
    MW->>RD: 校验验证码 vc:{target}:reset_password
    alt 验证码无效
        MW-->>C: 400 110001/110002/110004
    else 验证通过
        MW->>SVC: 查找账号 (防枚举：不存在也返回成功)
        alt 账号存在
            SVC->>PG: 更新 password_hash (bcrypt cost=12)
            SVC->>PG: ① 撤销全部 sessions (revoked_at=now)
            SVC->>RD: ② 批量写入 bl:{jti} 黑名单 (失败则补偿重试, ARCH-004)
            SVC->>RD: XADD audit (auth.reset_password)
            SVC->>RD: 失效该账号所有权限缓存
        end
        SVC-->>C: 200 {data: null} (账号不存在也返回成功, 防枚举)
    end
```

## 十一、成员移除流程（ARCH-012 修复）

```mermaid
sequenceDiagram
    participant C as Client (管理员)
    participant API as API
    participant MW as 中间件链(②③④⑤)
    participant PG as PG
    participant RD as Redis

    C->>API: DELETE /organizations/{org_id}/members/{account_id}
    API->>MW: ②JWT ③黑名单 ④Membership ⑤Permission(org.member.remove)
    MW->>MW: ④ 确定生效角色 + ⑤ 校验权限
    alt 无权限
        MW-->>C: 403
    else 有权限
        MW->>PG: 检查是否最后一名 core_admin (ADR-014)
        alt 是最后管理员
            MW-->>C: 400 300006 最后管理员不可移除
        else 非最后管理员
            MW->>PG: 软删除 org_members (deleted_at=now)
            MW->>PG: 联级软删除该账号在 team_members/group_members 中的记录
            MW->>RD: 失效权限缓存 perm:account:{id}:org:{org_id} 及下级 scope
            MW->>RD: XADD audit (org.member.remove)
            MW-->>C: 200 移除成功
        end
    end
```

## 十二、SuperAdmin 强制降级流程（ARCH-012 修复）

```mermaid
sequenceDiagram
    participant S as SuperAdmin
    participant API as API
    participant MW as 中间件链(②③④⑤)
    participant PG as PG
    participant RD as Redis

    S->>API: POST /admin/organizations/{org_id}/members/{account_id}/force-downgrade
    API->>MW: ②JWT ③黑名单 ④Membership(super_admin) ⑤Permission(admin.force_downgrade)
    MW->>MW: ⑤ 校验 SuperAdmin 权限 (跨组织操作)
    MW->>PG: 查询目标账号在该组织的角色
    alt 非核心管理员
        MW-->>S: 400 无需降级
    else 是核心管理员
        MW->>PG: 检查是否最后一名 core_admin
        alt 是最后管理员且无 SuperAdmin 干预
            MW->>PG: 强制降级为 regular_member (绕过保护, 因 SuperAdmin 操作)
            MW->>PG: 记录 admin.force_downgrade 审计
        else 非最后管理员
            MW->>PG: 降级为 regular_member
        end
        MW->>RD: 失效权限缓存
        MW->>RD: XADD audit (admin.force_downgrade, details={before, after})
        MW-->>S: 200 降级成功
    end
```
