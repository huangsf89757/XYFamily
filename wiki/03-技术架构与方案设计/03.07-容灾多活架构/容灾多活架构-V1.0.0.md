## 二、应用层（无状态多实例）

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| 无状态 | 实例不保存业务状态，所有状态外置到 PG + Redis |
| 水平扩展 | 根据 QPS 动态增减实例，LB 自动分发 |
| 优雅降级 | 实例关闭时 drain 已有连接，不中断请求 |

### 2.2 负载均衡

| 方案 | 说明 |
|------|------|
| 内网 LB | Nginx / ALB，Round Robin + 健康检查 |
| 健康检查 | `GET /healthz` 检查 PG + Redis 连通性 |
| 会话亲和 | 不需要（JWT 无状态，不依赖 sticky session） |

### 2.3 健康检查

```go
// cmd/xyfamily/main.go

func HealthCheck() gin.HandlerFunc {
    return func(c *gin.Context) {
        checks := make(map[string]string)

        // PG 连通性
        if err := db.Ping(); err == nil {
            checks["pg"] = "ok"
        } else {
            checks["pg"] = "error: " + err.Error()
        }

        // Redis 连通性
        if err := redis.Ping(); err == nil {
            checks["redis"] = "ok"
        } else {
            checks["redis"] = "error: " + err.Error()
        }

        status := 200
        for _, v := range checks {
            if !strings.HasPrefix(v, "ok") {
                status = 503
            }
        }

        c.JSON(status, checks)
    }
}

r.GET("/healthz", HealthCheck())
```

## 三、PostgreSQL 主从架构

### 3.1 架构

```
┌─────────────────┐    流复制    ┌─────────────────┐
│  PostgreSQL 主库 │ ─────────► │ PostgreSQL 从库  │
│  (写 + 读)       │  (WAL)      │  (只读，读请求)  │
└─────────────────┘             └─────────────────┘
```

### 3.2 读写分离

| 类型 | 路由 |
|------|------|
| 写操作 | 主库 |
| 读操作（审计日志查询、成员列表、组织列表） | 从库 |
| 读操作（账号信息、密码校验） | 主库（需强一致性） |

```go
// internal/repository/db.go

var (
    dbMaster *gorm.DB // 写库
    dbSlave  *gorm.DB // 读库
)

func MasterDB() *gorm.DB { return dbMaster }
func SlaveDB() *gorm.DB  { return dbSlave }

// Repository 层显式选择
func (r *AccountRepo) FindByID(id uuid.UUID) (*Account, error) {
    var a Account
    return &a, SlaveDB().Where("id = ?", id).First(&a).Error // 读从库
}

func (r *AccountRepo) Create(a *Account) error {
    return MasterDB().Create(a).Error // 写主库
}
```

### 3.3 备份策略

| 备份类型 | 频率 | 保留期 | 说明 |
|----------|------|--------|------|
| 全量备份 | 每日凌晨 2:00 | 30 天 | `pg_dump -Fc` |
| WAL 归档 | 实时 | 7 天 | 支持 PITR（按时间点恢复） |
| 增量快照 | 每周日 | 90 天 | 用于灾难恢复演练 |

### 3.4 故障切换

| 场景 | 处理方式 |
|------|---------|
| 主库故障 | 手动/自动提升从库为主库，更新 LB 写路由 |
| 从库故障 | 自动重建从库（从主库流复制） |
| 网络分区 | 主库继续写入，从库降级；分区恢复后增量同步 |

## 四、Redis 高可用（Sentinel）

### 4.1 架构

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│ Sentinel1 │  │ Sentinel2 │  │ Sentinel3 │  (监控 + 故障切换)
└─────┬─────┘  └─────┬─────┘  └─────┬─────┘
      └──────────────┼──────────────┘
                     │
              ┌──────▼──────┐
              │  Redis Master│  (缓存 + Stream + 黑名单)
              └──────┬──────┘
                     │ 复制
              ┌──────▼──────┐   ┌──────────┐
              │ Redis Rep-1  │   │ Redis Rep-2│
              └─────────────┘   └──────────┘
```

### 4.2 配置

| 参数 | 值 | 说明 |
|------|-----|------|
| Sentinel 数量 | 3 | 奇数，防止脑裂 |
| Master 故障判定 | 2 个 Sentinel 同意 | quorum=2 |
| 故障切换超时 | 30s | 自动选举新 Master |
| 复制模式 | 异步 | 性能优先，数据可丢失 |

### 4.3 应用连接

```go
// 应用通过 Sentinel 发现 Master，无需硬编码地址
opt := &redis.FailoverOptions{
    MasterName:    "xyfamily",
    SentinelAddrs: []string{"sentinel1:26379", "sentinel2:26379", "sentinel3:26379"},
    Password:      config.RedisPassword,
    DB:            0,
}
client := redis.NewFailoverClient(opt)
```

## 五、数据一致性

### 5.1 事务边界

| 场景 | 一致性要求 | 实现 |
|------|-----------|------|
| 成员邀请 | 强一致 | PG 事务（invitations + members） |
| 权限缓存 | 最终一致 | 业务写入 PG 后异步删除 Redis cache |
| 审计日志 | 最终一致 | Redis Stream → Consumer → PG，允许毫秒级延迟 |
| Token 黑名单 | 强一致（安全关键） | 写入 Redis 是请求的一部分，不异步 |

### 5.2 分布式锁

当多个实例可能同时操作同一资源时使用 Redis 分布式锁：

```go
// 示例：超级管理员初始化（仅允许一个实例执行）
lockKey := "lock:admin_init"
ttl := 30 * time.Second

if ok, _ := redis.SetNX(ctx, lockKey, "1", ttl); !ok {
    return errors.New("初始化已进行中，请稍后重试")
}
defer redis.Del(ctx, lockKey)

// 执行业务逻辑...
```

## 六、灾备演练

### 6.1 演练计划

| 场景 | 频率 | 验证目标 |
|------|------|---------|
| 应用实例宕机 | 每月 | LB 自动摘除，其余实例正常服务 |
| Redis Master 宕机 | 每季度 | Sentinel 自动切换，应用无感知 |
| PG 主库宕机 | 每季度 | 从库提升，读/写路由切换 |
| 全量数据恢复 | 每半年 | PITR 恢复到指定时间点 |

### 6.2 监控指标

| 指标 | 告警阈值 |
|------|---------|
| 应用实例存活数 | < 2 告警 |
| PG 主从延迟 | > 10s 告警 |
| Redis Sentinel 存活数 | < 2 告警 |
| PG 连接池使用率 | > 80% 告警 |
| 审计 Stream 积压 | > 10000 条告警 |
