## 模块概述

邀请模块处理用户被邀请加入组织/团队/小组后的确认流程。

**安全说明**：接受/拒绝邀请涉及账号加入组织/团队/小组，属于高风险操作。
- 查看邀请为公开接口（通过 account_id 参数查询，不暴露他人邀请）
- **接受/拒绝必须登录**（Bearer Token），且 Token 中的 account_id 必须与请求体 account_id 及邀请目标三者一致，否则返回 403

**依赖关系**：
- **数据库表**：`invitations`, `organization_members`, `team_members`, `group_members`
- **缓存**：Redis（权限缓存清除）

## 1. 查看待确认邀请

| 项目 | 内容 |
|------|------|
| 方法 | `GET` |
| 路径 | `/api/v1/auth/invitations` |
| 认证 | ❌ 公开 |

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account_id | string | ✅ | 查询目标账号的待确认邀请 |

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "items": [
      {
        "id": "uuid",
        "target_type": "organization",
        "target_name": "XX公司",
        "target_role": "RegularMember",
        "inviter_name": "李四",
        "expires_at": "2026-07-20T10:00:00Z",
        "created_at": "2026-07-13T10:00:00Z"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 邀请 ID |
| target_type | string (枚举) | `organization` / `team` / `group` |
| target_name | string | 目标名称（组织/团队/小组名称） |
| target_role | string (枚举) | 邀请的角色（如 `RegularMember`） |
| inviter_name | string | 邀请人昵称 |
| expires_at | string (ISO 8601) | 邀请过期时间 |
| created_at | string (ISO 8601) | 邀请创建时间 |

### 错误码

本接口为公开查询接口，除系统错误外不返回业务错误码。

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100003 | 必填字段缺失（account_id） |
| 500 | 805000 | 内部服务错误 |

## 2. 接受邀请

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/invitations/:id/accept` |
| 认证 | ✅ |
| HTTP Status | 200 |

### 请求 Header

```
Authorization: Bearer <access_token>
```

### 请求体

```json
{
  "account_id": "uuid"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account_id | string | ✅ | 必须与 access_token 中 account_id 一致，且与邀请目标匹配 |

### 业务流程

1. 校验 `Authorization` Header 中的 access_token
2. 提取 Token 中的 `account_id`
3. 校验三者一致：Token account_id = 请求体 account_id = 邀请目标 account_id
4. 校验邀请状态为 `pending` 且未过期
5. 创建 membership（角色 = invitation.target_role）
6. 邀请状态改为 `accepted`
7. 删除 Redis 权限缓存
8. 写入操作审计日志

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "member": {
      "id": "uuid",
      "target_type": "organization",
      "target_id": "uuid",
      "role": "RegularMember",
      "joined_at": "2026-07-13T10:00:00Z"
    }
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| member.id | string (UUID) | 成员关系 ID |
| member.target_type | string (枚举) | `organization` / `team` / `group` |
| member.target_id | string (UUID) | 目标 ID（组织/团队/小组 ID） |
| member.role | string (枚举) | 成员角色 |
| member.joined_at | string (ISO 8601) | 加入时间 |

### 错误响应

```json
{
  "code": 210002,
  "message": "account_id 与邀请不匹配",
  "detail": "Token 中的 account_id 与邀请目标账号不一致",
  "data": null,
  "pagination": null
}
```

### 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 403 | 603002 | 权限不足（account_id 不一致） |
| 404 | 210404 | 邀请不存在或已过期 |
| 400 | 210001 | 邀请状态不允许操作（非 pending） |
| 400 | 210002 | account_id 与邀请不匹配 |
