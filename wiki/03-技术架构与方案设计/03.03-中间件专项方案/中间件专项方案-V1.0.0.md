## 一、中间件链总览

所有受保护请求经过统一的中间件链，顺序严格固定，前一步失败即短路径返回。

```
请求
  → 1. RateLimiter           （登录接口限流，全局）
  → 2. JWTValidator          （校验 Access Token 签名 + 有效期）
  → 3. TokenBlacklist        （Redis 黑名单检查 jti）
  → 4. MembershipValidator   （校验当前 scope 的 membership + 层级继承）
  → 5. PermissionChecker     （校验角色权限点，Redis 缓存 + DB 兜底）
  → 6. 业务 Handler
```

**跳过规则**：
| 接口 | 跳过的中间件 | 理由 |
|------|-------------|------|
| POST /auth/register | 全部 | 公开接口 |
| POST /auth/login | JWT/Blacklist/Membership/Permission | 仅走 RateLimiter |
| POST /auth/send-code | RateLimiter | 仅走限流 |
| POST /auth/reset-password | JWT/Blacklist/Membership/Permission | 仅走 RateLimiter |
| GET /auth/invitations | 全部（除 RateLimiter） | 匿名可查看待确认邀请 |
| POST /auth/invitations/:id/accept | 全部（除 RateLimiter） | 匿名可确认邀请 |

## 二、RateLimiter 中间件

### 2.1 设计目标

| 场景 | 规则 |
|------|------|
| 登录失败限流 | 单 IP 5 次失败 / 5 分钟 → 锁定 15 分钟 |
| 验证码发送限流 | 单手机号 1 次 / 60 秒；同 IP 5 次 / 5 分钟 |

### 2.2 Redis 实现

```go
// internal/middleware/rate_limit.go

const (
    LoginFailKey     = "ratelimit:login_fail:{ip}"
    LoginFailWindow  = 5 * time.Minute
    LoginFailMax     = 5
    LoginLockKey     = "ratelimit:login_lock:{ip}"
    LoginLockWindow  = 15 * time.Minute
    VcodeSendKey     = "ratelimit:vcode:{target}"
    VcodeSendWindow  = 60 * time.Second
    VcodeIPKey       = "ratelimit:vcode_ip:{ip}"
    VcodeIPWindow    = 5 * time.Minute
    VcodeIPMax       = 5
)

func RateLimiter() gin.HandlerFunc {
    return func(c *gin.Context) {
        switch {
        case isLoginPath(c):
            handleLoginLimit(c)
        case isVcodePath(c):
            handleVcodeLimit(c)
        }
        c.Next()
    }
}

func handleLoginLimit(c *gin.Context) {
    ip := c.ClientIP()
    // 先检查是否处于锁定期
    locked, _ := redis.Get(ctx, LoginLockKey(ip))
    if locked != "" {
        c.JSON(429, errors.New(429001, "账号锁定", "15分钟内登录失败次数过多"))
        c.Abort()
        return
    }
    // 记录失败：在 Handler 中写入 incr，此处仅做前置检查
    count, _ := redis.Incr(ctx, LoginFailKey(ip), LoginFailWindow)
    if count > LoginFailMax {
        redis.Set(ctx, LoginLockKey(ip), "1", LoginLockWindow)
        c.JSON(429, errors.New(429001, "账号锁定", "登录失败次数过多，15分钟后重试"))
        c.Abort()
        return
    }
}
```

**登录失败记录时机**：在 Login Handler 中密码校验失败时执行 `redis.Incr(ratelimit:login_fail:{ip})`，成功时执行 `redis.Del` 重置。

### 2.3 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 429 | 429001 | 登录锁定 |
| 429 | 429002 | 验证码发送频率超限（手机号） |
| 429 | 429003 | 验证码发送频率超限（IP） |

## 三、JWTValidator 中间件

### 3.1 Token 格式

```
Authorization: Bearer <access_token>

Claims:
{
  "account_id": "uuid",
  "org_ids": ["uuid-1", "uuid-2"],
  "roles": {
    "uuid-1": "organization_core_admin",
    "uuid-2": "RegularMember"
  },
  "jti": "uuid",
  "exp": 1234567890,
  "iat": 1234567800,
  "alg": "HS256"
}
```

### 3.2 实现

```go
// internal/middleware/auth.go

func JWTValidator() gin.HandlerFunc {
    return func(c *gin.Context) {
        raw := c.GetHeader("Authorization")
        if raw == "" {
            c.JSON(401, errors.New(101001, "缺少 Token", "请在 Header 中提供 Authorization: Bearer <token>"))
            c.Abort()
            return
        }

        token, err := jwt.Parse(raw[7:])
        if err != nil {
            c.JSON(401, errors.New(101002, "Token 无效", err.Error()))
            c.Abort()
            return
        }

        // 注入 context，供后续中间件和 Handler 使用
        c.Set("account_id", token.AccountID)
        c.Set("org_ids", token.OrgIDs)
        c.Set("roles", token.Roles)
        c.Set("jti", token.JTI)
        c.Next()
    }
}

// ctx 中提取
func GetAccountID(c *gin.Context) uuid.UUID {
    return c.MustGet("account_id").(uuid.UUID)
}
```

### 3.3 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或过期 |
| 401 | 101003 | Token 已被撤销（黑名单） |

## 四、TokenBlacklist 中间件

### 4.1 设计

- 登出 / 改密 / 强制下线时，将 `jti` 写入 Redis：`token:blacklist:{jti}`，TTL = Access Token 剩余有效期
- 每次校验 Access Token 时查 Redis（< 1ms）
- 自然过期，无需主动清理

### 4.2 实现

```go
// internal/middleware/auth.go

func TokenBlacklist() gin.HandlerFunc {
    return func(c *gin.Context) {
        jti := c.MustGet("jti").(string)
        key := fmt.Sprintf("token:blacklist:%s", jti)

        exists, _ := redis.Exists(ctx, key)
        if exists {
            c.JSON(401, errors.New(101003, "Token 已失效", "请重新登录"))
            c.Abort()
            return
        }
        c.Next()
    }
}

// 主动写入（登出时调用）
func BlacklistToken(c *gin.Context) {
    jti := c.MustGet("jti").(string)
    key := fmt.Sprintf("token:blacklist:%s", jti)
    ttl := AccessTokenDuration - time.Since(iat) // 剩余有效期
    redis.Set(ctx, key, "1", ttl)
}
```

## 五、MembershipValidator 中间件

### 5.1 设计目标

校验当前账号在请求所声明的 scope（org/team/group）内是否有有效的 membership，支持层级继承。

**层级继承规则**：
| 角色 | 可访问的范围 |
|------|-------------|
| SuperAdmin | 全局所有组织/团队/小组 |
| 组织核心管理员（organization_core_admin） | 本组织 + 下属所有团队/小组 |
| 团队核心管理员（team_core_admin） | 本团队 + 下属所有小组 |
| 小组核心管理员（group_core_admin） | 仅本小组 |
| RegularMember | 仅所在 container |
| Public | 无 membership |

### 5.2 实现

```go
// internal/middleware/membership.go

func MembershipValidator() gin.HandlerFunc {
    return func(c *gin.Context) {
        accountID := GetAccountID(c)
        scope := parseScope(c) // 从 Header 解析 scope

        if isSuperAdmin(accountID) {
            c.Set("scope", scope)
            c.Next()
            return
        }

        member, err := repo.FindMembershipByHierarchy(accountID, scope)
        if err != nil {
            c.JSON(403, errors.New(603001, "无权限", "不在当前 scope 内"))
            c.Abort()
            return
        }

        c.Set("scope", scope)
        c.Set("role", member.Role)
        c.Set("container_id", member.ContainerID)
        c.Next()
    }
}

func parseScope(c *gin.Context) Scope {
    return Scope{
        OrgID:  c.GetHeader("X-Organization-ID"),
        TeamID: c.GetHeader("X-Team-ID"),
        GroupID: c.GetHeader("X-Group-ID"),
    }
}
```

### 5.3 层级回溯逻辑

```
给定 account 和请求 scope（如 team_id），检查顺序：
1. account 是否是该 team 的直接 member？→ 是则返回 (team, role)
2. account 是否是该 team 所属 org 的 organization_core_admin？→ 是则返回 (org, organization_core_admin)
3. account 是否是 SuperAdmin？→ 返回 (global, SuperAdmin)
4. 均不满足 → 403
```

## 六、PermissionChecker 中间件

### 6.1 设计

通过 `gin.HandlerFunc` 接受权限点 code 参数，校验当前角色是否拥有该权限。

```go
// 路由使用方式
r.POST("/org/:id/members", membershipMW, permissionMW("org.member.invite"), handler.InviteMember)
```

### 6.2 实现

```go
// internal/middleware/permission.go

func PermissionChecker(requiredPerm string) gin.HandlerFunc {
    return func(c *gin.Context) {
        accountID := GetAccountID(c)
        scope := c.MustGet("scope").(Scope)
        role := c.MustGet("role").(string)

        if isSuperAdmin(accountID) {
            c.Next()
            return
        }

        // 1. 尝试 Redis 缓存
        cacheKey := buildPermCacheKey(accountID, scope)
        permSet, found := redis.GetPermSet(ctx, cacheKey)
        if !found {
            // 2. DB 查询 + 写入缓存（含层级继承的权限点）
            permSet = repo.LoadPermissionSet(accountID, scope)
            redis.SetPermSet(ctx, cacheKey, permSet, 24*time.Hour)
        }

        // 3. 校验
        if !permSet.Contains(requiredPerm) {
            c.JSON(403, errors.New(603002, "权限不足", "无权限执行此操作"))
            c.Abort()
            return
        }
        c.Next()
    }
}
```

### 6.3 Redis 缓存标准

| 缓存 Key | 类型 | TTL | 失效时机 |
|----------|------|-----|---------|
| `perm:account:{account_id}:org:{org_id}` | String (JSON Set) | 24h | 角色变更 / 成员邀请 / 成员移除时 DEL |
| `perm:account:{account_id}:team:{team_id}` | String (JSON Set) | 24h | 同上 |
| `perm:account:{account_id}:group:{group_id}` | String (JSON Set) | 24h | 同上 |
| `token:blacklist:{jti}` | String | Access Token 剩余有效期 | 自然过期 |

### 6.4 缓存失效规则

以下操作必须主动删除相关缓存：

| 操作 | 需失效的 Key |
|------|-------------|
| 分配/降级角色 | 该 account 在所有受影响 scope 的 perm cache |
| 成员邀请/确认 | 该 account 在该 container 的 perm cache |
| 成员移除 | 该 account 在该 container 的 perm cache |
| 组织/团队/小组删除 | 该 container 下所有成员的 perm cache |

## 七、审计日志中间件

### 7.1 设计

操作审计日志通过 Gin 中间件在响应结束后写入 Redis Stream（异步，不阻塞主流程）。

```go
// internal/middleware/audit.go

func OperationAudit() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Next() // 先执行业务逻辑

        // 只记录成功操作
        if c.Writer.Status() >= 400 {
            return
        }

        record := AuditRecord{
            AccountID:    GetAccountID(c),
            OperationType: inferOperationType(c),
            TargetType:    inferTargetType(c),
            TargetID:      inferTargetID(c),
            OrgID:         c.GetHeader("X-Organization-ID"),
            Details:       buildDetails(c),
            IPAddress:     c.ClientIP(),
            CreatedAt:     time.Now(),
        }

        // 异步写入 Stream（Fire-and-forget，使用 goroutine）
        go func() {
            redis.XAdd(ctx, "audit:operation_stream", record)
        }()
    }
}

func inferOperationType(c *gin.Context) audit_operation_type {
    // 根据路由方法 + path 推断操作类型
    method := c.Request.Method
    path := c.FullPath()
    // ... 路由映射表
    return opType
}
```

### 7.2 Stream Consumer Group

```go
// cmd/worker/audit_consumer.go

// 启动独立的 consumer goroutine，从 Stream 消费并写入 DB
func StartAuditConsumer() {
    stream := "audit:operation_stream"
    group := "audit_writer"
    consumer := "worker-1"

    // 创建 consumer group（幂等）
    redis.XGroupCreateMkStream(ctx, stream, group, "0")

    for {
        messages, _ := redis.XReadGroup(ctx, stream, group, consumer, 0, 100, -1)
        for _, msg := range messages {
            record := parseRecord(msg)
            repo.InsertOperationAuditLog(record)
            redis.XAck(ctx, stream, group, msg.ID)
        }
        time.Sleep(100 * time.Millisecond)
    }
}
```

### 7.3 断点续写

- 使用 `XReadGroup` 的 `last_id: 0` 参数，只读取未确认的消息
- `XAck` 确认已处理消息，Redis 重启后从最后一个未 ACK 的消息继续
- 消费者崩溃不影响数据安全，Stream 中的数据持久化在 Redis AOF

## 八、中间件注册

```go
// cmd/xyfamily/main.go

// 公开路由（跳过所有中间件）
r.POST("/api/v1/auth/register", handler.Register)
r.POST("/api/v1/auth/login", middleware.RateLimiter(), handler.Login)
r.POST("/api/v1/auth/send-code", middleware.RateLimiter(), handler.SendCode)
r.POST("/api/v1/auth/reset-password", middleware.RateLimiter(), handler.ResetPassword)
r.GET("/api/v1/auth/invitations", handler.ListInvitations)
r.POST("/api/v1/auth/invitations/:id/accept", handler.AcceptInvitation)
r.POST("/api/v1/auth/invitations/:id/reject", handler.RejectInvitation)

// 受保护路由
auth := r.Group("/api/v1")
auth.Use(
    middleware.JWTValidator(),
    middleware.TokenBlacklist(),
    middleware.MembershipValidator(),
    middleware.OperationAudit(),
)
{
    // 需要权限校验的接口
    auth.POST("/org/:id/members/invitations",
        middleware.PermissionChecker("org.member.invite"), handler.InviteMember)
    // ... 其他接口
}
```
