## 一、注册流程

```mermaid
sequenceDiagram
    autonumber
    participant C as 客户端
    participant API as API (Gin)
    participant Redis as Redis
    participant PG as PostgreSQL

    C->>API: POST /api/v1/auth/send-code<br/>{target, type: "register"}
    API->>API: RateLimiter 校验 IP/手机号频率
    API->>Redis: 写入验证码 (TTL=5min)
    API->>C: 200 {code: 0}

    C->>API: POST /api/v1/auth/register<br/>{phone, code, password, nickname}
    API->>API: 校验密码强度 (≥12位)
    API->>Redis: 校验 + 消费验证码
    API->>PG: 检查 phone/email/username 唯一性
    API->>PG: INSERT accounts (bcrypt cost=12)
    API->>Redis: XAdd audit:login_stream (auth_register)
    API->>C: 201 {data: {account_id, nickname}}
```

## 二、登录流程

```mermaid
sequenceDiagram
    autonumber
    participant C as 客户端
    participant API as API (Gin)
    participant Redis as Redis
    participant PG as PostgreSQL

    C->>API: POST /api/v1/auth/login<br/>{phone, password}
    API->>API: RateLimiter 检查 IP 锁定状态
    API->>PG: 查询 accounts (phone)
    alt 账号不存在
        API->>Redis: XAdd audit:login_stream (failed)
        API->>C: 401 {code: 101007}
    else 密码错误
        API->>Redis: INCR ratelimit:login_fail:{ip}
        API->>Redis: XAdd audit:login_stream (failed)
        API->>API: 检查是否超过 5 次 → 锁定 15min
        API->>C: 401 {code: 101008}
    else 密码正确
        API->>PG: 查询 memberships → 获取 org_ids + roles
        API->>API: SignToken (Access 30min + Refresh 7d)
        API->>PG: INSERT sessions (refresh_token)
        API->>Redis: DEL ratelimit:login_fail:{ip} (重置)
        API->>Redis: XAdd audit:login_stream (success)
        API->>C: 200 {data: {access_token, refresh_token, expires_in}}
    end
```

## 三、权限校验流程

```mermaid
sequenceDiagram
    autonumber
    participant C as 客户端
    participant RL as RateLimiter
    participant JWT as JWTValidator
    participant BL as TokenBlacklist
    participant MV as MembershipValidator
    participant PC as PermissionChecker
    participant Redis as Redis
    participant PG as PostgreSQL
    participant H as 业务Handler

    C->>RL: POST /api/v1/org/:id/members<br/>Header: Authorization, X-Organization-ID
    RL->>RL: 登录接口才限流，此处跳过
    RL->>JWT: pass
    JWT->>JWT: 校验 Access Token 签名 + 有效期
    JWT->>C: 401 (Token 无效)
    JWT->>BL: pass
    BL->>Redis: EXISTS token:blacklist:{jti}
    Redis-->>BL: 0 (不在黑名单)
    BL->>MV: pass
    MV->>MV: 检查是否 SuperAdmin
    MV->>PG: 查询 memberships (account, scope)
    PG-->>MV: 返回 role + membership
    MV->>MV: 层级回溯 (account→group→team→org→global)
    MV->>PC: pass (注入 scope + role)
    PC->>Redis: GET perm:account:{id}:org:{org_id}
    alt 缓存命中
        Redis-->>PC: permSet
        PC->>PC: 校验 code ∈ permSet
        PC->>H: pass
    else 缓存未命中
        PC->>PG: 查询 roles + role_permissions (level <= 当前)
        PG-->>PC: 返回权限点列表
        PC->>Redis: SET perm:account:{id}:org:{org_id} (TTL=24h)
        PC->>PC: 校验 code ∈ permSet
        PC->>H: pass
    end
    H->>H: 执行业务逻辑
    H->>C: 200
```

## 四、Token 刷新流程（Refresh Rotation）

```mermaid
sequenceDiagram
    autonumber
    participant C as 客户端
    participant API as API (Gin)
    participant PG as PostgreSQL

    Note over C,PG: Access Token 过期后
    C->>API: POST /api/v1/auth/refresh<br/>{refresh_token}
    API->>PG: 查询 sessions (refresh_token)
    alt Session 不存在
        API->>C: 401 {code: 101004}
    else Session 已撤销
        API->>C: 401 {code: 101005}
    else Session 已过期
        API->>PG: DELETE session
        API->>C: 401 {code: 101006}
    else 有效
        API->>PG: UPDATE sessions SET revoked=TRUE (撤销旧)
        API->>API: SignToken (新 Access + 新 Refresh)
        API->>PG: INSERT sessions (新 refresh_token)
        API->>C: 200 {data: {access_token, refresh_token, expires_in}}
    end
```

## 五、成员邀请流程（统一邀请流）

```mermaid
sequenceDiagram
    autonumber
    participant Admin as 邀请方 (Admin)
    participant API as API (Gin)
    participant PG as PostgreSQL
    participant Invitee as 被邀请方

    Admin->>API: POST /api/v1/org/:id/members/invitations<br/>{identifier, role}
    API->>API: PermissionChecker("org.member.invite")
    API->>PG: 查询 accounts (identifier) → invitee_account_id
    API->>PG: INSERT invitations<br/>(target_type: "organization",<br/>target_id, role, expires_at: +7d)
    API->>PG: 写入 operation_audit_logs (org_member_invite)
    API->>Admin: 201 {data: {invitation_id, expires_at}}

    Note over Invitee,API: 被邀请方查看 + 确认
    Invitee->>API: GET /api/v1/auth/invitations
    API->>PG: 查询 invitations (invitee_account_id, status="pending")
    API->>Invitee: 200 {data: [invitations]}

    Invitee->>API: POST /api/v1/auth/invitations/:id/accept
    API->>PG: 校验 invitation 有效 + 未过期
    API->>PG: INSERT organization_members (role)
    API->>PG: UPDATE invitations SET status="accepted"
    API->>PG: 删除 Redis perm cache (被邀请方)
    API->>Invitee: 200 {data: {member}}

    alt 拒绝
        Invitee->>API: POST /api/v1/auth/invitations/:id/reject
        API->>PG: UPDATE invitations SET status="rejected"
        API->>Invitee: 200
    end
```
