## 一、个人信息（PII）识别

### 1.1 敏感字段清单

| 字段 | 所在表 | 敏感等级 | 说明 |
|------|--------|---------|------|
| phone | accounts | 🔴 高 | 手机号，唯一标识 |
| email | accounts | 🔴 高 | 邮箱，唯一标识 |
| username | accounts | 🟡 中 | 用户名 |
| nickname | accounts | 🟢 低 | 昵称，可公开 |
| password_hash | accounts | 🔴 高 | bcrypt 哈希，不可逆 |
| avatar_url | accounts | 🟢 低 | 头像 URL，可公开 |
| ip_address | login_audit_logs | 🟡 中 | 登录 IP，关联用户行为 |
| user_agent | login_audit_logs | 🟢 低 | 浏览器信息 |
| device_info | sessions | 🟡 中 | 设备信息 |

### 1.2 非敏感字段

| 字段 | 所在表 | 说明 |
|------|--------|------|
| account_id | accounts | UUID，对外可见 |
| id | accounts | UUID，数据库主键，对外隐藏 |
| status | accounts | 账号状态 |
| org_id | 各表 | 组织 ID |
| operation_type | operation_audit_logs | 操作类型枚举 |
| details | operation_audit_logs | JSONB，不存 PII |

## 二、账号注销与匿名化

### 2.1 注销流程

```
账号申请注销
  → 状态变更为 deactivating（30 天宽限期）
  → membership role 清空
  → 宽限期内可恢复
  → 宽限期过后执行匿名化
```

### 2.2 匿名化规则

| 字段 | 匿名化方式 | 说明 |
|------|-----------|------|
| phone | 替换为 `anon_{account_id}` | 原始值不可恢复 |
| email | 替换为 `anon_{account_id}@anon.xyfamily` | |
| username | 替换为 `anon_{account_id}` | |
| nickname | 替换为 `匿名用户` | |
| password_hash | 清空 | 注销后不可登录 |
| membership.role | 清空 | 保留 membership 记录 |
| avatar_url | 清空 | |

### 2.3 核心约束

- **绝不硬删除** accounts 表记录，保护 referential integrity
- membership 记录保留但 role 清空
- audit 日志中的 account_id 保留，对应 PII 已匿名化（审计上下文不丢失）

## 三、审计日志合规

### 3.1 审计日志要求

| 要求 | 说明 |
|------|------|
| 完整性 | 所有登录 + 操作行为均有记录，不可篡改 |
| 不可逆 | 审计日志仅 append，不 update/delete |
| 保留期 | 1 年 |
| 清理方式 | 超期记录标记 `deleted_at`（软删除），不清理硬删除 |
| 权限 | 仅 SuperAdmin + 组织管理员可查本组织日志 |

### 3.2 审计日志字段合规

| 字段 | 是否含 PII | 合规要求 |
|------|-----------|---------|
| account_id | ❌ | UUID，不关联 PII |
| login_method | ❌ | 枚举值 |
| ip_address | ✅ | 登录 IP，用于安全审计 |
| operation_type | ❌ | 枚举值 |
| details (JSONB) | ❌ | 不得包含密码、手机号等 PII |

## 四、数据保留策略

### 4.1 各表保留策略

| 表 | 保留期 | 清理方式 |
|----|--------|---------|
| accounts | 永久（匿名化） | 注销后匿名化，不硬删除 |
| organization_members | 永久 | 成员关系保留 |
| teams / groups | 永久（软删除） | deleted_at 标记 |
| login_audit_logs | 1 年 | 超期 soft delete |
| operation_audit_logs | 1 年 | 超期 soft delete |
| verification_codes | 5 天 | 过期自动清理 |
| sessions | Refresh Token 过期后清理 | 自然过期 |

### 4.2 定期清理任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 验证码清理 | 每日 | 删除 `expires_at < NOW()` 的记录 |
| Session 清理 | 每日 | 删除 `expires_at < NOW()` 的记录 |
| 审计日志清理 | 每日 | 标记 `created_at < NOW() - 1 year` 的记录为 `deleted_at` |
| 账号匿名化 | 每日 | 扫描 `expires_at < NOW()` 且 status=deactivating 的账号 |

## 五、数据库安全

### 5.1 连接安全

| 措施 | 说明 |
|------|------|
| 传输加密 | TLS 1.2+ 连接 PostgreSQL |
| 访问控制 | 应用专用数据库账号，权限最小化 |
| 密码管理 | 数据库密码通过环境变量注入，禁止硬编码 |

### 5.2 查询安全

| 措施 | 说明 |
|------|------|
| 参数化查询 | 所有 SQL 使用 GORM 参数化，禁止字符串拼接 |
| 多租户隔离 | 所有查询通过 `WithOrgScope` 强制带 org_id |
| 索引设计 | 避免全表扫描，性能安全兼顾 |

---

## 六、密码安全

| 要求 | 说明 |
|------|------|
| 存储 | bcrypt cost=12 |
| 传输 | HTTPS，密码不在 URL/Header 中明文传输 |
| 强度 | ≥12 位，含大小写字母 + 数字 + 特殊字符 |
| 日志 | 密码不在任何日志中明文记录 |
| 重置 | 通过验证码重置，新旧密码不可相同 |
