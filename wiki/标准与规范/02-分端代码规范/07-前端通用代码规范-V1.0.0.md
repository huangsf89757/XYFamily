## 一、各平台概览

| 平台 | 语言/框架 | 定位 |
|------|-----------|------|
| 安卓 | Kotlin + Jetpack Compose | Android 原生 App |
| iOS | Swift + SwiftUI | iOS 原生 App |
| H5 | React + TypeScript | 移动 H5 页面（浏览器） |
| 微信小程序 | 原生 / uni-app | 微信生态小程序 |
| Web | React + TypeScript | 管理后台（PC 浏览器） |

**原则**：跨平台 UI 组件使用共享设计语言（Design System），API 调用层统一封装，平台特有功能各端自实现。

## 二、API 调用统一规范

### 2.1 请求封装

所有平台使用统一的 HTTP Client 封装，统一处理：
- 自动携带 `Authorization` Header
- 自动携带 `X-Organization-ID` / `X-Team-ID` / `X-Group-ID` Header
- Token 过期自动刷新（Access Token 30min → 使用 Refresh Token 刷新）
- 统一错误处理（HTTP 401 → 跳转登录）

```typescript
// 各平台统一封装接口（TypeScript 示例，各端按语言实现等价逻辑）
interface RequestOptions {
  method: 'GET' | 'POST' | 'PUT' | 'DELETE';
  url: string;
  headers?: Record<string, string>;
  body?: any;
  skipAuth?: boolean;  // 公开接口跳过认证
}

interface APIResponse<T> {
  code: number;       // 0=成功，10xxx-80xxx=业务错误
  message: string;    // 错误摘要
  detail?: string;    // 详细错误
  data?: T;
}
```

### 2.2 错误码处理（跨平台统一）

| HTTP Status | 业务 Code 段 | 处理策略 |
|-------------|------------|---------|
| 401 | 101001-101009 | 统一跳转登录 |
| 401 | 101004-101006 | Refresh Token 失效 → 强制重新登录 |
| 429 | 104290-114291 | 展示限流提示，禁用操作 |
| 403 | 603001-603002 | 展示"无权限"提示 |
| 500 | 805000-805003 | 展示"服务异常，请稍后重试" |

**各平台错误提示统一文案**：
- 401 Token 过期：`"登录已过期，请重新登录"`
- 429 限流：`"操作过于频繁，请稍后再试"`
- 403 无权限：`"当前账号无权限执行此操作"`

## 三、多租户上下文传递

### 3.1 Header 传递规则

| Header | 必填时机 | 说明 |
|--------|---------|------|
| `X-Organization-ID` | 操作组织资源时 | 从当前选中的组织上下文获取 |
| `X-Team-ID` | 操作团队资源时 | 从当前选中的团队上下文获取 |
| `X-Group-ID` | 操作小组资源时 | 从当前选中的小组上下文获取 |

### 3.2 多组织切换

用户可能同时属于多个组织，前端需支持组织切换：
- 登录后展示用户所属组织列表
- 切换组织时更新全局上下文（`currentOrgId`）
- 切换组织后 Token Claims 中的 `org_ids` 不变，但 Header 中的 `X-Organization-ID` 更新

## 四、本地存储规范

### 4.1 存储内容

| 存储项 | 平台 | 存储方式 |
|--------|------|---------|
| Access Token | 全部 | 内存（不持久化） |
| Refresh Token | 全部 | 平台安全存储（Android Keystore / iOS Keychain / SecureStorage） |
| 用户信息 | 全部 | 本地持久化（加密） |
| 当前组织 ID | 全部 | 本地持久化 |

### 4.2 安全要求

- Access Token 不持久化到本地存储（防持久化泄露）
- Refresh Token 使用平台安全存储，不可明文存储
- 登出时清除所有本地 Token

## 五、Token 自动刷新策略

```
请求 → 401 (Access Token 过期)
  → 使用 Refresh Token 请求 /auth/refresh
  → 刷新成功：用新 Access Token 重试原请求
  → 刷新失败（Refresh Token 已撤销/过期）：清除本地 Token → 跳转登录
```

**并发处理**：多个请求同时过期时，只发起一次刷新，其他请求等待刷新结果。

## 六、命名规范（跨平台统一）

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件/类名 | PascalCase | `UserProfile`, `OrganizationList` |
| 函数/方法 | camelCase | `fetchUser`, `handleLogin` |
| 常量 | UPPER_SNAKE_CASE | `MAX_LOGIN_ATTEMPTS` |
| 布尔值 | is/has/can/should 前缀 | `isLoading`, `hasPermission` |
| 事件名 | on + PascalCase | `onLoginSuccess`, `onLogout` |
| 状态变量 | 直接语义化 | `currentOrg`, `memberList` |

## 七、响应式设计

| 平台 | 分辨率策略 |
|------|-----------|
| 安卓 | dp 适配，density 自动处理 |
| iOS | pt 适配，safe area 自动处理 |
| H5 | vw/vh + rem，最小宽度 320px |
| 微信小程序 | rpx 适配（750rpx = 屏幕宽度） |
| Web | 响应式布局，最小宽度 1200px（管理后台） |
