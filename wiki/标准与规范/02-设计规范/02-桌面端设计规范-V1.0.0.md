## 一、设计概述

### 1.1 桌面端定位

XYFamily 桌面客户端是面向管理员和重度业务用户的原生桌面应用，通过 Electron 或 Tauri 封装 Web 端能力，在保留 Web 端完整功能的基础上，提供更接近原生应用的窗口管理、系统集成与离线体验。

| 维度 | 说明 |
|------|------|
| 技术方案 | Electron / Tauri 封装 Web 端（基于 Web 管理后台代码复用） |
| 目标平台 | macOS（11 Big Sur 及以上）、Windows（10 及以上） |
| 用户角色 | 组织管理员、超级管理员、需要高频操作权限与组织数据的用户 |
| 核心价值 | 原生窗口体验、系统集成、快捷键提效、离线可用、开机自启 |
| 分发方式 | 官网下载安装包（.dmg / .exe / .msi）、自动更新 |

### 1.2 与 Web 端的关系

桌面端与 Web 端共享同一套设计令牌体系与组件库，差异仅体现在原生能力的增强与交互模式的平台化适配。

| 层面 | 共享内容 | 桌面端增强 |
|------|----------|-----------|
| 设计令牌 | 色彩、字体、间距、圆角、阴影、主题、图标全部复用全局规范 | 无变更 |
| 组件库 | 按钮、表单、表格、卡片、对话框等基础组件完全复用 | 增加原生右键菜单、系统文件对话框、托盘菜单 |
| 布局体系 | 侧边栏 + 内容区结构一致 | 增加原生标题栏融合、窗口控制按钮 |
| 交互模式 | 鼠标点击、Hover、表单交互一致 | 增加 Hover 丰富度、右键菜单、拖拽排序、键盘快捷键全覆盖 |
| 数据流 | 共享同一套后端 API 与状态管理 | 增加本地缓存层，支持离线降级 |
| 原生能力 | 无 | 窗口管理、系统托盘、原生通知、深度链接、开机自启、自动更新 |

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 原生优先 | 能力允许时优先使用平台原生控件与交互模式，降低用户学习成本 |
| 平台一致性 | 同一功能在 macOS 与 Windows 上遵循各自平台习惯（如快捷键修饰键、菜单栏位置） |
| Web 复用最大化 | 业务界面复用 Web 端实现，避免重复开发，仅在窗口外壳和原生桥接层做适配 |
| 键盘友好 | 桌面端用户重度依赖键盘，所有高频操作必须提供快捷键 |
| 离线韧性 | 网络中断时保留可用状态，恢复后自动同步 |

## 二、窗口管理

### 2.1 窗口尺寸

| 参数 | 值 | 说明 |
|------|----|------|
| 默认宽度 | 1440 px | 首次启动窗口宽度 |
| 默认高度 | 900 px | 首次启动窗口高度 |
| 最小宽度 | 1024 px | 用户缩放下限，低于此值布局可能错乱 |
| 最小高度 | 600 px | 用户缩放下限 |
| 最大宽度 | 不限制 | 支持超宽屏自适应 |
| 最大高度 | 不限制 | 支持超高屏自适应 |

> 首次启动以 1440×900 居中显示；非首次启动恢复上次窗口位置与尺寸（见 2.5 窗口恢复）。

### 2.2 窗口标题栏

桌面端按平台采用不同的标题栏方案，保持与操作系统原生应用一致的视觉表现。

#### macOS 标题栏

| 属性 | 值 | 说明 |
|------|----|------|
| 标题栏样式 | `hiddenInset`（Tauri）/ `titleBarStyle: 'hiddenInset'`（Electron） | 隐藏标题文字，保留交通灯按钮 |
| 交通灯按钮 | 系统原生（关闭、最小化、全屏） | 位于左上角，使用系统默认样式 |
| 标题栏区域 | 透明，内容延伸至标题栏下方 | 侧边栏顶部留出交通灯按钮避让区域（约 80 px 宽） |
| 拖拽区域 | 标题栏空白区域可拖拽移动窗口 | 通过 `app-region: drag` CSS 属性实现 |

#### Windows 标题栏

| 属性 | 值 | 说明 |
|------|----|------|
| 标题栏样式 | 自定义标题栏（隐藏系统原生标题栏） | 通过 `frame: false` 实现完全自定义 |
| 窗口控制按钮 | 自定义（最小化、最大化/还原、关闭） | 位于右上角，样式遵循 Fluent Design |
| 标题栏高度 | 40 px | 包含应用图标 + 应用名称 + 窗口控制按钮 |
| 拖拽区域 | 标题栏整体可拖拽 | 控制按钮区域排除拖拽 |
| 最大化行为 | 最大化时窗口边缘吸附屏幕边界 | 不遮挡系统任务栏 |

### 2.3 多窗口支持

桌面端支持多个独立窗口，各窗口独立管理生命周期与状态。

| 窗口 | 触发方式 | 尺寸 | 模态 | 说明 |
|------|----------|------|------|------|
| 主窗口 | 应用启动 | 默认 1440×900 | 非模态 | 核心业务界面，承载侧边栏与内容区 |
| 设置 / 偏好设置窗口 | 菜单栏「偏好设置」/ 快捷键 | 720×560 | 模态（macOS 为独立窗口非模态） | 应用偏好、主题、通知、快捷键、关于 |
| 关于窗口 | 菜单栏「关于」/ Dock 右键 | 480×360 | 模态 | 显示应用版本、构建号、版权信息 |
| 帮助窗口 | 菜单栏「帮助」 | 1000×700 | 非模态 | 打开在线帮助文档 |

> macOS 上「偏好设置」与「关于」遵循平台惯例以独立面板（Panel）形式弹出；Windows 上以模态对话框形式弹出。

### 2.4 全屏模式

| 平台 | 全屏方式 | 说明 |
|------|----------|------|
| macOS | 绿色交通灯按钮 / 菜单栏「进入全屏」 / `Cmd+Ctrl+F` | 原生全屏，进入独立 Space |
| Windows | 最大化按钮 / `F11` | 窗口最大化铺满工作区（不含任务栏） |

全屏模式下行为约定：

| 行为 | 说明 |
|------|------|
| 侧边栏 | 保留显示，可通过快捷键收起/展开 |
| 菜单栏（macOS） | 鼠标移至屏幕顶部时自动显示 |
| 退出全屏 | macOS：`Esc` 或绿色按钮；Windows：`F11` 或最大化按钮 |

### 2.5 窗口恢复

应用退出时持久化窗口状态，下次启动时恢复。

| 持久化字段 | 存储位置 | 说明 |
|-----------|----------|------|
| 窗口 X 坐标 | 本地配置文件 | 窗口左上角横坐标 |
| 窗口 Y 坐标 | 本地配置文件 | 窗口左上角纵坐标 |
| 窗口宽度 | 本地配置文件 | 上次关闭时的宽度 |
| 窗口高度 | 本地配置文件 | 上次关闭时的高度 |
| 是否最大化 | 本地配置文件 | 标识窗口是否处于最大化状态 |
| 是否全屏 | 本地配置文件 | 标识窗口是否处于全屏状态 |

恢复逻辑：

| 场景 | 处理方式 |
|------|----------|
| 坐标在屏幕可视范围内 | 直接恢复上次位置和尺寸 |
| 坐标超出屏幕范围（如外接显示器已断开） | 居中显示于主屏幕，使用默认尺寸 |
| 首次启动 | 居中显示，使用默认尺寸 1440×900 |
| 上次为最大化状态 | 启动后直接最大化，并记录非最大化时的尺寸 |

## 三、原生菜单栏

### 3.1 macOS 菜单栏结构

macOS 应用菜单栏固定显示在屏幕顶部，遵循 Apple Human Interface Guidelines 结构：

```
XYFamily │ 文件 │ 编辑 │ 视图 │ 窗口 │ 帮助
```

Windows 端不显示传统菜单栏条，通过自定义标题栏提供菜单入口（或使用 Alt 键激活隐藏菜单栏）：

## 四、侧边栏与导航

### 4.1 与 Web 端共享侧边栏设计

桌面端侧边栏与 Web 管理后台共享同一套导航结构、图标和交互逻辑，复用全局设计令牌中的 `--sidebar` 系列语义令牌。

| 属性 | 值 | 说明 |
|------|----|------|
| 侧边栏宽度 | 260 px（展开）/ 72 px（收起） | 与 Web 端一致 |
| 导航结构 | 与 Web 端完全一致 | 组织管理、成员管理、权限管理、审计日志、超级管理员等 |
| 图标系统 | Lucide Icons，icon-lg (24 px) | 复用全局图标令牌 |
| 选中态 | `--sidebar-primary` 背景 + `--sidebar-primary-foreground` 文字 | 复用语义令牌 |
| 折叠/展开 | 侧边栏底部切换按钮 / 快捷键 | 持久化折叠状态 |

### 4.2 macOS 侧边栏样式

macOS 侧边栏采用原生 Vibrancy 半透明效果，呈现 macOS Big Sur+ 风格的毛玻璃质感。

| 属性 | 值 | 说明 |
|------|----|------|
| 背景效果 | Vibrancy（`NSVisualEffectView`） | 毛玻璃半透明，透出桌面壁纸 |
| 材质类型 | `sidebar` 材质 | macOS 侧边栏专用材质，自动适配明暗模式 |
| 文字渲染 | 系统次级标签文字色 | 半透明效果下自动调整文字对比度 |
| 分割线 | 无可见分割线 | 侧边栏与内容区通过材质差异自然分隔 |
| 交通灯避让 | 顶部留出 80 px 安全区 | 避免导航项被交通灯按钮遮挡 |
| 明暗模式 | 跟随系统自动切换 | Vibrancy 材质自动适配 |

### 4.3 Windows 侧边栏样式

Windows 侧边栏采用不透明背景，可选启用 Mica 或 Acrylic 材质效果。

| 属性 | 值 | 说明 |
|------|----|------|
| 背景效果 | 不透明（默认）/ Mica（可选）/ Acrylic（可选） | Mica 透出桌面壁纸色调；Acrylic 为毛玻璃模糊 |
| 背景色 | `--sidebar` 语义令牌 | 不启用材质时使用纯色背景 |
| 文字渲染 | `--sidebar-foreground` 语义令牌 | 标准不透明文字色 |
| 分割线 | `--sidebar-border`，1 px | 侧边栏右侧与内容区之间的分割线 |
| 标题栏融合 | 侧边栏顶部与自定义标题栏视觉融合 | 标题栏区域与侧边栏背景一致 |
| 明暗模式 | 跟随系统或用户设置切换 | Mica/Acrylic 材质自动适配明暗 |

### 4.4 导航项规范

| 元素 | 规范 | 说明 |
|------|------|------|
| 导航项高度 | 40 px | 紧凑密度，适合桌面端鼠标操作 |
| 导航项圆角 | `--brand-radius-sm` (8 px) | 与全局圆角一致 |
| 图标尺寸 | icon-lg (24 px) | 复用全局图标尺寸 |
| 文字字号 | sm (14 px) | 复用全局字号 |
| 选中态 | 主色背景 + 白色文字 | 使用 `--sidebar-primary` 令牌 |
| Hover 态 | `--sidebar-accent` 背景 | 轻微背景着色 |
| 分组标题 | xs (12 px)，`--muted-foreground` | 分组导航的标题文字 |
| Badge 角标 | 12 px 圆点或数字气泡 | 显示未读消息或待办数量 |

## 五、键盘快捷键

### 5.1 快捷键设计原则

| 原则 | 说明 |
|------|------|
| 平台习惯优先 | macOS 使用 `Cmd` 作为主修饰键，Windows 使用 `Ctrl` |
| 禁止冲突 | 快捷键不得与系统级快捷键冲突（如 `Cmd+Space`、`Cmd+Tab`、`Ctrl+Alt+Del` 等） |
| 全覆盖 | 所有高频操作必须提供快捷键，桌面端用户重度依赖键盘 |
| 可发现性 | 通过「帮助 → 键盘快捷键参考」提供完整速查表 |
| 可自定义 | 偏好设置中支持冲突检测与部分快捷键自定义 |

### 5.2 全局快捷键表

| 功能 | macOS 快捷键 | Windows 快捷键 | 说明 |
|------|-------------|---------------|------|
| 全局搜索 / 命令面板 | `Cmd+K` | `Ctrl+K` | 打开全局搜索与快速跳转面板 |
| 新建 | `Cmd+N` | `Ctrl+N` | 新建当前上下文对象 |
| 刷新当前页面 | `Cmd+R` | `Ctrl+R` | 刷新当前页面数据 |
| 关闭窗口 | `Cmd+W` | `Ctrl+W` | 关闭当前窗口 |
| 退出应用 | `Cmd+Q` | — | 退出应用（仅 macOS） |
| 偏好设置 | `Cmd+,` | `Ctrl+,` | 打开偏好设置 |
| 查找 | `Cmd+F` | `Ctrl+F` | 当前页面查找 |
| 撤销 | `Cmd+Z` | `Ctrl+Z` | 撤销操作 |
| 重做 | `Cmd+Shift+Z` | `Ctrl+Y` | 重做操作 |
| 复制 | `Cmd+C` | `Ctrl+C` | 复制选中内容 |
| 剪切 | `Cmd+X` | `Ctrl+X` | 剪切选中内容 |
| 粘贴 | `Cmd+V` | `Ctrl+V` | 粘贴内容 |

| 属性 | 值 |
|------|----|

支持的深度链接路由：

| 链接 | 目标页面 | 说明 |
|------|----------|------|
| `xyfamily://` | 主窗口首页 | 唤起应用并显示首页 |
| `xyfamily://login` | 登录页 | 未登录时跳转登录 |

深度链接行为约定：

| 场景 | 处理方式 |
|------|----------|

| 应用已启动（前台） | 在主窗口跳转目标页面 |
| 用户未登录 | 启动/唤起 → 登录页 → 登录成功后跳转目标页面 |
| 链接无效 | 显示错误提示，停留在当前页面 |

## 七、离线能力

### 7.1 本地缓存策略

桌面端在本地维护缓存层，用于离线降级和快速恢复。

| 缓存类型 | 存储位置 | 持久化 | 说明 |
|----------|----------|--------|------|
| Token / 会话信息 | 系统 Keychain（macOS）/ Credential Manager（Windows） | 是 | 加密存储，安全可靠 |
| 用户偏好设置 | 本地配置文件 | 是 | 主题、语言、窗口尺寸、侧边栏状态 |
| 主题设置 | 本地配置文件 | 是 | 明暗模式偏好 |
| 导航菜单结构 | IndexedDB / 本地存储 | 是 | 缓存菜单结构，离线可展示框架 |
| 最近列表数据 | IndexedDB | 否（过期失效） | 组织/成员列表的最近一次请求结果 |
| 表单草稿 | IndexedDB | 是 | 未提交的长表单自动保存草稿 |

缓存失效策略：

| 数据类型 | 失效条件 | 说明 |
|----------|----------|------|
| Token | 服务端返回 401 | Token 过期后清除并跳转登录 |
| 导航菜单结构 | 应用版本更新 | 升级后重新拉取 |
| 列表数据 | 网络恢复后刷新 | 重新连接后自动请求最新数据 |
| 表单草稿 | 表单提交成功后 | 提交成功后清除对应草稿 |

### 7.2 离线提示 UI

网络断开时，桌面端在界面顶部显示非阻塞式提示条。

| 属性 | 值 | 说明 |
|------|----|------|
| 提示位置 | 内容区顶部，侧边栏右侧 | 固定吸顶 |
| 提示样式 | Warning 状态色背景 + 警告图标 | 使用 `--warning` 语义令牌 |
| 提示文案 | 「网络连接已断开，正在以离线模式运行」 | 简洁明确 |
| 交互行为 | 非阻塞，用户可继续浏览缓存数据 | 不遮挡主界面操作 |
| 关闭方式 | 网络恢复后自动消失 | 无需手动关闭 |
| 高度 | 40 px | 紧凑高度，最小化对内容区的占用 |

网络状态切换时的 UI 反馈：

| 状态切换 | UI 反馈 | 说明 |
|----------|----------|------|
| 在线 → 离线 | 顶部显示离线提示条 | 立即显示 |
| 离线 → 在线 | 提示条消失 + 自动刷新当前页面数据 | 提示条淡出，数据请求触发 |
| 离线时操作需联网 | 表单提交按钮置灰 + Tooltip 提示 | 防止用户提交无法同步的数据 |
| 离线时访问未缓存页面 | 显示空状态：「当前页面在离线模式下不可用」 | 引导用户等待网络恢复 |

### 7.3 数据同步策略

| 策略 | 说明 |
|------|------|
| 自动刷新 | 网络恢复后自动刷新当前页面数据 |
| 增量同步 | 恢复连接后请求增量数据（基于上次同步时间戳），而非全量拉取 |
| 草稿恢复 | 网络恢复后提示用户恢复未提交的表单草稿 |
| 冲突处理 | 数据冲突时以服务端为准，本地缓存覆盖更新 |
| 同步状态指示 | 列表页右上角显示「数据已更新」轻提示（Toast），告知用户数据已同步 |
| 后台同步 | 应用在后台时保持长连接，收到推送后静默更新本地缓存 |

## 九、自动更新

### 9.1 更新检查频率

| 检查时机 | 频率 | 说明 |
|----------|------|------|
| 应用启动时 | 每次启动 | 后台静默检查，不阻塞启动 |
| 运行中定时检查 | 每 4 小时 | 应用长时间运行时定期检查 |
| 手动检查 | 用户触发 | 偏好设置 → 关于 → 检查更新 |
| 强制更新推送 | 服务端控制 | 安全补丁等紧急更新可强制推送 |

### 9.2 更新提示 UI

更新检查发现新版本时，使用非阻塞式横幅提示，不打断用户当前操作。

| 属性 | 值 | 说明 |
|------|----|------|
| 提示位置 | 内容区顶部（离线提示条下方） | 固定吸顶，非模态 |
| 提示样式 | Info 状态色浅色背景 + 下载图标 | 使用 `--info` 系列语义令牌 |
| 提示文案 | 「发现新版本 V1.1.0，点击查看更新内容」 | 包含版本号 |
| 交互按钮 | 「立即更新」「稍后提醒」「查看详情」 | 三个操作选项 |
| 关闭方式 | 用户点击「稍后提醒」后 4 小时内不再提示 | 非阻塞，可手动关闭 |
| 高度 | 48 px | 略高于离线提示条 |

更新提示状态：

| 状态 | UI 表现 | 说明 |
|------|----------|------|
| 发现新版本 | 显示更新横幅 | 提示用户有可用更新 |
| 下载中 | 横幅显示进度条 + 「下载中…XX%」 | 显示下载进度 |
| 下载完成 | 横幅变为「更新已就绪，重启后生效」+ 「立即重启」按钮 | 等待用户重启 |
| 更新失败 | 横幅变为 Error 样式 + 「更新失败，点击重试」 | 允许重试 |
| 强制更新 | 弹出模态对话框，不可跳过 | 安全补丁等紧急场景 |

### 9.3 更新流程

| 阶段 | 操作 | 说明 |
|------|------|------|
| 1. 检查更新 | 应用向更新服务请求最新版本信息 | 对比本地版本号 |
| 2. 版本对比 | 本地版本 < 服务端版本时触发更新流程 | 比较语义化版本号 |
| 3. 显示提示 | 展示更新横幅 + 更新日志摘要 | 用户决定是否更新 |
| 4. 下载安装包 | 后台下载差量或全量更新包 | 显示下载进度 |
| 5. 校验签名 | 下载完成后校验安装包签名与哈希 | 确保完整性与安全性 |
| 6. 提示重启 | 下载校验完成后提示用户重启 | 非强制（强制更新除外） |
| 7. 应用更新 | 用户重启时应用更新包 | 替换旧版本文件 |
| 8. 启动新版 | 更新完成后启动新版本 | 显示「更新成功」欢迎提示 |

> 差量更新优先：优先下载差量更新包以减少下载量，差量应用失败时回退为全量更新。
