## 一、设计概述

### 1.1 小程序定位

XYFamily 微信小程序定位为**轻量级入口**，依托微信生态优势为用户提供快捷、即用即走的服务体验。

| 维度 | 说明 |
|------|------|
| 轻量级入口 | 无需下载安装，扫码或搜索即开即用，降低用户使用门槛 |
| 微信生态集成 | 深度整合微信登录、订阅消息、分享邀请、小程序码等原生能力 |
| 扫码加入组织 | 通过微信扫一扫扫描组织邀请码/小程序码，快速完成加入组织流程 |
| 消息通知 | 利用订阅消息推送组织动态、权限变更、审批提醒等通知 |
| 功能聚焦 | 提供核心高频功能（查看组织信息、接收通知、扫码加入），复杂管理操作引导至 Web 端或移动 App |

### 1.2 技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | 微信小程序原生框架 | 原生框架性能最优，推荐 V1.0 采用原生开发 |
| 备选框架 | uni-app | 如需多端复用可迁移至 uni-app，设计规范保持一致 |
| 语言 | TypeScript / JavaScript | 推荐使用 TypeScript 提升代码可维护性 |
| 样式 | WXSS（WeiXin Style Sheets） | 支持 rpx 单位、CSS 变量（部分基础库版本） |
| 脚本 | WXS（WeiXin Script） | 用于在 WXML 中执行表达式计算 |
| 网络 | wx.request | 封装统一请求层，支持 Token 刷新与队列管理 |
| 状态管理 | 全局 App 实例 / 状态库 | 轻量场景使用 App globalData，复杂场景引入 MobX-mini-program |
| 基础库 | ≥ 2.21.0 | 支持暗色模式、订阅消息等特性 |
| CI/CD | 微信开发者工具 + CI 插件 | 支持自动化预览与上传发布 |

### 1.3 功能范围

| 功能模块 | 核心能力 | 优先级 |
|----------|----------|--------|
| 微信授权登录 | wx.login 获取 code → 后端换取 Token；静默登录 | P0 |
| 手机号绑定 | getPhoneNumber 按钮组件授权手机号 | P0 |
| 组织信息查看 | 查看用户加入的组织列表、组织详情、成员列表 | P0 |
| 团队信息查看 | 查看组织下团队列表、团队详情 | P1 |
| 消息通知 | 订阅消息推送 + 站内消息列表查看 | P0 |
| 扫码加入组织 | wx.scanCode 扫描小程序码加入组织 | P0 |
| 邀请分享 | 生成小程序卡片分享给微信好友/群聊 | P0 |
| 个人设置 | 查看个人信息、切换组织、退出登录 | P1 |
| 消息免打扰 | 订阅消息类型管理 | P2 |

### 1.4 与移动端 App 的差异

| 对比维度 | 微信小程序 | 移动端 App（iOS/Android） |
|----------|-----------|--------------------------|
| 安装方式 | 无需安装，即开即用 | 需通过应用商店下载安装 |
| 包体限制 | 主包 ≤ 2MB，总包 ≤ 20MB | 无硬性限制（建议 ≤ 100MB） |
| 原生能力 | 受限，依赖微信开放能力 | 完整原生能力 |
| 推送通知 | 订阅消息（用户主动订阅，一次性） | APNs / FCM 持久推送 |
| 后台运行 | 不支持后台运行，退出即销毁 | 支持后台保活 |
| 页面栈深度 | 最大 10 层，建议 ≤ 5 层 | 无硬性限制 |
| 组件能力 | 小程序原生组件 + 自定义组件 | 原生 UI 组件库 |
| 支付能力 | 微信支付（预留） | Apple Pay / Google Pay / 微信支付 |
| 用户体系 | 依赖微信用户体系（openid） | 自有账号体系 |
| 启动速度 | 冷启动约 1~2s | 冷启动约 0.5~1s |
| 分发渠道 | 微信内分享、搜索、扫码 | App Store / Google Play |

## 二、rpx 单位体系

### 2.1 rpx 原理

rpx（responsive pixel）是微信小程序专属的响应式像素单位，核心原理为 **750rpx 固定等于屏幕宽度**，小程序引擎根据设备屏幕宽度自动进行等比换算，实现全设备自适应布局。

| 设备 | 屏幕宽度（物理 px） | 1rpx = ? px | 1px = ? rpx |
|------|---------------------|-------------|-------------|
| iPhone 5 | 320px | 0.42px | 2.34rpx |
| iPhone 6 | 375px | 0.5px | 2rpx |
| iPhone 6 Plus | 414px | 0.552px | 1.81rpx |
| iPhone 12/13/14 | 390px | 0.52px | 1.92rpx |
| Android 常见 | 360px | 0.48px | 2.08rpx |
| iPad | 768px | 1.024px | 0.98rpx |

> 设计基准：以 iPhone 6（375px 屏幕宽度）为标准设计稿，此时 **1px = 2rpx**。

### 2.2 px 与 rpx 换算

| 设计稿宽度 | 换算公式 | 说明 |
|-----------|----------|------|
| 375px（推荐） | `1px = 2rpx` | iPhone 6 基准，设计稿标注 px × 2 = rpx |
| 750px | `1px = 1rpx` | 1:1 换算，设计师可直接按 rpx 标注 |
| 414px | `1px = 1.81rpx` | iPhone Plus 基准 |

> 统一采用 **375px 设计稿**，设计师按 px 标注，开发时 px × 2 = rpx。

| space-4 | 16px | 32rpx | 卡片内间距 |
| space-5 | 20px | 40rpx | 区块间距 |
| space-6 | 24px | 48rpx | 卡片间距 |
| space-8 | 32px | 64rpx | 区块间距（大） |
| space-10 | 40px | 80rpx | 页面边距 |
| space-12 | 48px | 96rpx | 大区块间距 |
| space-16 | 64px | 128rpx | 页面垂直间距 |

| 令牌 | px 值 | rpx 值 | 用途 |
|------|-------|--------|------|

| radius-md | 12px | 24rpx | 中等卡片、下拉菜单、弹出层 |
| radius-lg | 19.2px | 38.4rpx | 大卡片、模态框、页面容器 |
| radius-full | 9999px | 9999rpx | 头像、圆形按钮、Pill 标签 |

#### 2.3.3 字号令牌（Font Size）

| 令牌 | px 值 | rpx 值 | 字重 | 行高 | 用途 |
|------|-------|--------|------|------|------|
| xs | 12px | 24rpx | 400 | 1.5 | 辅助说明、标签 |
| sm | 14px | 28rpx | 400 | 1.5 | 次要正文、表单提示 |

| 2xl | 24px | 48rpx | 600 | 1.2 | 页面标题 |
| 3xl | 30px | 60rpx | 700 | 1.2 | 大标题 |
| 4xl | 36px | 72rpx | 700 | 1.2 | Hero 标题 |

> 字重映射：normal=400，medium=500，semibold=600，bold=700。

#### 2.3.4 图标尺寸令牌（Icon Size）

| 令牌 | px 值 | rpx 值 | 用途 |
|------|-------|--------|------|
| icon-xs | 14px | 28rpx | 行内图标、标签前缀 |
| icon-sm | 16px | 32rpx | 按钮内图标、表单后缀 |
| icon-md | 20px | 40rpx | 默认尺寸 |

#### 2.3.5 阴影令牌（Shadow）

WXSS 直接使用 CSS box-shadow 语法，数值与全局规范一致：

| 令牌 | 值 | 用途 |
|------|----|------|
| shadow-2xs | `0 1px 2px -1px rgba(0,0,0,0.04)` | 细线（Hairline） |
| shadow-xs | `0 1px 2px 0 rgba(0,0,0,0.04)` | 微妙（Subtle） |
| shadow-sm | `0 1px 2px 0 rgba(0,0,0,0.05), 0 1px 3px -1px rgba(0,0,0,0.05)` | 卡片（Card） |
| shadow | `0 2px 4px -1px rgba(0,0,0,0.06), 0 1px 2px -1px rgba(0,0,0,0.05)` | 悬浮（Raised） |
| shadow-md | `0 4px 8px -2px rgba(0,0,0,0.06), 0 2px 4px -2px rgba(0,0,0,0.05)` | 卡片 Hover |
| shadow-lg | `0 8px 24px -8px rgba(0,0,0,0.08), 0 4px 8px -4px rgba(0,0,0,0.05)` | 浮层（Float） |
| shadow-xl | `0 16px 40px -10px rgba(0,0,0,0.10), 0 8px 16px -8px rgba(0,0,0,0.06)` | 遮罩层（Overlay） |
| shadow-2xl | `0 24px 64px -12px rgba(0,0,0,0.12)` | 模态框（Modal） |

### 2.4 rpx 使用规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 布局尺寸用 rpx | 所有宽度、高度、间距、圆角、字号统一使用 rpx | `width: 750rpx;` |
| 边框使用 1px | 边框、分割线使用 `1px` 而非 `2rpx`，保持各设备细线效果一致 | `border: 1px solid #E5E5EA;` |
| 字号使用 rpx | 所有字号使用 rpx，保证等比缩放 | `font-size: 32rpx;` |
| 阴影使用 px | box-shadow 偏移量和模糊半径使用 px | `box-shadow: 0 1px 2px rgba(0,0,0,0.04);` |
| 禁止混用 | 同一属性不得混用 px 和 rpx | 禁止：`margin: 16px 32rpx;` |
| 动画使用 px | transform 位移值使用 px，避免 rpx 计算精度问题 | `transform: translateX(100px);` |

## 三、导航模式

### 3.1 tabBar 配置

XYFamily 小程序采用底部 tabBar 导航，提供 4 个一级页面入口。

| Tab 项 | 页面路径 | 图标 | 文案 | 说明 |
|--------|----------|------|------|------|

| 消息 | `pages/message/index` | bell | 消息 | 消息通知列表 |

### 3.2 tabBar 图标要求

| 属性 | 规格 | 说明 |
|------|------|------|
| 尺寸 | 81 × 81 px | 微信官方推荐尺寸 |
| 格式 | PNG（支持透明背景） | 不支持 SVG、GIF |
| 图标数量 | 每项 2 张 | 未选中态（灰色）+ 选中态（品牌色） |
| 未选中色 | `#8E8E93` | icon-500，次级图标色 |
| 选中色 | `#007AFF` | brand-500，品牌主色 |
| 文案颜色 | 同图标色 | selectedColor / color 配置 |
| 文案字数 | ≤ 4 字 | 超出截断 |
| Tab 数量 | 2 ~ 5 个 | 微信限制最多 5 个 |
| 文件大小 | 单张 ≤ 40KB | 控制主包体积 |
| 建议位置 | `assets/tabbar/` 目录 | 统一管理 |

> 图标设计遵循 Lucide Icons 风格，线宽 1.5px，圆角端点，与全局图标系统保持一致。

### 3.3 tabBar 颜色配置

| 配置项 | 值 | 对应令牌 | 说明 |
|--------|----|----------|------|
| selectedColor | `#007AFF` | brand-500 | 选中态文字/图标颜色 |
| color | `#8E8E93` | icon-500 / text-400 | 默认态文字/图标颜色 |
| backgroundColor | `#FFFFFF` | background-50 | tabBar 背景色 |

### 3.4 页面导航

微信小程序提供 4 种页面导航 API，适用于不同场景：

| API | 功能 | 适用场景 | 是否可关闭当前页 |
|-----|------|----------|-----------------|
| `wx.navigateTo` | 保留当前页，跳转到新页 | 普通页面跳转（详情页、表单页） | 否（保留在栈中） |
| `wx.redirectTo` | 关闭当前页，跳转到新页 | 表单提交后跳转、避免栈过深 | 是 |
| `wx.switchTab` | 跳转到 tabBar 页 | 从非 Tab 页切换到 Tab 页 | 是（清空非 Tab 页） |
| `wx.navigateBack` | 返回上一页 | 返回操作、表单取消 | 是（弹出当前页） |
| `wx.reLaunch` | 关闭所有页面，打开新页 | 退出登录后重定向到登录页 | 是（清空所有栈） |

#### 导航使用规则

| 场景 | 推荐方式 | 说明 |
|------|----------|------|
| 列表 → 详情 | `navigateTo` | 保留列表页，返回时恢复滚动位置 |
| 表单提交成功 → 列表 | `redirectTo` | 避免返回到已提交的表单页 |
| Tab 页间切换 | `switchTab` | 仅用于 tabBar 页面 |
| 非层级返回 | `navigateBack` | 支持指定返回层数 `delta` |
| 退出登录 → 登录页 | `reLaunch` | 清空所有页面栈 |
| 扫码结果 → 加入组织 | `navigateTo` | 保留扫码页上下文 |

#### 页面跳转参数传递

```javascript
// 页面跳转携带参数
wx.navigateTo({
  url: '/pages/organization/detail?id=org_001&name=技术部'
});

// 目标页接收参数
Page({
  onLoad(options) {
    const { id, name } = options;
    // id = 'org_001', name = '技术部'
  }
});
```

#### 自定义导航栏适配

使用自定义导航栏时需通过 `wx.getSystemInfoSync()` 获取状态栏高度和导航栏高度进行适配：

```javascript
// 获取系统信息
const systemInfo = wx.getSystemInfoSync();
const statusBarHeight = systemInfo.statusBarHeight;   // 状态栏高度（px）
const menuButton = wx.getMenuButtonBoundingClientRect(); // 胶囊按钮位置
const navBarHeight = (menuButton.top - statusBarHeight) * 2 + menuButton.height;
```

| 设备 | 状态栏高度 | 胶囊按钮高度 | 导航栏总高度（rpx） |
|------|-----------|-------------|-------------------|
| iPhone 6 | 20px | 32px | ~128rpx |
| iPhone X 系列 | 44px | 32px | ~176rpx |
| Android 常见 | 24px | 32px | ~128rpx |

### 3.6 页面栈管理

| 约束 | 说明 |
|------|------|
| 最大栈深 | 10 层（微信硬性限制） |
| 建议栈深 | ≤ 5 层，超过后使用 `redirectTo` 替代 `navigateTo` |
| Tab 页不入栈 | `switchTab` 跳转的 tabBar 页面不计入栈深 |
| 栈满处理 | 栈深达 10 时 `navigateTo` 失败，需先 `navigateBack` 或 `redirectTo` |

#### 页面栈管理策略

| 策略 | 场景 | 操作 |
|------|------|------|
| 列表 → 详情 → 子详情 | 3 层以内 | 正常 `navigateTo` |
| 表单提交后 | 避免返回已提交表单 | `redirectTo` 替换当前页 |
| 多步流程完成 | 流程结束后返回首页 | `reLaunch` 清空栈或 `switchTab` |
| 页面栈过深 | 当前栈深 ≥ 5 | 使用 `redirectTo` 而非 `navigateTo` |

### 4.16 上拉加载

通过页面 `onReachBottom` 生命周期函数实现。

| 属性 | 规格 | 说明 |
|------|------|------|
| onReachBottomDistance | 50（默认，单位 px） | 距底部多远触发 |
| 加载状态 | 底部旋转图标 + "加载中..." | 加载态展示 |
| 无更多数据 | "— 没有更多了 —" | 分页结束提示 |
| 加载失败 | "加载失败，点击重试" | 可点击重试 |
| 防重复 | isLoading 标志位 | 防止重复触发 |

```javascript
Page({
  data: {
    list: [],
    page: 1,
    hasMore: true,
    isLoading: false
  },

  onReachBottom() {
    if (this.data.isLoading || !this.data.hasMore) return;
    this.loadMore();
  },

  async loadMore() {
    this.setData({ isLoading: true });
    try {
      const res = await fetchList({ page: this.data.page + 1 });
      this.setData({
        list: [...this.data.list, ...res.items],
        page: this.data.page + 1,
        hasMore: res.hasMore,
        isLoading: false
      });
    } catch {
      this.setData({ isLoading: false });
      wx.showToast({ title: '加载失败', icon: 'none' });
    }
  }
});
```

## 六、微信生态集成

### 6.1 微信登录（wx.login）

XYFamily 小程序采用微信静默登录 + 手机号绑定的双步登录策略。

#### 登录流程

```
小程序启动 / Token 过期
  → wx.login() 获取临时 code
  → wx.request 发送 code 到后端 /auth/wx-login
  → 后端调用微信 jscode2session 获取 openid + session_key
  → 后端根据 openid 查找/创建用户
    → 用户已存在且已绑定手机号
      → 返回 access_token + refresh_token
      → 前端存储 Token → 登录完成
    → 用户不存在或未绑定手机号
      → 返回 needs_phone_bind 状态 + 临时 token
      → 前端展示手机号绑定按钮
```

#### 登录代码示例

```javascript
// 微信登录
async function wxLogin() {
  try {
    // 1. 获取微信 code
    const { code } = await wx.login();

    // 2. 发送 code 到后端
    const res = await request({
      url: '/auth/wx-login',
      method: 'POST',
      data: { code },
      skipAuth: true
    });

    // 3. 处理登录结果
    if (res.access_token) {
      // 登录成功
      setToken(res.access_token);
      wx.setStorageSync('xyfamily_refresh_token', res.refresh_token);
      wx.switchTab({ url: '/pages/dashboard/index' });
    } else if (res.needs_phone_bind) {
      // 需要绑定手机号
      wx.setStorageSync('xyfamily_temp_token', res.temp_token);
      this.setData({ showPhoneBind: true });
    }
  } catch (err) {
    wx.showToast({ title: '登录失败，请重试', icon: 'none' });
  }
}
```

| 配置项 | 值 | 说明 |
|--------|----|------|
| AppID | 项目专属 AppID | 微信公众平台注册获取 |
| AppSecret | 后端保管 | 仅在后端使用，前端不可暴露 |
| session_key | 后端保管 | 用于解密手机号等加密数据 |
| openid | 用户唯一标识 | 微信用户在当前小程序的唯一 ID |
| unionid | 可选 | 跨小程序统一标识（需绑定开放平台） |
| Token 有效期 | access_token 2h / refresh_token 7d | 过期后自动刷新 |

### 6.2 手机号授权（getPhoneNumber）

使用 `<button open-type="getPhoneNumber">` 组件获取用户手机号。

#### 实现方式

```xml
<button
  open-type="getPhoneNumber"
  bindgetphonenumber="handleGetPhoneNumber"
  class="btn btn-primary"
>
  微信手机号一键绑定
</button>
```

```javascript
async function handleGetPhoneNumber(e) {
  if (e.detail.errMsg !== 'getPhoneNumber:ok') {
    // 用户拒绝授权
    wx.showToast({ title: '需要手机号才能使用', icon: 'none' });
    return;
  }

  try {
    // 发送加密数据到后端解密
    const res = await request({
      url: '/auth/bind-phone',
      method: 'POST',
      data: {
        code: e.detail.code,          // 微信新增的动态令牌（推荐）
        encrypted_data: e.detail.encryptedData, // 加密数据（兼容旧版）
        iv: e.detail.iv,              // 加密向量
        temp_token: wx.getStorageSync('xyfamily_temp_token')
      },
      skipAuth: true
    });

    // 绑定成功，存储正式 Token
    setToken(res.access_token);
    wx.setStorageSync('xyfamily_refresh_token', res.refresh_token);
    wx.removeStorageSync('xyfamily_temp_token');
    wx.switchTab({ url: '/pages/dashboard/index' });
  } catch (err) {
    wx.showToast({ title: '绑定失败，请重试', icon: 'none' });
  }
}
```

| 属性 | 说明 |
|------|------|
| open-type | `getPhoneNumber`，触发手机号授权弹窗 |
| e.detail.code | 微信动态令牌，后端调用 getPhoneNumber 接口获取手机号（推荐方案） |
| e.detail.encryptedData | 加密手机号数据（兼容旧基础库） |
| e.detail.iv | 加密算法的初始向量（配合 encryptedData） |
| errMsg | `getPhoneNumber:ok` 表示用户同意，`getPhoneNumber:deny` 表示拒绝 |

> 推荐使用 `code` 方式获取手机号（基础库 ≥ 2.21.2），后端直接调用微信接口换取手机号，无需 session_key 解密，安全性更高。

### 6.3 消息通知（订阅消息）

微信小程序使用**订阅消息**替代旧版模板消息，每次发送需用户主动订阅授权。

#### 订阅消息流程

```
用户触发操作（如提交加入申请）
  → wx.requestSubscribeMessage 申请订阅权限
    → 用户选择"总是保持以上选择"或"允许"
    → 后端记录用户订阅状态
  → 后端在事件发生时调用 subscribeMessage.send API 推送消息
    → 用户在微信消息列表收到通知
    → 点击通知打开小程序对应页面
```

#### 订阅消息模板

| 模板名称 | 触发场景 | 模板字段 | 说明 |
|----------|----------|----------|------|
| 加入组织结果通知 | 加入申请审批完成 | 组织名称、审批结果、审批时间、备注 | 通知用户审批结果 |
| 角色变更通知 | 管理员修改成员角色 | 组织名称、原角色、新角色、变更时间 | 通知成员角色变更 |
| 新成员加入通知 | 新成员加入组织 | 组织名称、新成员姓名、加入时间 | 通知管理员有新成员 |
| 组织动态通知 | 组织信息变更 | 组织名称、变更内容、变更时间 | 通知组织成员 |

#### 前端订阅代码

```javascript
async function requestSubscribe(templateIds) {
  try {
    const res = await wx.requestSubscribeMessage({
      tmplIds: templateIds // 需订阅的模板 ID 列表
    });

    // 记录用户订阅结果
    const subscriptions = {};
    templateIds.forEach(id => {
      subscriptions[id] = res[id]; // 'accept' / 'reject' / 'ban'
    });

    // 上报订阅状态到后端
    await request({
      url: '/notification/subscribe',
      method: 'POST',
      data: { subscriptions }
    });
  } catch (err) {
    console.error('订阅失败', err);
  }
}
```

### 6.6 小程序码（wxacode）

生成带参数的小程序码，用于组织邀请场景。

#### 生成方式

小程序码由**后端调用微信接口**生成，前端仅负责展示和保存。

| 接口 | 说明 | 限制 |
|------|------|------|
| `wxacode.get` | 获取小程序码 | 每分钟调用 ≤ 10000 次 |
| `wxacode.getUnlimited` | 获取无数量限制小程序码 | 推荐使用，scene 参数最多 32 字符 |

#### 后端生成示例

```javascript
// 后端调用微信接口生成小程序码（伪代码）
async function generateWxaCode(inviteCode) {
  const accessToken = await getWxAccessToken();
  const res = await axios.post(
    'https://api.weixin.qq.com/wxa/getwxacodeunlimit',
    {
      scene: inviteCode,              // 场景参数（最多 32 字符）
      page: 'pages/scan/join',        // 扫码后打开的页面
      env_version: 'release',        // 正式版
      width: 430,                     // 小程序码宽度（px）
      auto_color: false,              // 是否自动配置颜色
      line_color: { r: 0, g: 122, b: 255 }, // 品牌色 #007AFF
      is_hyaline: false               // 是否透明背景
    },
    {
      params: { access_token: accessToken },
      responseType: 'arraybuffer'     // 返回二进制图片
    }
  );
  return Buffer.from(res.data).toString('base64');
}
```

```javascript
class RequestQueue {
  constructor(maxConcurrent = 8) {
    this.maxConcurrent = maxConcurrent;
    this.running = 0;
    this.queue = [];
  }

  add(requestFn) {
    return new Promise((resolve, reject) => {
      this.queue.push({ requestFn, resolve, reject });
      this.next();
    });
  }

  next() {
    while (this.running < this.maxConcurrent && this.queue.length > 0) {
      const { requestFn, resolve, reject } = this.queue.shift();
      this.running++;
      requestFn()
        .then(resolve)
        .catch(reject)
        .finally(() => {
          this.running--;
          this.next();
        });
    }
  }
}
```

### 7.4 setData 性能

setData 是小程序性能优化的核心关注点，频繁或大量 setData 会导致渲染卡顿。

| 约束 | 限制值 | 说明 |
|------|--------|------|
| 单次 setData 数据量 | ≤ 1MB | 数据序列化后大小 |
| setData 频率 | 建议 ≤ 5 次/秒 | 避免高频渲染 |
| 单次 setData 字段数 | 建议 ≤ 100 个 | 减少 diff 计算开销 |
| 图片数据 | 禁止 setData 传递 base64 图片 | 使用临时文件路径替代 |

#### setData 优化规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 局部更新 | 只更新变化的数据字段，不整体覆盖 | `this.setData({ 'list[0].name': '新名称' })` |
| 合并 setData | 多次 setData 合并为一次 | `this.setData({ a: 1, b: 2 })` 而非两次单独调用 |
| 路径更新 | 使用数据路径精确更新列表项 | `this.setData({ 'list[2].checked': true })` |
| 避免大对象 | 不传递与渲染无关的数据 | 后端返回的大数据仅提取需要的字段 |
| 延迟更新 | 非关键数据使用 `wx.nextTick` 或 `setTimeout` 延迟 | 先渲染首屏，再更新次要数据 |

```javascript
// 优化前（不推荐）
this.setData({ list: newList }); // 整体替换整个列表

// 优化后（推荐）
// 仅更新变化的项
this.setData({
  [`list[${index}].name`]: newName,
  [`list[${index}].avatar`]: newAvatar
});
```

### 7.5 不支持 DOM 操作

小程序运行环境非浏览器，无 DOM / BOM API。

| 不支持能力 | 替代方案 | 说明 |
|-----------|----------|------|
| document.querySelector | WXML 数据绑定 + setData | 通过数据驱动视图 |
| window / document 对象 | wx.getSystemInfoSync() | 通过 wx API 获取系统信息 |
| localStorage | wx.setStorageSync() / wx.getStorageSync() | 同步存储 API |
| XMLHttpRequest / fetch | wx.request() | 小程序网络请求 API |
| canvas 2D Context | wx.createCanvasContext() / Canvas 2D | 小程序 Canvas API |
| addEventListener | bind / catch 事件绑定 | WXML 事件系统 |
| setInterval / setTimeout | 支持，但需注意页面销毁时清理 | — |

### 7.6 样式限制

WXSS 是 CSS 的子集，存在以下限制：

| 限制项 | 说明 | 替代方案 |
|--------|------|----------|
| 不支持级联选择器 | `A > B`、`A + B` 等不支持 | 使用 class 选择器 |
| 不支持 `*` 通配符 | `* { margin: 0 }` 无效 | 明确指定元素或使用 `page` |
| 伪元素有限支持 | 仅支持 `::before` / `::after` | `content` 属性必须设置 |
| 不支持 `@import` 外部链接 | 仅支持本地 WXSS 文件引入 | `@import "common.wxss";` |
| 不支持属性选择器 | `[type="text"]` 不支持 | 使用 class |
| 不支持 `!important`（部分版本） | 基础库 ≥ 2.8.0 支持 | 尽量避免使用 |
| 全局样式 | 仅 `app.wxss` 为全局 | 页面样式仅作用于当前页面 |
| 本地图片引用 | WXSS 中 `url()` 不支持本地路径 | 使用 base64 或网络图片 |
| 不支持 `calc()` 混合单位 | `calc(100% - 32rpx)` 支持有限 | 使用固定值或 flex 布局 |

## 八、暗色模式适配

### 8.1 暗色模式配置

小程序暗色模式跟随微信全局深色模式设置，通过 `app.json` 配置启用。

```json
// app.json
{
  "darkmode": true,
  "themeLocation": "theme.json",
  "window": {
    "navigationBarBackgroundColor": "@navBgColor",
    "navigationBarTextStyle": "@navTxtStyle",
    "backgroundColor": "@bgColor",
    "backgroundTextStyle": "@bgTxtStyle"
  },
  "tabBar": {
    "color": "@tabFontColor",
    "selectedColor": "@tabSelectedColor",
    "backgroundColor": "@tabBgColor",
    "borderStyle": "@tabBorderStyle"
  }
}
```

### 8.2 theme.json 色彩变量定义

`theme.json` 定义 Light 和 Dark 两套色彩变量，通过 `@变量名` 语法在 app.json 中引用。

```json
// theme.json
{
  "light": {
    "navBgColor": "#FFFFFF",
    "navTxtStyle": "black",
    "bgColor": "#F7F7FA",
    "bgTxtStyle": "dark",
    "tabFontColor": "#8E8E93",
    "tabSelectedColor": "#007AFF",
    "tabBgColor": "#FFFFFF",
    "tabBorderStyle": "white"
  },
  "dark": {
    "navBgColor": "#1C1C1E",
    "navTxtStyle": "white",
    "bgColor": "#000000",
    "bgTxtStyle": "light",
    "tabFontColor": "#8E8E93",
    "tabSelectedColor": "#2E8DFF",
    "tabBgColor": "#1C1C1E",
    "tabBorderStyle": "black"
  }
}
```

### 8.3 theme.json 变量映射表

| 变量名 | Light 值 | Dark 值 | 对应令牌 | 用途 |
|--------|----------|---------|----------|------|
| navBgColor | `#FFFFFF` | `#1C1C1E` | background-50 / background-800 | 导航栏背景 |
| navTxtStyle | `black` | `white` | — | 导航栏文字样式 |
| bgColor | `#F7F7FA` | `#000000` | background-100 / background-900 | 页面背景 |
| bgTxtStyle | `dark` | `light` | — | 下拉刷新文字样式 |
| tabFontColor | `#8E8E93` | `#8E8E93` | text-400 | tabBar 默认文字色 |
| tabSelectedColor | `#007AFF` | `#2E8DFF` | brand-500 / brand-400 | tabBar 选中色 |
| tabBgColor | `#FFFFFF` | `#1C1C1E` | background-50 / background-800 | tabBar 背景色 |
| tabBorderStyle | `white` | `black` | — | tabBar 边线样式 |

### 8.4 WXSS 暗色模式适配

WXSS 中使用媒体查询 `prefers-color-scheme` 适配暗色模式：

```css
/* app.wxss - 全局样式 */

/* Light 模式（默认） */
page {
  --primary: #007AFF;
  --background: #FFFFFF;
  --foreground: #1D1D1F;
  --card: #FFFFFF;
  --card-foreground: #1D1D1F;
  --secondary: #F2F2F7;
  --secondary-foreground: #1D1D1F;
  --muted: #F2F2F7;
  --muted-foreground: #8E8E93;
  --border: #E5E5EA;
  --input: #D1D1D6;
  --destructive: #FF3B30;
  --success: #34C759;
  --warning: #FF9500;
}

/* Dark 模式 */
@media (prefers-color-scheme: dark) {
  page {
    --primary: #2E8DFF;        /* brand-400 */
    --background: #000000;      /* background-900 */
    --foreground: #F5F5F7;      /* text-50 */
    --card: #1C1C1E;            /* background-800 */
    --card-foreground: #F5F5F7; /* text-50 */
    --secondary: #1C1C1E;       /* background-800 */
    --secondary-foreground: #F5F5F7;
    --muted: #1C1C1E;
    --muted-foreground: #8E8E93;
    --border: #3A3A3C;          /* background-700 */
    --input: #3A3A3C;
    --destructive: #FF453A;     /* error-dark */
    --success: #30D158;         /* success-dark */
    --warning: #FF9F0A;         /* warning-dark */
  }
}
```

### 8.5 各组件暗色模式适配要点

| 组件 | Light 模式 | Dark 模式 | 适配要点 |
|------|-----------|-----------|----------|
| 页面背景 | `#F7F7FA` | `#000000` | 使用 `var(--background)` |
| 卡片 | `#FFFFFF` + shadow-sm | `#1C1C1E` + shadow-sm（增透明度） | 背景色用 `var(--card)`，阴影在暗色模式增大透明度 |
| 分割线 | `#E5E5EA` | `#3A3A3C` | 使用 `var(--border)` |
| 主文字 | `#1D1D1F` | `#F5F5F7` | 使用 `var(--foreground)` |
| 次要文字 | `#8E8E93` | `#8E8E93` | 不变，使用 `var(--muted-foreground)` |
| 按钮主色 | `#007AFF` | `#2E8DFF` | 使用 `var(--primary)` |
| 输入框背景 | `#F2F2F7` | `#1C1C1E` | 使用 `var(--secondary)` |
| tabBar | 白色背景 | `#1C1C1E` 背景 | 通过 theme.json 配置 |
| 图片资源 | 正常色彩 | 可选反色处理 | 图标使用 CSS filter 或提供暗色版 |

#### 图片暗色适配

```css
/* 暗色模式下图标反色（适用于线框图标） */
@media (prefers-color-scheme: dark) {
  .icon-dark-invert {
    filter: invert(1) hue-rotate(180deg);
  }
}
```

### 8.6 暗色模式开发注意事项

| 注意事项 | 说明 |
|----------|------|
| 基础库要求 | ≥ 2.11.0 支持 `darkmode` 配置 |
| 调试方法 | 微信开发者工具 → 模拟器 → 切换深色模式 |
| 图片适配 | 提供暗色模式专用图片或使用 CSS filter |
| canvas 适配 | Canvas 需手动检测暗色模式并重绘 |
| tabBar 图标 | 不支持 theme.json 配置图标切换，需保持图标在两种模式下可见 |

## 九、无障碍设计

### 9.1 aria-role 属性支持

小程序基础库 ≥ 2.10.4 起支持 `aria-role` 和 `aria-label` 属性，用于辅助功能访问。

| 属性 | 说明 | 使用场景 |
|------|------|----------|
| `aria-role` | 定义元素语义角色 | `button`、`link`、`image`、`heading` |
| `aria-label` | 元素的文本描述 | 图标按钮、无文字的可交互元素 |
| `aria-hidden` | 对辅助功能隐藏元素 | 装饰性元素 |

```xml
<!-- 图标按钮添加无障碍标注 -->
<view
  class="icon-btn"
  aria-role="button"
  aria-label="扫码加入组织"
  bindtap="onScanJoin"
>
  <image src="/assets/icons/scan.svg" mode="aspectFit" />
</view>

<!-- 装饰性图片隐藏 -->
<image
  src="/assets/icons/decoration.svg"
  mode="aspectFit"
  aria-hidden="true"
/>
```

### 9.2 文字大小自适应

小程序支持用户在系统设置中调整字体大小，页面应适配大字体场景。

| 适配策略 | 说明 |
|----------|------|
| 文字不截断 | 标题和正文允许换行，不使用 `white-space: nowrap` |
| 固定高度改最小高度 | Cell 高度使用 `min-height` 而非 `height`，文字增大时自动撑开 |
| 弹性布局 | 使用 flex 布局，文字区域 `flex: 1` 自适应宽度 |
| 省略号策略 | 仅在必要场景使用 `text-overflow: ellipsis`，保留 `display: -webkit-box` 多行截断 |
| 按钮文字适配 | 按钮宽度使用 `min-width`，文字增长时按钮自动加宽 |

```css
/* 文字自适应布局 */
.cell-title {
  font-size: 32rpx;
  line-height: 1.5;
  /* 允许换行 */
  word-break: break-all;
}

.cell {
  /* 使用 min-height 代替固定高度 */
  min-height: 96rpx;
  /* flex 布局自适应 */
  display: flex;
  align-items: center;
}

.btn {
  /* 按钮最小宽度 */
  min-width: 200rpx;
  /* 文字不截断 */
  white-space: normal;
}
```

### 9.3 触控目标

| 元素类型 | 最小触控目标 | 说明 |
|----------|-------------|------|
| 可点击元素 | 88rpx × 88rpx（44 × 44pt） | 符合 Apple HIG 标准 |
| tabBar 图标 | 微信原生固定 | 无需处理 |
| 图标按钮 | 88rpx × 88rpx | 图标 48rpx，点击区域 88rpx |
| Cell 列表项 | 最小高度 96rpx | 保证可点击区域 |
| 表单元素 | 高度 ≥ 96rpx | 输入框、选择器 |

### 9.4 焦点管理

小程序无传统键盘焦点概念，但需关注以下无障碍交互：

| 规则 | 说明 |
|------|------|
| 点击区域明确 | 可点击区域有清晰的视觉边界和按压反馈 |
| 按压反馈 | 所有可点击元素使用 `:active` 状态样式 |
| 防止误触 | 危险操作需二次确认（showModal） |
| 操作反馈 | 点击后立即给出视觉反馈（toast、loading、颜色变化） |

## 十、性能优化

### 10.1 图片懒加载

使用 `<image>` 组件的 `lazy-load` 属性实现图片懒加载。

```xml
<image
  src="{{item.avatar}}"
  mode="aspectFill"
  lazy-load
/>
```

| 属性 | 说明 |
|------|------|
| lazy-load | 图片进入可视区域前 3 屏时不加载，进入后加载 |
| 适用场景 | 列表中的图片、头像、长页面图片 |
| 不适用 | 首屏可见图片、tabBar 图标 |
| mode | `aspectFill`（裁剪填充）/ `aspectFit`（完整展示）/ `widthFix`（宽度固定高度自适应） |

### 10.2 分包预下载

通过 `preloadRule` 配置，在用户访问特定页面时预下载分包，减少分包跳转的等待时间。

| 策略 | 说明 |
|------|------|
| 首页加载后预下载 | 预下载组织管理和消息分包 |
| WiFi 环境预下载 | 非核心分包仅在 WiFi 下预下载 |
| 分包跳转预加载 | 进入组织列表时预下载消息分包 |

```json
{
  "preloadRule": {
    "pages/dashboard/index": {
      "network": "all",
      "packages": ["subpackages/org", "subpackages/message"]
    }
  }
}
```

### 10.3 列表渲染优化

#### key 设置

```xml
<!-- 正确：使用唯一 key -->
<view wx:for="{{list}}" wx:key="id">
  {{item.name}}
</view>

<!-- 错误：使用 index 作为 key -->
<view wx:for="{{list}}" wx:key="index">
  {{item.name}}
</view>
```

| 规则 | 说明 |
|------|------|
| 必须设置 wx:key | 使用数据中的唯一字段作为 key |
| 禁止使用 index | 列表增删时 index 会变，导致渲染异常 |
| key 值稳定 | 同一条数据 key 不应变化 |

#### 虚拟列表

长列表（> 100 条）使用虚拟列表技术，仅渲染可视区域内的元素：

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| `recycle-view` | 微信官方虚拟列表组件 | 超长列表（> 500 条） |
| 分页加载 | 每页 20 条，滚动到底部加载更多 | 常规列表（< 500 条） |
| 按需渲染 | 仅渲染可视区域 + 上下缓冲区域 | 极长列表优化 |

### 10.4 减少 setData 频率和数据量

| 优化策略 | 说明 | 示例 |
|----------|------|------|
| 合并 setData | 多次数据更新合并为一次 | `this.setData({ a: 1, b: 2 })` |
| 局部更新 | 使用路径更新单个字段 | `this.setData({ 'list[0].name': '新名称' })` |
| 纯数据字段 | 不参与渲染的数据标记为纯数据 | `pureDataPattern: /^_/` |
| 减少数据量 | 后端返回数据仅提取渲染所需字段 | `list.map(item => ({ id, name, avatar }))` |
| 防抖处理 | 频繁输入场景使用防抖 | 搜索输入防抖 300ms |

#### 纯数据字段配置

```javascript
Component({
  options: {
    pureDataPattern: /^_/ // 以 _ 开头的字段不参与渲染
  },
  data: {
    _scrollTop: 0,    // 纯数据字段，不触发渲染
    _timer: null,     // 纯数据字段
    list: []          // 普通字段，参与渲染
  }
});
```

### 10.5 骨架屏替代 Loading 动画

在数据加载期间展示骨架屏，而非空白页面 + Loading 转圈，提升用户感知性能。

| 场景 | 方案 | 说明 |
|------|------|------|
| 首屏加载 | 骨架屏 | 页面结构已渲染，数据未到位时展示骨架 |
| 列表加载 | 骨架屏 | 列表项骨架占位 |
| 下拉刷新 | 原生刷新动画 | 使用微信原生下拉刷新指示器 |
| 上拉加载 | 底部 Loading 图标 | 列表底部小旋转图标 |
| 按钮提交 | 按钮内 Loading | 按钮变为 Loading 态，禁止重复点击 |
| 全屏请求 | wx.showLoading | 阻塞式全屏 Loading |

#### 骨架屏使用规则

| 规则 | 说明 |
|------|------|
| 结构匹配 | 骨架屏布局与真实内容一致，避免布局抖动 |
| 数据切换无闪烁 | 数据到达后平滑替换骨架屏 |
| 动画效果 | shimmer 扫光动画，1.5s 循环 |
| 骨架色 | Light 模式 `#E5E5EA`，Dark 模式 `#3A3A3C` |
| 显示时机 | `loading === true` 时展示骨架屏 |

### 10.6 其他性能优化

| 优化项 | 说明 |
|--------|------|
| 图片压缩 | PNG 图片使用 TinyPNG 压缩，JPG 质量 ≤ 80% |
| 图片格式 | 简单图标用 SVG，照片用 JPG/WebP，透明图用 PNG |
| 代码压缩 | 上传时勾选"压缩代码" |
| 按需引入 | 组件按需引入，不全局注册未使用的组件 |
| 事件节流 | 高频事件（scroll、resize）使用节流处理 |
| 页面卸载清理 | onUnload 中清除定时器、事件监听 |
| 减少 WXS 计算 | 复杂计算逻辑放到 JS 层，WXS 仅用于简单格式化 |
| 合理使用 catch | `catchtap` 阻止冒泡，避免不必要的事件冒泡 |

```javascript
// 页面卸载时清理资源
Page({
  data: {
    _timer: null
  },

  onUnload() {
    if (this.data._timer) {
      clearTimeout(this.data._timer);
    }
    // 取消未完成的请求
    if (this.requestTask) {
      this.requestTask.abort();
    }
  }
});
```
