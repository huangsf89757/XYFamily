## 项目背景

XYFamily 是一个通用多功能工具集平台，当前阶段需要构建**多租户账号权限底座**，为后续业务工具的接入提供统一的身份认证、权限管理和数据隔离能力。

## 核心业务价值

- **统一身份管理**：支持多种登录方式，跨组织身份复用
- **多租户隔离**：组织间完全数据隔离，满足企业级安全合规
- **精细化权限控制**：六层角色体系（含 Public 第 0 层）+ 45 个权限点矩阵
- **可扩展性架构**：分层设计，便于后续业务工具快速接入

## 战略意义

- **奠定平台基础**：账号权限底座决定平台安全性和可扩展性
- **加速业务落地**：统一认证鉴权框架减少重复开发
- **提升用户体验**：统一登录、跨组织协作能力

## 技术栈

### 后端

| 类别 | 技术选型 |
|------|----------|
| 语言 | Go 1.22+ |
| Web 框架 | Gin |
| 数据库 | PostgreSQL 17 |
| 缓存/消息 | Redis 7.x |
| 认证 | JWT (HS256) + RBAC |
| 密码存储 | bcrypt (cost factor 12) |
| API 规范 | RESTful, /api/v1/, JSON |
| 部署策略 | 无状态部署，支持水平扩展 |

### 前端（5 个平台）

| 平台 | 技术选型 |
|------|----------|
| 安卓 (Android) | Kotlin + Jetpack Compose + Retrofit + Room + Hilt |
| iOS | Swift 5.9+ + SwiftUI + URLSession + Core Data + Keychain |
| H5（移动浏览器） | React 18+ + TypeScript + Vite + Ant Design Mobile / Vant |
| 微信小程序 | 原生框架 / uni-app + TypeScript + wx.request |
| Web（PC 管理后台） | React 18+ + TypeScript + Vite + Ant Design 5.x + ECharts |

**跨平台统一**：各前端平台统一 API 调用层封装（Token 自动刷新 + 多租户上下文传递），统一错误码处理，统一错误提示文案。

详见：[前端通用代码规范](../04-开发规范与编码手册/04.02-分端代码规范/前端通用代码规范-V1.0.0.md)

## 组织架构层级

```
组织 → 团队 → 小组 → 成员
```

## 角色体系（六层，含 Public 第 0 层）

| 角色 | 层级 | 职责范围 |
|------|------|----------|
| Public | L0 | 公共角色，每个用户自动拥有，仅拥有注册/登录/重置密码三个公开权限点 |
| 超级管理员 (SuperAdmin) | L8 | 全局管理所有组织和系统配置 |
| 组织核心管理员 (organization_core_admin) | L7 | 管理整个组织及下属团队/小组，含结构性操作与审计查看 |
| 组织普通管理员 (organization_ordinary_admin) | L6 | 负责组织成员管理（邀请/移除/列表）与资源读取，不可做结构性操作 |
| 团队核心管理员 (team_core_admin) | L5 | 管理团队内小组和成员，含结构性操作 |
| 团队普通管理员 (team_ordinary_admin) | L4 | 负责团队成员管理（邀请/移除/列表）与资源读取，不可做结构性操作 |
| 小组核心管理员 (group_core_admin) | L3 | 管理小组内成员和权限，含结构性操作 |
| 小组普通管理员 (group_ordinary_admin) | L2 | 负责小组成员管理（邀请/移除/列表）与资源读取，不可做结构性操作 |
| 普通成员 (RegularMember) | L1 | 使用平台业务工具 |
