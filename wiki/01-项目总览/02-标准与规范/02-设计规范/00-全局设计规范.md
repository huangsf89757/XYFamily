## 一、设计哲学

XYFamily 采用 **Pinguo** 设计语言，核心理念为「克制、优雅、苹果风」。设计风格受 Apple Human Interface Guidelines 启发，追求内容优先、视觉留白和交互直觉性。

| 原则 | 说明 |
|------|------|
| **内容优先** | 界面为内容服务，装饰性元素最小化，让用户聚焦于核心操作 |
| **克制使用色彩** | 全局仅使用单一品牌主色（蓝色 #007AFF），状态色仅用于真实语义状态（成功/错误/警告/信息），禁止装饰性多色 |
| **层级清晰** | 通过留白、字号、色彩明度建立视觉层级，避免过度使用边框和分割线 |
| **直觉交互** | 交互模式遵循平台原生习惯，降低学习成本 |
| **一致性** | 相同语义的元素在全端保持一致的视觉表现，仅在尺度上做平台适配 |

### 2.5 色彩使用规则

| 规则 | 说明 |
|------|------|
| 禁止硬编码色值 | 组件代码中不得出现 `#007AFF` 等原始色值，必须使用语义令牌 |
| 禁止装饰性多色 | 仅使用单一品牌主色 + 中性灰阶 + 状态色，禁止引入额外色相 |
| 状态色语义化 | 绿色仅表示成功，红色仅表示错误/危险，橙色仅表示警告，蓝色仅表示信息/品牌 |
| 对比度达标 | 文字与背景对比度 ≥ 4.5:1（正文）/ 3:1（大号文字），详见无障碍章节 |

## 四、间距系统

采用基准间距乘数系统，基准值为 `0.24rem`（约 3.84px），通过乘数生成完整间距阶：

| 令牌 | 计算 | 值 (px) | 用途 |
|------|------|---------|------|
| space-0 | 0 × base | 0 | 无间距 |
| space-1 | 1 × base | 4 | 图标与文字间距 |
| space-2 | 2 × base | 8 | 紧凑元素内间距 |
| space-3 | 3 × base | 12 | 元素间默认间距 |
| space-4 | 4 × base | 16 | 卡片内间距 |
| space-5 | 5 × base | 20 | 区块间距 |
| space-6 | 6 × base | 24 | 卡片间距 |
| space-8 | 8 × base | 32 | 区块间距（大） |
| space-10 | 10 × base | 40 | 页面边距 |
| space-12 | 12 × base | 48 | 大区块间距 |
| space-16 | 16 × base | 64 | 页面垂直间距 |

### 间距使用规则

| 场景 | 推荐间距 |
|------|----------|
| 图标 + 文字（行内） | space-1 (4px) |
| 表单元素内间距 | space-2 (8px) ~ space-3 (12px) |
| 卡片内边距 | space-4 (16px) ~ space-6 (24px) |
| 卡片之间间距 | space-4 (16px) ~ space-6 (24px) |
| 页面区块间距 | space-8 (32px) ~ space-12 (48px) |
| 页面左右边距（Web） | space-8 (32px) ~ space-10 (40px) |

## 八、主题系统

### 8.1 主题模式

| 模式 | 标识 | 说明 |
|------|------|------|
| Light | `:root` / `class="light"` | 默认浅色模式 |
| Dark | `class="dark"` | 深色模式 |

### 8.2 主题切换策略

| 策略 | 说明 |
|------|------|
| 默认跟随系统 | 首次访问读取 `prefers-color-scheme` |
| 用户可手动切换 | 切换后持久化到 localStorage |
| 全局生效 | 切换主题后所有页面同步更新 |
| 无闪烁加载 | 通过 `<script>` 在 `<head>` 中提前设置 class |

### 8.3 主题切换 CSS 示例

```css
:root {
  --background: var(--background-50);
  --foreground: var(--text-800);
  /* ... light tokens ... */
}

.dark {
  --background: var(--background-900);
  --foreground: var(--text-50);
  /* ... dark tokens ... */
}
```

### 9.1 动效原则

| 原则 | 说明 |
|------|------|
| 自然 | 使用缓动函数模拟物理运动，避免线性动画 |
| 克制 | 动效服务于交互反馈，不做纯装饰性动画 |
| 高性能 | 仅动画 `transform` 和 `opacity`，避免触发重排 |

### 9.2 缓动函数（Easing）

| 令牌 | 值 | 用途 |
|------|----|------|
| ease-default | `cubic-bezier(0.4, 0, 0.2, 1)` | 通用缓动 |
| ease-in | `cubic-bezier(0.4, 0, 1, 1)` | 元素退出 |
| ease-out | `cubic-bezier(0, 0, 0.2, 1)` | 元素进入 |
| ease-in-out | `cubic-bezier(0.4, 0, 0.2, 1)` | 状态切换 |
| spring | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 弹性效果（微交互） |

### 9.3 动效时长（Duration）

| 令牌 | 值 | 用途 |
|------|----|------|
| instant | 0ms | 即时反馈（无动画） |
| fast | 150ms | 微交互（按钮按压、Toggle） |
| normal | 250ms | 常规过渡（颜色、透明度） |
| slow | 400ms | 大范围过渡（面板展开、模态框） |
| slower | 600ms | 页面过渡（路由切换） |

### 9.4 常见动效模式

| 场景 | 时长 | 缓动 | 效果 |
|------|------|------|------|
| 按钮 Hover 背景色 | fast | ease-default | 背景色渐变 |
| 模态框进入 | slow | ease-out | 从中心缩放 + 淡入 |
| 模态框退出 | normal | ease-in | 向中心缩放 + 淡出 |
| 下拉菜单展开 | normal | ease-out | 从顶部展开 + 淡入 |
| Toast 提示 | normal | ease-out | 从顶部/底部滑入 |
| 侧边栏展开/收起 | slow | ease-in-out | 宽度过渡 |
| 列表项删除 | normal | ease-in | 高度收起 + 淡出 |

## 十、无障碍设计

### 10.1 对比度要求

遵循 WCAG 2.1 AA 标准：

| 元素类型 | 最小对比度 |
|----------|-----------|
| 正文文字（< 18px） | ≥ 4.5:1 |
| 大号文字（≥ 18px 或 ≥ 14px bold） | ≥ 3.0:1 |
| 图标和图形元素 | ≥ 3.0:1 |
| 禁用态元素 | 不强制，但应可辨识 |

### 10.2 焦点管理

| 规则 | 说明 |
|------|------|
| 可见焦点环 | 所有可交互元素必须显示焦点环，使用 `--ring` 令牌 |
| 焦点顺序 | Tab 键导航顺序与视觉顺序一致 |
| 键盘可操作 | 所有功能可通过键盘完成，不依赖鼠标 |
| 跳过导航 | 提供跳过导航栏的快捷链接（Web 端） |
| 模态框焦点陷阱 | 模态框打开时焦点锁定在内部，关闭后返回触发元素 |

### 10.3 屏幕阅读器

| 规则 | 说明 |
|------|------|
| 语义化标签 | 使用 `<nav>`、`<main>`、`<section>`、`<button>` 等语义标签 |
| ARIA 标注 | 动态内容和自定义组件添加 `aria-label`、`aria-role` |
| 替代文本 | 所有图片提供 `alt` 属性，装饰性图片用空 `alt` |
| 状态通告 | 使用 `aria-live` 区域通告动态更新（如 Toast、表单错误） |

### 10.4 触控目标

| 端 | 最小触控目标 |
|----|-------------|
| Web（移动端适配） | 44 × 44 px |
| iOS | 44 × 44 pt |
| Android | 48 × 48 dp |
| 微信小程序 | 48 × 48 rpx |

## 十二、设计令牌映射表

以下为 CSS 自定义属性（CSS Custom Properties）完整定义，Web 端直接使用，各端按平台机制映射：

### 12.1 令牌命名规范

```
--<category>-<role>[-<state>]

示例：
--brand-500              → 品牌色 500 级
--background-200          → 背景色 200 级
--state-success           → 成功状态色
--shadow-md               → 中等阴影
--brand-radius-sm         → 品牌小圆角
```

### 12.2 各端映射方式

| 端 | 令牌载体 | 示例 |
|----|----------|------|
| Web / H5 | CSS Custom Properties | `var(--primary)` |
| iOS | Swift Enum / Asset Catalog | `Color.brand500` |
| Android | XML Resources / Compose | `R.color.brand_500` |
| 鸿蒙 | Resource 文件 | `$r('app.color.brand_500')` |
| 微信小程序 | WXSS 变量 / JS 常量 | `var(--primary)` in WXSS |

### 12.3 令牌消费规则

| 规则 | 说明 |
|------|------|
| 组件仅消费语义令牌 | 禁止在组件代码中直接引用原始色板（如 `--brand-500`） |
| 原始色板仅用于令牌定义 | 原始色板仅在 `:root` 语义映射中使用 |
| 禁止硬编码色值 | 不允许在组件中出现 `#007AFF` 等字面色值 |
| 状态色独立使用 | 状态色（success/error/warning/info）仅在对应语义场景使用 |
