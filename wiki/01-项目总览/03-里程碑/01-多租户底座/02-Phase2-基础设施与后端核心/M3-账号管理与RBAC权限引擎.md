## 四、技术要点

- 角色体系：9 种预定义角色（含 Public 第 0 层），6 层继承；45 权限点矩阵。
- 鉴权流程：JWT 校验 → 提取 `X-Organization-ID/Team-ID/Group-ID` → membership 校验（含层级回溯 group→team→org→global）→ 权限校验（权限缓存）→ 数据范围控制。
- 权限缓存降低鉴权延迟，保障 API 95% < 100ms（NFR-PERF-001）。

## 五、接口清单（代表）

- `GET/PUT /api/v1/account/profile`
- `POST /api/v1/account/password/change`
- `POST /api/v1/account/deactivate`、`POST /api/v1/account/reactivate`
- 鉴权中间件统一应用于受保护路由（401 / 403 标准返回）

## 六、质量门禁与 DOD

- **DOD**：权限校验主链路单测 ≥ 80%；权限继承与数据范围控制有专项用例；接口契约一致；SAST 零高危。
