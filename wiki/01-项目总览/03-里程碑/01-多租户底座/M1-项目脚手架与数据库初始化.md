## 四、技术要点

### 4.1 工程目录结构

```
code/backend/
  cmd/xyfamily/main.go         # 入口
  internal/
    handler/                    # API 入口（按模块分包）
    service/                    # 业务逻辑
    repository/                 # 数据访问
    model/                      # 领域模型
    middleware/                 # 横切中间件
    init/                       # 系统初始化
  pkg/
    jwt/                        # JWT 封装
    bcrypt/                     # 密码哈希
    response/                   # 统一响应
    errors/                     # 错误码
    logger/                     # 日志
  configs/config.yaml
  migrations/
```

### 4.2 数据库初始化

- 13 张表：accounts, verification_codes, invitations, sessions, organizations, organization_members, teams, team_members, groups, group_members, roles, permissions, role_permissions, login_audit_logs, operation_audit_logs
- 预初始化：6 角色 + 45 权限点 + role_permissions 映射
- 幂等执行：使用 `INSERT ... ON CONFLICT DO NOTHING`

### 4.3 配置管理

- 优先级：环境变量 > config.yaml > 代码默认值
- 敏感配置（JWT 密钥、DB 密码）不得写入配置文件
