## 四、测试策略

### 4.1 测试金字塔

```
        ┌───────────┐
        │  E2E (少)  │  ← 前端核心流程
        ├───────────┤
        │ 集成 (中)  │  ← 接口级测试
        ├───────────┤
        │ 单元 (多)  │  ← service/repository 层
        └───────────┘
```

### 4.2 多租户隔离测试矩阵

| 场景 | 预期结果 |
|------|---------|
| 用户 A 查询组织 A 的成员 | ✅ 200 |
| 用户 A 查询组织 B 的成员 | ❌ 403 |
| 用户 A 修改组织 B 的配置 | ❌ 403 |
| 用户 A 邀请到组织 B | ❌ 403 |
| 未登录用户访问受保护接口 | ❌ 401 |
| 过期 Token 访问 | ❌ 401 + 自动刷新 |

### 4.3 安全扫描清单

| 检查项 | 工具 | 标准 |
|--------|------|------|
| 代码静态分析 | golangci-lint | 0 error |
| 依赖漏洞 | go vuln check | 无高危 |
| SQL 注入 | 手动 + sqlmap | 无注入点 |
| XSS | 手动 + 扫描器 | 输入过滤 |
| CSRF | 手动 | Token 校验 |
| 越权访问 | 手动 | 多租户隔离 |
| 密码暴力破解 | 手动 | 限流生效 |
| Token 伪造 | 手动 | HS256 签名校验 |
