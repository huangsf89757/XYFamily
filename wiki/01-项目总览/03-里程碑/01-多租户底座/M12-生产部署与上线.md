## 四、技术要点

### 4.1 K8s 部署架构

```
Ingress (Nginx / ALB)
  ├── xyfamily-1 (Go 后端)
  ├── xyfamily-2
  └── xyfamily-3
        │
        ├── PostgreSQL 主库 ←→ PostgreSQL 从库
        ├── Redis Master ←→ Redis Replica
        │   └── Sentinel × 3
        └── MinIO / S3
```

### 4.2 数据库高可用

- **流复制**：主库 → 从库，同步模式
- **WAL 归档**：持续归档到对象存储
- **PITR**：Point-in-Time Recovery，支持恢复到任意时间点
- **备份策略**：全量备份 1 次/天 + WAL 持续归档

### 4.3 Redis 高可用

- **Sentinel 集群**：3 个 Sentinel 节点
- **自动故障转移**：Master 宕机 → 自动选举新 Master
- **应用层适配**：使用 Sentinel 模式连接，自动感知 Master 切换

### 4.4 监控指标

| 指标 | 告警阈值 |
|------|---------|
| HTTP 5xx 错误率 | > 1% |
| P99 响应时间 | > 500ms |
| PG 连接数 | > 80 (max 100) |
| Redis 内存使用 | > 80% |
| CPU 使用率 | > 80% |
| 磁盘使用率 | > 85% |

### 4.5 灰度发布

```
10% 流量 → 验证 30min → 50% 流量 → 验证 1h → 100% 流量
```

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 部署环境差异 | 高 | 本地 Docker Compose 先验证 → 预发布环境验证 → 生产 |
