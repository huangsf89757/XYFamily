## 二、模块依赖与开发顺序

```
后端主线（串行）：
M0(文档) → M1(脚手架) → M2(认证) → M3(账号+RBAC) → M4(组织) → M5(团队+小组) → M6(审计+超管)

前端主线（M4 完成后并行）：
                    M4 完成后 ──┬── M7(Web 管理后台)
                               ├── M8(H5 + 小程序)
                               └── M9(iOS) → M10(Android) → M11(集成测试) → M12(上线)
                                                                          ↓
                                                                   M13(鸿蒙) → M14(P2)
```

**关键依赖关系**：
- M3 依赖 M2（认证模块提供 JWT 基础设施）
- M4 依赖 M3（组织管理需要 RBAC 权限中间件）
- M7/M8/M9 依赖 M4（前端需要组织管理 API 才能开发核心功能）
- M10 依赖 M9（Android 复用 iOS 验证过的 API 对接方案与交互设计）
- M11 依赖 M6 + M10（后端 + iOS/Android 端开发完成后才能集成测试）
- M12 依赖 M11（安全审查通过后才能上线）
- M13 依赖 M12（系统上线稳定后开发鸿蒙端）
- M14 依赖 M13（P2 功能在所有端就绪后迭代）

### 移动端三阶段开发路线

```
阶段一          阶段二          阶段三（上线后）
M9 iOS    →    M10 Android  →    M13 鸿蒙
Swift         Kotlin            ArkTS
SwiftUI       Compose           ArkUI
Keychain      EncryptedPrefs    HUKS
```
