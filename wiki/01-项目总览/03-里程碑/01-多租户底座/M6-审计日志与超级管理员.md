## 四、技术要点

### 4.1 审计日志架构

```
请求 → Handler → Service → Repository → DB
  │
  └── 审计中间件 → Redis Stream (XADD) ──→ Consumer ──→ operation_audit_logs
```

- **异步写入**：审计日志通过 Redis Stream 异步写入，不阻塞主流程
- **断点续写**：Consumer 使用 consumer group + XACK，保证不丢失
- **操作类型**：33 种 `audit_operation_type` 枚举（认证/账号/组织/团队/小组/管理）

### 4.2 登录审计 vs 操作审计

| 维度 | 登录审计 | 操作审计 |
|------|---------|---------|
| 表 | `login_audit_logs` | `operation_audit_logs` |
| 写入方式 | 同步写入（M2 已实现） | 异步写入（Redis Stream） |
| 内容 | login_method, login_result, failure_reason | operation_type, target_type, target_id, details(JSONB) |
| 保留 | 1 年 | 1 年 |

### 4.3 SuperAdmin CLI

```bash
# 交互式创建（生产环境）
xyfamily admin init
# > 输入手机号: 13800138000
# > 输入密码（不回显）: ********
# > 确认密码: ********
# ✅ 超级管理员创建成功

# 环境变量创建（开发环境）
XY_SUPER_ADMIN_PHONE=13800138000 \
XY_SUPER_ADMIN_PASSWORD=xxx \
xyfamily admin init
```

### 4.4 强制降级

SuperAdmin 可以跨组织强制降级任何用户的角色，不受最后管理员保护限制。降级后触发 Redis 缓存失效。

## 六、风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| Redis Stream 宕机导致审计丢失 | 高 | Stream 持久化 + consumer group 断点续写 |
| 审计日志写入性能 | 中 | 异步 + 批量写入 + 索引优化 |
| CLI 密码泄露 | 中 | 交互式输入 + 不进 shell history |
| 超管权限过大 | 中 | 全局审计记录超管所有操作 |
