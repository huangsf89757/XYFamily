## 四、技术要点

### 4.1 组织创建流程

```
用户创建组织 → 写入 organizations → 创建者写入 organization_members (role=organization_core_admin)
  └── 若指定 initial_admin → 发送邀请 (role=organization_core_admin)
```

### 4.2 邀请流程

```
管理员发起邀请 → identifier 查找账号 → 写入 invitations (status=pending)
  ├── 被邀请人接受 → 创建 membership → 删除 Redis perm cache → invitation status=accepted
  └── 被邀请人拒绝 → invitation status=rejected
```

- identifier 支持：手机号 / 邮箱 / 用户名 / account_id
- 邀请有效期：7 天

### 4.3 最后管理员保护

- 组织中最后一个 organization_core_admin 不可降级为 RegularMember
- 组织中最后一个 organization_core_admin 不可被移除
- 校验方式：`SELECT COUNT(*) FROM organization_members WHERE organization_id=? AND role='organization_core_admin' AND status='active'`

### 4.4 多租户隔离

- 所有查询强制带 `organization_id` 条件
- Repository 层封装 `FindByOrgID(orgID, ...)`
- 禁止跨组织查询
