### 阶段目标

将 XYFamily 从「多租户账号权限底座」升级为「可插拔工具集平台」，实现：

1. **插件 SDK** — 屏蔽底座细节，提供统一 API（用户上下文、权限校验、数据访问、事件订阅）
2. **插件市场** — 插件发现、安装、配置、评分、更新、卸载的全生命周期管理
3. **首批业务插件** — 开发 1-2 个示范插件，验证端到端流程
4. **插件安全** — 沙箱隔离、权限越界检测、操作审计

## 三、核心能力设计方向

### 3.1 插件生命周期

```
开发/注册 → 审核 → 上架 → 安装 → 启用 → 配置 → 运行 → 停用 → 卸载
                                    │                          │
                                    └──── 可回滚到停用 ─────────┘
```

| 阶段 | 说明 | 关键操作 |
|------|------|---------|
| 开发/注册 | 开发者编写插件，提交注册信息 | 声明权限点、依赖项、版本 |
| 审核 | 平台管理员审核插件安全性 | 安全扫描 + 人工审核 |
| 上架 | 审核通过后在插件市场可见 | 版本发布、更新说明 |
| 安装 | 组织管理员选择安装到本组织 | 权限确认、依赖检查 |
| 启用 | 激活插件功能 | 初始化数据表、注册路由 |
| 配置 | 管理员配置插件参数 | 按插件定义的配置项 |
| 运行 | 插件正常运行，处理业务 | 自动审计、权限校验 |
| 停用 | 暂时停用插件功能 | 保留数据，路由不可用 |
| 卸载 | 彻底卸载插件 | 清理数据表（需确认）、移除路由 |

### 3.2 插件权限模型

| 维度 | 设计方向 |
|------|---------|
| 权限声明 | 插件在注册时声明所需权限点集合 |
| 权限审批 | 组织管理员在安装时审批授权，可随时收回 |
| 权限校验 | 复用底座 RBAC 引擎，插件调用 API 时自动校验 |
| 权限粒度 | 组织级 / 团队级 / 小组级（插件可挂载到不同层级） |

### 3.3 插件隔离方案

> 具体方案待 ADR 决策，以下为候选方案：

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| Go Plugin（动态加载） | 同进程，低延迟 | 仅 Linux，GC 约束 | 内部可信插件 |
| 微服务（独立进程） | 语言无关，完全隔离 | 网络开销，运维复杂 | 第三方插件 |
| WASM（WebAssembly） | 沙箱安全，跨平台 | 生态不成熟，性能折中 | 轻量计算型插件 |

### 3.4 插件 SDK 接口（草案）

```go
// Go SDK 核心接口（草案）
type PluginSDK interface {
    // 用户上下文
    GetCurrentUser() (User, error)
    GetOrganization() (Organization, error)

    // 权限校验
    HasPermission(code string) bool
    RequirePermission(codes ...string) error

    // 数据访问（按 org_id 自动隔离）
    Query(table string, filter Filter) (Result, error)
    Insert(table string, data Data) (ID, error)
    Update(table string, filter Filter, data Data) error
    Delete(table string, filter Filter) error

    // 事件订阅
    Subscribe(event string, handler EventHandler) error
    Publish(event string, payload Payload) error

    // 审计日志
    Audit(action string, detail Detail) error

    // 配置
    GetConfig() (Config, error)
    SetConfig(config Config) error
}
```

## 四、与 Phase 1 的衔接点

| 底座能力（Phase 1） | 插件复用方式（Phase 2） |
|---------------------|------------------------|
| JWT 认证体系 | 插件 SDK 自动注入当前用户上下文 |
| RBAC 权限引擎 | 插件声明权限点 → 底座统一校验 |
| 多租户 org_id 隔离 | 插件数据自动按 org_id 隔离 |
| 审计日志中间件 | 插件操作自动记录审计日志 |
| 组织/团队/小组层级 | 插件可挂载到不同层级 |
| 超级管理员 | 超管管理全局插件市场、审核上架 |

## 六、风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 插件隔离方案不成熟 | P2-M0 阻塞 | 提前调研 Go plugin / 微服务 / WASM，Phase 1 末期完成 ADR |
| 底座 RBAC 不支持插件动态权限点 | P2-M1 阻塞 | Phase 1 权限引擎设计预留扩展点 |
| 插件 SDK 接口设计不当导致重构 | P2-M4 返工 | P2-M0 充分评审，首批插件验证接口可行性 |
| 插件市场冷启动 | P2-M4 延期 | 内部团队开发示范插件，验证流程后开放 |
