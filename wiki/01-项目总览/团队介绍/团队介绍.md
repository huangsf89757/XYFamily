## 一、团队定位

XYFamily 是一套面向 SaaS 场景的**多租户账号权限底座**，团队围绕「账号、组织、团队、角色、权限」等核心能力，协同交付 Web / H5 / 小程序 / 移动端（安卓 / iOS / 鸿蒙）四端产品。

团队成员与工号（XYF-00x）为**示例占位**，待真实成员确定后由技术负责人替换回填。本目录的价值在于：**在成员确定后，可直接依据各成员档案中的「AI Agent 配置建议」快速创建专属 Agent**，让 Agent 继承成员的角色、职责、技能与常用 Wiki 入口，提升协作效率。

## 三、团队结构与协作关系

### 3.1 三层结构

- **决策层**：技术负责人（张明远）与产品经理（李思琪）共同作为 G0–G4 评审门禁决策人，对里程碑准出负责。
- **交付层**：后端（陈浩然）+ 四端（Web / H5 / 小程序 / iOS / 安卓 / 鸿蒙）协同，统一消费同一套账号权限底座与接口契约。
- **保障层**：测试（黄诗涵）与运维（徐天磊）贯穿全生命周期，与评审门禁、风险登记册共同构成研发治理闭环。

### 3.2 协作链路

```
产品经理(李思琪) ──需求/PRD──▶ UI设计师(王梓萱) ──设计稿──┐
                                                          ▼
技术负责人(张明远) ──架构/接口契约──▶ 后端(陈浩然) ──接口──▶ 各端开发
                                                          │
测试(黄诗涵) ──用例/验收◀──────────────────────────────┘
                                                          │
运维(徐天磊) ──部署/监控/应急◀──发布── 测试准出
```

各阶段准出需决策层（张明远 / 李思琪）在 [评审门禁记录](../07-评审门禁记录/评审门禁记录.md) 中确认；风险同步登记至 [风险登记册](../08-风险登记册/风险登记册.md)。

## 四、AI Agent 创建指南

为每个成员创建 Agent 时，按以下步骤操作，无需从零设计角色与上下文：

1. **读取档案**：打开对应成员档案（如 `./01-张明远.md`），定位「七、AI Agent 配置建议」。
2. **复制模板**：将 YAML 配置块复制到 Agent 配置中（Agent 名称、角色画像、知识库、行为规范、协作边界）。
3. **挂载知识**：按 `knowledge` 列表将对应 Wiki 目录作为 Agent 可检索上下文。
4. **绑定责任人**：Agent 的「署名」填写该成员姓名 / 工号（依据《文档规范》1.3）。
5. **联调协作**：按 `collaborators` 配置 Agent 间的转交 / 通知关系。

> 12 份成员档案已内置可直接落地的 YAML 模板，复制即可使用。

## 五、AI Agent 配置总览

每个成员档案均包含「AI Agent 配置建议」章节，给出可直接落地的 Agent 配置要素：

| 要素 | 说明 |
|------|------|
| Agent 名称 | 以成员姓名 / 工号命名，便于追溯 |
| 角色画像 | 成员的职责、技能栈、交付物 |
| 可用知识 | 该成员最常用的 Wiki 文档入口 |
| 行为规范 | 编写人填写、文档规范、评审门禁要求 |
| 协作关系 | 上下游合作成员与对接边界 |

## 七、命名与版本约定

- **Agent 命名**：`姓名-工号`，如 `张明远-XYF-001`，保证唯一与可追溯。
- **文档署名**：严格遵循《文档规范》1.3，以真实责任人姓名 / 工号署名，AI 工具名仅记入变更内容。
- **配置版本**：成员档案随 Wiki 语义化版本递增（V1.x.0）；成员变动（替换 / 新增）时由技术负责人回填并追加变更记录。
