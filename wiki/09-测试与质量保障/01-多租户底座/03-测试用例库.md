## 1. 文档定位

本文件是测试用例的组织规范与索引入口，承接《测试策略 - 测试与质量保障》的分级测试模型，为功能 / 接口 / 异常用例提供统一编写与归档标准。

## 2. 用例组织维度

- **按模块**：对齐《产品 PRD - 多租户底座》9 大功能模块（注册认证、账号管理与 RBAC、组织管理、团队与小组、审计日志与超管等）。
- **按类型**：功能用例、接口契约用例、异常 / 边界用例、多租户隔离用例（见《专项测试 - 测试与质量保障》）。
- **按端**：网页 / 桌面 / 移动 / 小程序 + iOS / 鸿蒙 的差异化用例。

## 3. 用例编写规范

| 字段 | 说明 |
|------|------|
| 用例 ID | 模块前缀 + 序号（如 AUTH-001） |
| 标题 | 简述测试点 |
| 前置条件 | 环境 / 数据 / 权限上下文（含 X-Organization-ID） |
| 步骤 | 可复现操作序列 |
| 预期结果 | 明确断言，关联 PRD 验收标准 |
| 优先级 | P0 / P1 / P2 |
| 类型 | 功能 / 接口 / 异常 / 隔离 |
