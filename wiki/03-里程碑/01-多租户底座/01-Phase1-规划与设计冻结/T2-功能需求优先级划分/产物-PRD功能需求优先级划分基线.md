## 1. 目的与范围

- **目的**：将 PRD §3 中分散在各模块的 FR 优先级，收敛为一份**首期范围基线契约**，作为 G0 门禁「范围与优先级（P0/P1）锁定」准出条件的证据。
- **范围**：9 大功能模块（认证 / 账号 / 组织 / 团队 / 小组 / 权限 / 超级管理员 / 邀请 / 审计）共 69 条 FR。
- **评审基准**：[多租户底座 PRD](../../../../../04-需求与产品设计/01-产品PRD/01-多租户底座/多租户底座.md)（V1.0.0）§3 功能需求规格。

## 2. 优先级划分原则

| 等级 | 定义 | 首期处置 |
|------|------|----------|
| P0 | 底座核心能力，缺失则系统不可用 / 不安全 | **锁定首期**（必须交付） |
| P1 | 首期完整性所需，影响管理闭环与审计合规 | **锁定首期**（纳入交付） |
| P2 | 增强体验 / 扩展能力，首期可缺失 | **延后入 Backlog**，目标 Phase 6（M17） |

> 首期边界与 [M0 §四 技术要点](../../M0-立项评审与需求设计冻结.md#四技术要点) 一致：P2 功能延后至 Phase 6（M17）。

## 3. P0 — 首期必须（45 条）

| 模块 | FR 编号 | 需求描述 |
|------|---------|----------|
| 用户认证 | FR-AUTH-001~011 | 手机号/邮箱 注册与登录（密码/验证码）、Token 刷新、登出、密码重置 |
| 账号管理 | FR-ACCT-001~005 | 获取/更新个人信息、修改密码、账号注销（30天宽限）、账号恢复 |
| 组织管理 | FR-ORG-001~009 | 创建组织、组织信息增改查、邀请/移除成员、成员列表、分配/降级核心管理员、创建团队 |
| 团队管理 | FR-TEAM-001~008 | 团队信息增改查、邀请/移除成员、成员列表、分配/降级核心管理员、创建小组 |
| 小组管理 | FR-GROUP-001~008 | 小组信息增改查、删除小组、邀请/移除成员、成员列表、分配/降级核心管理员 |
| 权限管理 | FR-PERM-001~004 | 角色与权限点初始化、权限点校验、数据范围控制、权限继承 |

## 4. P1 — 首期包含（19 条）

| 模块 | FR 编号 | 需求描述 |
|------|---------|----------|
| 团队管理 | FR-TEAM-009 | 归档团队（软删除） |
| 超级管理员 | FR-ADMIN-003/004 | 系统配置查看与修改（即时生效并记录审计） |
| 超级管理员 | FR-ADMIN-005 | 强制降级任意层级管理员 |
| 超级管理员 | FR-ADMIN-006 | 全局审计日志查看（跨组织） |
| 超级管理员 | FR-ADMIN-007 | 高危操作二次确认 |
| 邀请管理 | FR-INV-001~003、005~008 | 创建/多渠道发送/接受邀请、取消邀请、邀请查询、过期失效、指定目标角色 |
| 审计日志 | FR-AUDIT-001、002~005、006 | 登录审计、操作审计（组织/团队/小组/账号域）、审计日志查询 |

## 5. P2 — 延后入 Backlog（5 条）

| 模块 | FR 编号 | 需求描述 | 延后目标 |
|------|---------|----------|----------|
| 用户认证 | FR-AUTH-012 | 微信登录（扫码，未绑定需先绑定） | Phase 6（M17） |
| 用户认证 | FR-AUTH-013 | 登录自动注册（账号不存在时自动注册） | Phase 6（M17） |
| 账号管理 | FR-ACCT-006 | 绑定第三方身份（微信等） | Phase 6（M17） |
| 账号管理 | FR-ACCT-007 | 解绑第三方身份 | Phase 6（M17） |
| 邀请管理 | FR-INV-004 | 拒绝邀请（状态置 rejected） | 入 Backlog，随邀请模块迭代评估 |

> 说明：前 4 项与 M0 §四 技术要点明确的首期边界一致；FR-INV-004 在 PRD 中标记为 P2，其能力可由「管理员侧取消邀请（FR-INV-005）」与「用户忽略」覆盖，不阻塞首期邀请闭环，故一并延后。

## 6. 首期范围基线结论

- **首期交付规模**：P0（45 条）+ P1（19 条）= **64 条 FR**；P2（5 条）延后。
- **基线约束**：首期不实现任何 P2 能力；若评审/研发中需要新增或提升优先级，须走 Backlog 评审（见 M0 §七 RISK-009 范围蔓延防控）。
- 本基线为 G0「范围与优先级（P0/P1）锁定」准出条件的依据之一。
