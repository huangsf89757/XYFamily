## 模块概述

账号管理模块处理当前登录用户的个人信息管理。所有接口需认证（Bearer Token），操作对象为 Token 中 `account_id` 对应的账号。

**依赖关系**：
- **数据库表**：`accounts`, `sessions`
- **缓存**：Redis（Token 黑名单）

## 1. 查看个人信息

| 项目 | 内容 |
|------|------|
| 方法 | `GET` |
| 路径 | `/api/v1/account/profile` |
| 认证 | ✅ |

### 请求 Header

```
Authorization: Bearer <access_token>
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "uuid",
    "phone": "13800138000",
    "email": "user@example.com",
    "username": "zhangsan",
    "nickname": "张三",
    "avatar_url": "https://cdn.example.com/avatar.jpg",
    "status": "active",
    "created_at": "2026-07-13T10:00:00Z",
    "last_login_at": "2026-07-13T11:00:00Z"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| account_id | string (UUID) | 账号 ID |
| phone | string | 手机号（完整显示，本人可见） |
| email | string | 邮箱（完整显示，本人可见） |
| username | string | 用户名 |
| nickname | string | 昵称 |
| avatar_url | string/null | 头像 URL |
| status | string (枚举) | `active` / `deactivating` / `deactivated` |
| created_at | string (ISO 8601) | 注册时间 |
| last_login_at | string (ISO 8601) | 最后登录时间 |

### 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 3. 修改密码

| 项目 | 内容 |
|------|------|
| 方法 | `PATCH` |
| 路径 | `/api/v1/account/password` |
| 认证 | ✅ |

### 请求 Header

```
Authorization: Bearer <access_token>
```

### 请求体

```json
{
  "old_password": "Old@Pass#12345",
  "new_password": "New@Strong#Pass2024"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| old_password | string | ✅ | 当前密码 |
| new_password | string | ✅ | 新密码（≥12 位，含大小写+数字+特殊字符） |

### 业务流程

1. 校验 access_token
2. 验证 `old_password` 与数据库中密码哈希匹配（bcrypt.CompareHashAndPassword）
3. 验证新密码强度（≥12 位，含大小写+数字+特殊字符）
4. 更新密码哈希（bcrypt cost=12）
5. **当前 Access Token 写入 Redis 黑名单**（强制重新登录）
6. 写入操作审计日志

**副作用**：更新密码哈希 → 当前 Access Token 写入 Redis 黑名单 → 写入操作审计日志。

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": null,
  "pagination": null
}
```

### 错误响应

```json
{
  "code": 200005,
  "message": "当前密码错误",
  "detail": "旧密码不匹配，请重新输入",
  "data": null,
  "pagination": null
}
```

### 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 400 | 200001 | 新密码强度不足 |
| 400 | 200005 | 当前密码错误 |

## 4. 申请注销

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/account/deactivate` |
| 认证 | ✅ |

### 请求 Header

```
Authorization: Bearer <access_token>
```

### 业务流程

1. 校验 access_token
2. 账号状态从 `active` → `deactivating`
3. member role 清空（撤销所有组织/团队/小组成员角色）
4. 设置 30 天宽限期
5. 写入操作审计日志

**副作用**：账号状态 → `deactivating` → member role 清空 → 30 天宽限期。

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "status": "deactivating",
    "expires_at": "2026-08-12T10:00:00Z",
    "note": "30 天宽限期，期间可通过恢复接口取消注销"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string (枚举) | `deactivating`（注销中） |
| expires_at | string (ISO 8601) | 宽限期到期时间（30 天后） |
| note | string | 提示信息 |

### 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 5. 恢复账号

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/account/undeactivate` |
| 认证 | ✅ |
| 前提 | 账号状态 = `deactivating`（宽限期内） |

### 请求 Header

```
Authorization: Bearer <access_token>
```

### 业务流程

1. 校验 access_token
2. 检查账号状态是否为 `deactivating`
3. 检查是否在 30 天宽限期内
4. 账号状态恢复为 `active`
5. member role 恢复
6. 写入操作审计日志

**副作用**：账号状态恢复为 `active`，member role 恢复。

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "status": "active",
    "restored_at": "2026-07-13T10:00:00Z"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string (枚举) | `active`（已恢复） |
| restored_at | string (ISO 8601) | 恢复时间 |

### 错误响应

```json
{
  "code": 200006,
  "message": "宽限期已过，不可恢复",
  "detail": "账号注销宽限期已结束，账号已被永久注销",
  "data": null,
  "pagination": null
}
```

### 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 400 | 200006 | 宽限期已过，不可恢复 |

## 账号状态流转

```
active ──POST /account/deactivate──> deactivating ──30天到期──> deactivated
   ^                                      │
   └────POST /account/undeactivate────────┘
                （宽限期内）
```

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| `active` | 正常账号 | 所有操作 |
| `deactivating` | 注销宽限期（30 天） | 可登录、可恢复、可查看信息 |
| `deactivated` | 已永久注销 | 不可登录、不可恢复 |
