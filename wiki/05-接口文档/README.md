## 一、通用规范

### 1.1 请求规范

| 项目 | 值 |
|------|-----|
| Base Path | `/api/v1/` |
| Content-Type | `application/json` |
| 认证 Header | `Authorization: Bearer <access_token>` |
| 组织上下文 | `X-Organization-ID: {uuid}`（受保护接口必填） |
| 团队上下文 | `X-Team-ID: {uuid}`（团队相关接口必填） |
| 小组上下文 | `X-Group-ID: {uuid}`（小组相关接口必填） |

### 1.2 响应格式

**标准信封（所有接口统一）：**

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {},
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务状态码。`0`=成功；`10xxx`~`80xxx`=业务错误（详见 [错误码](./错误码-V1.0.0.md)） |
| message | string | 错误摘要（中文） |
| detail | string/null | 详细错误信息（成功时为 `null`） |
| data | any/null | 成功时返回业务数据；失败时为 `null` |
| pagination | object/null | 仅分页接口返回，成功且非分页时为 `null` |

### 1.3 分页格式

**分页接口的 `data` 字段结构：**

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  },
  "pagination": null
}
```

**分页查询参数（所有列表接口统一）：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码（从 1 开始） |
| page_size | int | 20 | 每页数量（最大 100） |

**筛选/排序查询参数（各接口按需支持）：**

| 参数 | 类型 | 说明 |
|------|------|------|
| keyword | string | 关键词搜索 |
| role | string | 角色筛选 |
| from | string | 起始时间（ISO 8601） |
| to | string | 结束时间（ISO 8601） |
| sort | string | 排序字段（如 `created_at`） |
| order | string | 排序方向（`asc`/`desc`，默认 `desc`） |

### 1.4 错误响应格式

```json
{
  "code": 101002,
  "message": "Token 无效或已过期",
  "detail": "Token 签名验证失败或已超出有效期",
  "data": null,
  "pagination": null
}
```

> HTTP Status 表示请求层级语义（401/403/404/429/500），业务 `code` 表示具体错误原因。两者分离。完整错误码列表见 [错误码](./错误码-V1.0.0.md)。

### 1.5 时间格式

- 所有时间字段使用 **ISO 8601** 格式
- 示例：`2026-07-13T10:00:00Z`
- 时区统一使用 **UTC**

## 三、系统接口

### 健康检查

| 项目 | 内容 |
|------|------|
| 方法 | `GET` |
| 路径 | `/healthz` |
| 认证 | ❌ 公开 |

**成功响应：**

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "status": "ok",
    "version": "1.0.0",
    "uptime": "72h15m30s",
    "dependencies": {
      "database": "ok",
      "redis": "ok"
    }
  },
  "pagination": null
}
```

> 依赖组件异常时，对应字段值为 `"error"`，HTTP Status 仍为 200（便于 LB/探针识别）。
