## 一、写入链路（异步不阻塞）

```mermaid
sequenceDiagram
    participant H as Handler
    participant RD as Redis Stream
    participant CG as Consumer Group
    participant PG as audit_logs

    H->>RD: XADD audit_stream * (event JSON, 含 event_id)
    H-->>Client: 直接返回业务响应
    CG->>RD: XREADGROUP 拉取事件
    CG->>PG: INSERT audit_logs ON CONFLICT (event_id) DO NOTHING
    CG->>RD: XACK 确认
```

- **依据**：ADR-011、NFR-PERF-006（P99 < 1s）。
- 主流程不等待落库；Stream 保证至少一次（at-least-once），consumer group + `XACK` 防重复消费。
- 断点续写：消费者崩溃后从未 `XACK` 的 pending 列表（`XPENDING`）恢复，不丢数据。
- **ARCH-003 修复——幂等去重**：每条审计事件携带 `event_id`（UUID），consumer 落库时使用 `INSERT ... ON CONFLICT (event_id) DO NOTHING`。当 consumer 在 `XACK` 之前崩溃并恢复时，重复消费同一消息不会产生重复记录，实现幂等。

## 二、事件结构

推入 Stream 的事件 JSON（与 `audit_logs` 字段对齐）：

```json
{
  "event_id": "uuid",
  "account_id": "uuid|null",
  "org_id": "uuid|null",
  "action_domain": "login|org|team|group|account|system_config",
  "action_type": "auth.login",
  "target_type": "organization",
  "target_id": "uuid|null",
  "result": "success|failed",
  "failure_reason": "password_error|...",
  "details": { "before": {}, "after": {} },
  "trace_id": "opentelemetry-trace-id",
  "ip_address": "1.2.3.4",
  "user_agent": "..."
}
```

> **ARCH-003 修复**：`event_id` 为审计事件唯一标识（UUID v4），由 Handler 在 `XADD` 前生成。Consumer 据此实现幂等去重。
>
> **ARCH-008 修复（MIN-008）**：`trace_id` 关联 OpenTelemetry 链路追踪，实现审计日志与请求链路的联动查询。

## 三、脱敏与防篡改

| 要求 | 实现 | 依据 |
|------|------|------|
| 凭证不落明文 | `details` 中密码/Token/验证码一律不记录；密码变更仅记 `{"changed":true}` | FR-AUDIT-005/010 |
| PII 加密存储 | phone/email 入库加密；查询/导出展示按需脱敏（`138****0000`） | NFR-SEC-009、FR-AUDIT-010 |
| 仅追加不可篡改 | `audit_logs` 无 `updated_at`/`deleted_at`，应用层禁止 UPDATE/DELETE；SuperAdmin 亦不可改 | FR-AUDIT-009 |
| 匿名化不可逆 | 注销后 `account_id` 关联 PII 替换为 `anon_` 前缀，逻辑单向 | NFR-SEC-007 |

## 四、保留与匿名化策略

| 策略 | 实现 |
|------|------|
| 保留期 | 默认 365 天（可 SuperAdmin 配置 `audit.retention_days`，FR-ADMIN-004） |
| 到期处理 | 定时任务清理超过保留期的日志（软归档至冷存储 / 删除，按合规要求） |
| 账号注销 | 触发 `account.deactivate` 流程后，由 [账号生命周期与身份方案](../05-支撑域/01-账号生命周期与身份方案.md) 对 `audit_logs` 中该账号 PII 匿名化 |
| 防丢失 | Stream 持久化 + PG 主从；详见 [容灾多活与高可用](../06-横切专项/02-容灾多活与高可用.md) |

## 五、查询实现

- **本组织查询**（FR-AUDIT-007）：`organization_core_admin` 经 [审计接口](../03-数据模型与契约/02-接口设计/09-审计接口.md) 查询，强制 `org_id = 当前组织`，防越权。
- **全局查询**（FR-AUDIT-008）：SuperAdmin 查全部，额外支持 `org_id` 过滤。
- **分页/筛选**：索引 `(account_id/org_id, created_at DESC)`，按时间倒序分页；操作类型/结果过滤。
- **详情**：返回完整 `details` 前后快照（脱敏后）。
