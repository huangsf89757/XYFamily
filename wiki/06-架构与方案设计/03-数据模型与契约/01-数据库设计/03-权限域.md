# 权限域

> P3 数据模型之三。定义 RBAC 引擎的落库结构：`roles` / `permission_points` / `role_permissions`，以及 SuperAdmin 系统配置 `system_configs`。本域承载 P2《RBAC 权限引擎方案》的「预填充固化」——初始化时把高层角色的权限点集合**直接展开写入** `role_permissions`，运行时权限校验免递归（NFR-PERF-001）。

## 一、本域表清单

| 表 | 说明 | 依据 |
|----|------|------|
| `roles` | 9 种角色定义 | ADR-008、FR-PERM-001 |
| `permission_points` | 45 个权限点 | ADR-008、FR-PERM-001 |
| `role_permissions` | 角色→权限点映射（含继承预填充） | ADR-007、FR-PERM-004 |
| `system_configs` | SuperAdmin 系统配置 | FR-ADMIN-003/004 |

> SuperAdmin（`super_admin`）是游离角色，不写入任何 `*_members` 表（见 [RBAC权限引擎方案](../../02-核心域/02-RBAC权限引擎方案.md)），其权限来自 `role_permissions` 中 `super_admin` 的全量映射。

## 二、`roles`

```sql
CREATE TABLE roles (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    role_key      VARCHAR(32) NOT NULL UNIQUE,   -- public|regular_member|...|super_admin
    name          VARCHAR(64) NOT NULL,
    level         SMALLINT NOT NULL,             -- L0~L8
    scope         VARCHAR(16) NOT NULL,          -- global|organization|team|group|personal
    description   TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**预初始化数据**（9 角色，level 与 scope）：

| role_key | level | scope | 说明 |
|----------|-------|-------|------|
| `public` | 0 | global | 公共角色，每用户自动拥有 |
| `regular_member` | 1 | personal | 普通成员 |
| `group_ordinary_admin` | 2 | group | 小组普通管理员 |
| `group_core_admin` | 3 | group | 小组核心管理员 |
| `team_ordinary_admin` | 4 | team | 团队普通管理员 |
| `team_core_admin` | 5 | team | 团队核心管理员 |
| `organization_ordinary_admin` | 6 | organization | 组织普通管理员 |
| `organization_core_admin` | 7 | organization | 组织核心管理员 |
| `super_admin` | 8 | global | 超级管理员 |

## 三、`permission_points`

```sql
CREATE TABLE permission_points (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    permission_key VARCHAR(64) NOT NULL UNIQUE,  -- 如 org.member.invite
    module        VARCHAR(16) NOT NULL,          -- auth|account|org|team|group|audit|admin
    description   TEXT,
    is_public     BOOLEAN NOT NULL DEFAULT FALSE, -- 公开权限点（Public 持有）
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**预初始化数据**（45 个，权威矩阵见 [权限管理模块](../../../04-需求与产品设计/01-产品PRD/01-多租户底座/06-权限管理模块/权限管理模块.md)）：

- 认证 5：auth.register、auth.login、auth.reset_password、auth.logout、auth.refresh
- 账号 7：account.profile.read/update、account.password.update、account.deactivate、account.undeactivate、account.third_party.bind/unbind
- 组织 9：org.create/read/update、org.member.invite/remove/list、org.role.assign/downgrade、org.team.create
- 团队 9：team.read/update、team.member.invite/remove/list、team.role.assign/downgrade、team.group.create、team.archive
- 小组 8：group.read/update/delete、group.member.invite/remove/list、group.role.assign/downgrade
- 审计 3：audit.login.read、audit.operation.read、audit.query
- 超级管理员 4：admin.config.read/update、admin.force_downgrade、admin.audit.global

其中 `auth.register` / `auth.login` / `auth.reset_password` 置 `is_public = TRUE`。

## 四、`role_permissions`（预填充固化）

```sql
CREATE TABLE role_permissions (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    role_key      VARCHAR(32) NOT NULL,
    permission_key VARCHAR(64) NOT NULL,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (role_key, permission_key)
);

CREATE INDEX idx_role_permissions_role ON role_permissions(role_key);
```

**固化规则**（来自 [RBAC权限引擎方案](../../02-核心域/02-RBAC权限引擎方案.md)）：初始化时高层的映射 = 自身权限点 + 其下所有低层角色的权限点（一次性展开，无需运行时递归）。

| 角色 | 有效权限点构成（含继承） |
|------|--------------------------|
| public(L0) | 3 个公开权限点 |
| regular_member(L1) | 自身权限点 |
| group_ordinary_admin(L2) | 小组普通 + regular_member |
| group_core_admin(L3) | 小组核心 + 小组普通 + regular_member |
| team_ordinary_admin(L4) | 团队普通 + 小组普通 + regular_member |
| team_core_admin(L5) | 团队核心 + 团队普通 + 小组核心(含小组普通) + regular_member |
| organization_ordinary_admin(L6) | 组织普通 + 团队普通 + 小组普通 + regular_member |
| organization_core_admin(L7) | 组织核心 + 组织普通 + 团队核心 + 小组核心 + regular_member |
| super_admin(L8) | 全部 45 个权限点 |

> 变更与刷新：角色/权限矩阵调整后重新执行初始化重新预填充，并主动失效相关账号权限缓存（`perm:account:{id}:org:{org_id}`，见 [缓存设计方案](../../05-支撑域/02-缓存设计方案.md)）。

## 五、`system_configs`

```sql
CREATE TABLE system_configs (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    config_key    VARCHAR(64) NOT NULL UNIQUE,   -- 如 token.access_ttl
    config_value  TEXT NOT NULL,                  -- 存 JSON 或标量字符串
    description   TEXT,
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_by    UUID                            -- 操作人 account_id（SuperAdmin）
);
```

**预初始化配置项**（FR-ADMIN-004）：

| config_key | 默认值 | 说明 |
|------------|--------|------|
| `token.access_ttl` | `1800` | Access Token 秒（NFR-SEC-004） |
| `token.refresh_ttl` | `604800` | Refresh Token 秒 |
| `login.rate_limit.threshold` | `5` | 失败阈值 |
| `login.rate_limit.window` | `300` | 统计窗口秒 |
| `login.rate_limit.lock` | `900` | 锁定时长秒 |
| `verification_code.ttl` | `300` | 验证码有效期秒 |
| `audit.retention_days` | `365` | 审计保留期（NFR-SEC-007） |
| `account.deactivate_grace_days` | `30` | 注销宽限期（ADR-010） |
| `login.auto_register` | `false` | 登录自动注册开关（P2 开放，首版关闭） |
