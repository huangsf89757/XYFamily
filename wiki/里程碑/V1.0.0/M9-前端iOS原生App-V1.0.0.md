### 目标

完成 iOS 原生 App 的核心功能开发，覆盖登录认证、个人中心、组织/团队/小组管理全流程。作为移动端三阶段开发的第一阶段，iOS 端先行验证移动端 API 对接与交互设计，为后续 Android 和鸿蒙端提供参考实现。

## 四、技术要点

### 4.1 iOS 技术栈

| 类别 | 选型 | 版本 |
|------|------|------|
| 语言 | Swift | 5.9+ |
| UI 框架 | SwiftUI | - |
| 网络 | URLSession | - |
| 本地存储 | Core Data | - |
| 安全存储 | Keychain | Refresh Token 存储 |
| 依赖注入 | - | Environment |

### 4.2 Token 管理

| 平台 | Access Token | Refresh Token |
|------|-------------|---------------|
| iOS | 内存（AccessTokenManager） | Keychain |

### 4.3 统一 API 层

- **Token 自动刷新**：401 时自动调用 `/auth/refresh`，成功后重试原请求
- **多租户 Header**：自动附加 `X-Organization-ID` / `X-Team-ID` / `X-Group-ID`
- **统一错误码处理**：错误码 → 中文文案映射 → Alert 展示
