## 一、预研背景与目标

M0 阶段在冻结需求（PRD）与确认原型后，需对落地技术栈进行预研，明确选型方向并初判关键技术风险，避免进入编码阶段后出现方向性返工。本报告为 M1 技术方案评审（G1）的输入。

## 二、技术选型预研

| 维度 | 候选方案 | 选型结论 | 依据（ADR/文档） | 状态 |
|------|----------|----------|------------------|------|
| 开发语言 | Go / Java / Node | Go 1.22+ | ADR-001 性能与并发 | 已定 |
| Web 框架 | Gin / Echo | Gin | 整体架构设计 | 已定 |
| 关系数据库 | PostgreSQL / MySQL | PostgreSQL 17 | ADR 多租户 | 已定 |
| 缓存 / 消息 | Redis 7（Sentinel） | Redis | 缓存 / 审计方案 | 已定 |
| 鉴权 | JWT（Access + Refresh） | HS256（可升级 RS256） | ADR-003 / ADR-016 | 已定 |
| PII 加密 | AES-GCM + 盲索引 | AES-256-GCM + HMAC 盲索引 | ARCH-001 修复方案 | 方案已定，待落地 |
| 对象存储 | MinIO（S3 兼容） | MinIO | 整体架构 | 已定 |
| 部署运维 | 无状态多实例 + LB + PG 主从 + Redis Sentinel | 采用 | 容灾多活方案 | 已定 |

> 完整决策见 `06-架构与方案设计/01-基座/02-ADR架构决策记录.md`（ADR-001~016）。

## 三、关键技术风险初判

| ID | 风险 | 等级 | 简述 | 关联文档 |
|----|------|------|------|----------|
| RISK-ARCH-001 | PII 加密后登录查询性能 | 极高 | 加密字段等值查询需盲索引，否则无法命中索引 | 架构评审记录 |
| RISK-ARCH-002 | 权限缓存粒度不足越权 | 高 | 缓存键需细化到 scope 级 | 架构评审记录 |
| RISK-ARCH-004 | 跨存储事务不一致 | 高 | 登出/改密需明确失败处理策略 | 架构评审记录 |
| RISK-ARCH-005 | PRD 与架构决策矛盾 | 高 | deactivating 登录决策需回溯 PRD | 架构评审记录 |
| RISK-ARCH-007 | 跨层级邀请悬空数据 | 高 | 级联成员关系需同事务 | 架构评审记录 |

> 其余架构风险（ARCH-003/006/008 等）见 [风险登记册](../../08-风险登记册/风险登记册.md)。

## 四、可行性结论

- 技术栈成熟、与 PRD / CLAUDE.md 约束一致，**总体可行**。
- 6 项 P0 级技术风险（ARCH-001/002/004/005/007 及 PII 加密）须在编码前关闭方案，并在 M1（G1）评审通过。
- 多租户隔离采用应用层 `org_id` 强制隔离（非 PG RLS），与约束一致。

## 五、预研待决项与进入 M1 前置条件

1. PII 加密 / 盲索引方案在数据库设计文档落地（ARCH-001）。
2. 权限缓存键细化到 scope 级（ARCH-002）。
3. 跨存储事务失败策略明确（ARCH-004）。
4. PRD 回溯修订（DEF-003/005 等）关闭（ARCH-005/007）。
5. 风险登记册随 G1–G4 持续维护。
