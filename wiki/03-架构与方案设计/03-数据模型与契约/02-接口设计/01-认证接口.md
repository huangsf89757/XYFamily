# 认证接口

> P3 接口契约之一。定义用户认证模块（注册 / 登录 / 登出 / 刷新 / 重置 / 发送验证码）的 API 契约。本接口是《P2 JWT 鉴权链》与 [账号与认证域](../01-数据库设计/01-账号与认证域.md) 的对外契约，遵循 NFR-COMP-001（`/api/v1/`）与 ADR-013 错误码体系。

## 一、通用约定

- **Base URL**：`/api/v1`，全部 JSON（NFR-COMP-001/002）。
- **统一响应**：`{ "code": 0, "message": "ok", "detail": null, "data": {} }`；创建类 201，其余 200。
- **鉴权**：除标注「公开」的接口外，需 `Authorization: Bearer <Access Token>`。
- **错误码段**：认证类 `10xxx`（详见 [认证错误码](#六错误码)；全局 `80xxx` / `400` 参数 `100001` / `429` 频控）。
- **多端会话**：Refresh/登出支持单设备；详见 [JWT鉴权链与Token方案](../../02-核心域/03-JWT鉴权链与Token方案.md)。
- **CORS**（DMC-008 修复，NFR-COMP-003）：在 Gin 中间件层全局启用跨域支持，来源可配置（环境变量 `XY_CORS_ORIGINS`），允许携带凭证（`Access-Control-Allow-Credentials: true`）。

## 二、发送验证码

`POST /api/v1/auth/verification-codes`　**公开**

请求：
```json
{ "target": "13800000000", "type": "register|login|reset_password" }
```
响应 `200`：`{ "data": { "expires_in": 300 } }`
错误：`114290`（target 频控）/ `114291`（IP 频控）/ `114293`（账号级频控）/ `100001`（参数错误）。

---

## 三、注册

`POST /api/v1/auth/register`　**公开（auth.register）**

请求：
```json
{ "type": "phone|email", "target": "13800000000", "code": "123456", "password": "Str0ng@Pass12" }
```
响应 `201`：
```json
{ "data": {
  "account_id": "uuid", "nickname": "用户1234",
  "access_token": "eyJ...", "refresh_token": "eyJ...",
  "expires_in": 1800 } }
```
错误：`200002`（手机已注册）/ `200003`（邮箱已注册）/ `110001~110004`（验证码）/ `200001`（密码强度不足）。

---

## 四、登录

`POST /api/v1/auth/login`　**公开（auth.login）**

请求：
```json
{ "identifier": "13800000000", "identifier_type": "phone|email|username|account_id",
  "credential_type": "password|code", "password": "...", "code": "123456" }
```
响应 `200`：`{ "data": { "account_id", "access_token", "refresh_token", "expires_in", "status": "active|deactivating" } }`

> **DMC-003 修复**（ARCH-005 对齐）：`deactivating` 状态的账号**允许登录**（返回 Token + `status: "deactivating"`），但登录后仅允许执行「恢复账号」（`account.undeactivate`）操作，其他业务操作返回 `403` + `200007`。`deactivated` 状态拒绝登录（`101009`）。前端根据 `status` 字段引导用户进入恢复流程。

错误：`101007`（账号不存在）/ `101008`（密码错误）/ `101009`（已注销 deactivated）/ `110001~110004`（验证码）/ `104290`（登录锁定）。

---

## 五、刷新 Token

`POST /api/v1/auth/refresh`　**auth.refresh**

请求：`{ "refresh_token": "eyJ..." }`
响应 `200`：`{ "data": { "access_token", "refresh_token", "expires_in" } }`（Rotation：旧 Refresh 立即作废）
错误：`101004`（无效）/ `101005`（已撤销）/ `101006`（过期）/ `101010`（被盗用，全员下线）/ `104291`（刷新频控）。

---

## 六、登出

`POST /api/v1/auth/logout`　**auth.logout**（需 Bearer Access）

请求：`{ "refresh_token": "eyJ..." }`（可选，带则单设备撤销；不带则按当前 Access 仅黑名单）
响应 `200`：`{ "data": null }`（**幂等**：已失效仍返回成功）

错误码段（认证 `10xxx`）：

| HTTP | code | 说明 |
|------|------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效/过期 |
| 401 | 101003 | Token 已被撤销（黑名单） |
| 401 | 101007 | 账号不存在 |
| 401 | 101008 | 密码错误 |
| 401 | 101009 | 账号已注销 |
| 401 | 101010 | Refresh 被盗用 |
| 400 | 110001~110004 | 验证码错误/过期/已用/超限 |
| 429 | 104290/104291 | 登录/刷新频控 |
| 429 | 114290/114291/114293 | 验证码频控（target/IP/账号级） |
| 400 | 200001/200002/200003/200005 | 密码强度/已注册/新密码相同 |
| 403 | 200007 | 账号处于注销宽限期，仅可执行恢复操作（DMC-003 修复，业务操作层拦截） |

---

## 七、重置密码（忘记密码）

`POST /api/v1/auth/reset-password`　**公开（auth.reset_password）**

请求：`{ "type": "phone|email", "target": "...", "code": "123456", "new_password": "New@Pass34" }`
响应 `200`：`{ "data": null }`（账号不存在也返回成功，防枚举，FR-AUTH-011）
错误：`200001`（强度不足）/ `200005`（新密码相同）/ `110001~110004`（验证码）/ `104292`（重置频控）。
