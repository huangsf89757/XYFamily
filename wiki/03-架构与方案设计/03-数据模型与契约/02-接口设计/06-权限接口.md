## 一、鉴权范围

- 查询自身权限需携带对应 scope Header（`X-Organization-ID` 等），由后端做层级回溯（[P2 多租户隔离 §五](../../02-核心域/01-多租户隔离方案.md)）确定生效角色。
- 角色列表 / 权限点矩阵为公开只读元数据（不含敏感）。
- 错误码段 `60xxx`。

---

## 二、查询当前 scope 我的有效权限

`GET /api/v1/rbac/me`　（需 scope Header，登录即可）

响应 `200`：
```json
{ "data": {
  "scope": { "org_id": "uuid", "team_id": null, "group_id": null },
  "effective_role": "organization_core_admin",
  "permission_points": ["org.read","org.update", "..."] } }
```
错误：`600001`（scope 上下文缺失/非法，含回溯后仍非成员）。

> 返回权限点集合直接来自 Redis 缓存或 `role_permissions` 预填充展开（[P2 RBAC §五/六](../../02-核心域/02-RBAC权限引擎方案.md)）。

---

## 三、角色列表

`GET /api/v1/rbac/roles`

响应 `200`：`{ "data": { "items": [
  {"role_key":"public","level":0,"scope":"global"},
  {"role_key":"organization_core_admin","level":7,"scope":"organization"}, ... ] } }`

---

## 四、权限点矩阵

`GET /api/v1/rbac/permission-points`

响应 `200`：`{ "data": { "items": [
  {"permission_key":"org.create","module":"org","is_public":false},
  {"permission_key":"auth.register","module":"auth","is_public":true}, ... ], "total": 45 } }`

> 权威矩阵见 [PRD 权限点矩阵](../../02-需求与产品设计/01-产品PRD/01-多租户底座/06-权限管理模块/权限管理模块-V1.0.0.md#五权限点矩阵)；共 45 个。

---

## 五、错误码段（60xxx 摘要）

| code | 说明 |
|------|------|
| 600001 | scope 上下文缺失/非成员 |
| 600002 | 权限点非法 |
