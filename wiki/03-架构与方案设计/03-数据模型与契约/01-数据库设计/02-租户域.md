# 数据库设计 · 租户域

> P3 数据模型之二。定义三级租户模型的落库结构：`organizations` / `teams` / `groups` 及其对应的 `*_members` 成员关系表。本域是多租户隔离（[P2 多租户隔离方案](../../02-核心域/01-多租户隔离方案.md)）与组织/团队/小组三大模块接口的数据基座，严格遵循「应用层隔离 + 全表 `org_id` 贯穿 + 复合唯一成员约束」。

## 一、租户层级与隔离约束

- 层级：`Organization → Team → Group`，组织间完全隔离（NFR-SEC-006）。
- 外键承载父子关系：`teams.org_id` → `organizations`，`groups.team_id` → `teams`，并**冗余 `org_id`** 便于索引与隔离校验。
- 隔离主键：所有业务表携带 `org_id`；查询强制带 `org_id` 条件（C2）。
- 成员关系：`*_members` 复合唯一 `(scope_id, account_id)`，单角色列 `role`（ADR-002）。

## 二、`organizations`

```sql
CREATE TABLE organizations (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id        UUID NOT NULL UNIQUE DEFAULT gen_random_uuid(),
    name          VARCHAR(128) NOT NULL,
    description   VARCHAR(256),           -- 组织描述（DMC-006 修复，与 teams.description 统一为 0-256 字符）
    owner_id      UUID NOT NULL,            -- 创建者/初始管理员 account_id
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    disabled_at   TIMESTAMPTZ,              -- 组织停用标记（ARCH-013，仅 SuperAdmin 操作，NULL=正常）
    deleted_at    TIMESTAMPTZ               -- 软删除（组织一般不硬删）
);

CREATE INDEX idx_organizations_deleted ON organizations(deleted_at);
CREATE INDEX idx_organizations_owner   ON organizations(owner_id);
```

## 三、`org_members`

```sql
CREATE TABLE org_members (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id        UUID NOT NULL,
    account_id    UUID NOT NULL,
    role          VARCHAR(32) NOT NULL,     -- organization_core_admin|organization_ordinary_admin|regular_member
    joined_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at    TIMESTAMPTZ,              -- 软删除（匿名化时仅清空 role，不删行）
    UNIQUE (org_id, account_id)
);

CREATE INDEX idx_org_members_org       ON org_members(org_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_org_members_account   ON org_members(account_id) WHERE deleted_at IS NULL;
```

**角色取值**：`organization_core_admin` / `organization_ordinary_admin` / `regular_member`（见 [P2 RBAC](../../02-核心域/02-RBAC权限引擎方案.md) §二）。
**最后管理员保护**：移除/降级操作需校验是否为该组织最后一名 `organization_core_admin`，是则禁止（ADR-014，由 SuperAdmin 强制处理）。

## 四、`teams`

```sql
CREATE TABLE teams (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    team_id       UUID NOT NULL UNIQUE DEFAULT gen_random_uuid(),
    org_id        UUID NOT NULL,            -- 冗余，便于隔离
    name          VARCHAR(128) NOT NULL,
    description   VARCHAR(256),             -- 团队描述（MIN-004 修复，PRD DEF-006 要求 0-256 字符）
    owner_id      UUID NOT NULL,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    archived_at   TIMESTAMPTZ,              -- 软归档（区别于硬删）
    deleted_at    TIMESTAMPTZ,
    UNIQUE (org_id, team_id)
);

CREATE INDEX idx_teams_org       ON teams(org_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_teams_archived  ON teams(archived_at);
CREATE INDEX idx_teams_deleted   ON teams(deleted_at);
```

## 八、`invitations` 邀请表

```sql
CREATE TABLE invitations (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id        UUID NOT NULL,            -- 冗余隔离键
    scope_type    VARCHAR(16) NOT NULL,     -- organization|team|group
    scope_id      UUID NOT NULL,            -- 指向 organizations/teams/groups 的 id
    inviter_id    UUID NOT NULL,            -- 发起邀请的 account_id
    invitee_email VARCHAR(255),             -- 邮箱渠道的被邀请人邮箱（链接/码渠道可空）
    invitee_account UUID,                   -- 接受后绑定的账号（接受前可空）
    role          VARCHAR(32) NOT NULL,     -- 接受时落地的成员角色
    channel       VARCHAR(16) NOT NULL,     -- email|link|code
    invite_code   VARCHAR(32) UNIQUE,       -- 邀请码（code/link 渠道）
    invite_link   VARCHAR(512),             -- 一次性链接
    token_hash    VARCHAR(128),             -- 链接/码签名摘要，防篡改
    status        VARCHAR(16) NOT NULL DEFAULT 'pending', -- pending|accepted|rejected|canceled|expired
    expires_at    TIMESTAMPTZ NOT NULL,     -- 失效时间（默认创建+7d，可配 1-30d）
    accepted_at   TIMESTAMPTZ,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at    TIMESTAMPTZ,
    UNIQUE (scope_type, scope_id, invitee_email, status) WHERE status = 'pending'
);

CREATE INDEX idx_invitations_org      ON invitations(org_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_invitations_scope    ON invitations(scope_type, scope_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_invitations_email    ON invitations(invitee_email) WHERE deleted_at IS NULL AND status = 'pending';
CREATE INDEX idx_invitations_expires  ON invitations(expires_at) WHERE status = 'pending';
```

**状态机**：`pending → accepted | rejected | canceled | expired`。接受成功后写入对应 `*_members` 表并建立成员关系（见 [03-组织接口](../02-接口设计/03-组织接口.md)）；邀请记录全表 `org_id` 贯穿，跨组织不可见（NFR-SEC-006）。

## 九、隔离与索引约定

- **所有租户表 `org_id` 必带索引**，隔离过滤走索引前缀（NFR-PERF-001：95% < 100ms）。
- 成员查询同时带 `org_id` + `scope_id` + `account_id` 复合条件，确保跨组织不可见。
- 父子归属校验：写入 `teams`/`groups` 时校验 `org_id`/`team_id` 父子自洽（与 [P2 上下文校验](../../02-核心域/01-多租户隔离方案.md) §四一致）。
- 软删除（ADR-006）：`teams.archived_at`（归档）、其余 `deleted_at`；所有查询排除已删除行。
