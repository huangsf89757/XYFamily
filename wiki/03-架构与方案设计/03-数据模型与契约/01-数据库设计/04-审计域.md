## 一、设计要点（来自 ADR-011 / 审计 PRD）

- **单表**：登录审计 + 操作审计同表，以 `action_type` 区分（原双表方案合并简化）。
- **仅追加不可篡改**：无 `updated_at` / `deleted_at`，禁止 UPDATE/DELETE。
- **异步写入**：经 Redis Stream + consumer group 落库（见 [P4 审计日志方案](../../04-链路实现/03-审计日志方案.md)），不阻塞主流程（NFR-PERF-006：P99 < 1s）。
- **PII 脱敏**：手机号/邮箱等加密存储；`details` 不记录密码/Token 明文（FR-AUDIT-010）。
- **保留与匿名化**：保留 1 年（NFR-SEC-007）；账号注销后对 `account_id` 关联信息匿名化（`anon_` 前缀），不可逆向还原。

## 二、`audit_logs`

```sql
CREATE TABLE audit_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    event_id        UUID NOT NULL UNIQUE,                 -- 审计事件唯一标识（ARCH-003 修复，用于消费者去重）
    account_id      UUID,                       -- 操作账号（登录失败/初始化可为 NULL）
    org_id          UUID,                       -- 组织维度（全局操作如 SuperAdmin 可为 NULL）
    action_domain   VARCHAR(16) NOT NULL,       -- login|org|team|group|account|system_config
    action_type     VARCHAR(48) NOT NULL,       -- 见下方枚举
    target_type     VARCHAR(32),                -- 操作对象类型（organization/team/...）
    target_id       UUID,                       -- 操作对象 ID
    result          VARCHAR(16),                -- success|failed（登录类）
    failure_reason  VARCHAR(64),                -- 失败原因（密码错误/验证码错误/限流...）
    login_method    VARCHAR(16),                -- 登录方式（DMC-001 修复）：password|sms_code|email_code|wechat（仅 login 域）
    details         JSONB,                      -- 前后快照/操作详情（不记明文凭证）
    trace_id        VARCHAR(64),                -- 链路追踪 ID（ARCH-008 修复，关联 OpenTelemetry）
    ip_address      INET,
    user_agent      TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_audit_account   ON audit_logs(account_id, created_at DESC);
CREATE INDEX idx_audit_org       ON audit_logs(org_id, created_at DESC) WHERE org_id IS NOT NULL;
CREATE INDEX idx_audit_action    ON audit_logs(action_type, created_at DESC);
CREATE INDEX idx_audit_login_method ON audit_logs(login_method, created_at DESC) WHERE login_method IS NOT NULL;  -- DMC-001 修复
CREATE INDEX idx_audit_created   ON audit_logs(created_at DESC);
-- event_id 唯一索引已在字段定义中（UNIQUE），用于消费者 INSERT ON CONFLICT DO NOTHING 去重
```

> 设计取舍：不设 `updated_at` / `deleted_at`，从表结构层面强制「仅追加」。查询高基索引走 `(account_id/org_id, created_at DESC)` 复合，支撑分页与筛选（FR-AUDIT-007/008）。

## 三、`action_type` 枚举

**登录域（action_domain=login）**

| action_type | 说明 | 关联需求 |
|-------------|------|----------|
| `auth.register` | 注册成功 | FR-AUTH-001/002 |
| `auth.login` | 登录尝试（含 success/failed） | FR-AUTH-003~008 |
| `auth.logout` | 登出 | FR-AUTH-010 |
| `auth.refresh` | Token 刷新 | FR-AUTH-009 |
| `auth.reset_password` | 密码重置 | FR-AUTH-011 |

**组织/团队/小组域**

| 域 | action_type |
|----|-------------|
| org | `org.create` / `org.update` / `org.member.invite` / `org.member.remove` / `org.role.assign` / `org.role.downgrade` |
| team | `team.update` / `team.archive` / `team.member.invite` / `team.member.remove` / `team.role.assign` / `team.role.downgrade` |
| group | `group.update` / `group.delete` / `group.member.invite` / `group.member.remove` / `group.role.assign` / `group.role.downgrade` |

**账号域（account）**：`account.password.update` / `account.password.reset` / `account.deactivate` / `account.undeactivate` / `account.third_party.bind` / `account.third_party.unbind`

**系统配置域（system_config）**：`admin.init` / `admin.config.update` / `admin.force_downgrade`

> **DMC-002 修复**：新增 `admin.init`（SuperAdmin 初始化），满足 PRD FR-ADMIN-001「初始化操作本身需记录审计日志」要求。`account_id` 初始为 NULL → 创建后回填新 `account_id`，`details` 记录 `{"method":"cli"/"config","email":"admin@x.com"}`（不记密码明文）。

> 操作类型可扩展注册（NFR-SCAL-004），新增 `action_type` 不影响核心逻辑。

## 四、`details` JSONB 结构约定

| 操作类型 | details 示例字段 |
|----------|------------------|
| 角色变更 | `{"before":{"role":"regular_member"},"after":{"role":"group_core_admin"}}` |
| 组织信息修改 | `{"before":{"name":"A"},"after":{"name":"B"}}` |
| 密码修改/重置 | `{"changed":true}`（不记明文） |
| 账号注销/恢复 | `{"before":{"status":"active"},"after":{"status":"deactivating"}}` |
| 配置修改 | `{"before":{"config_value":"1800"},"after":{"config_value":"3600"}}` |
| 强制降级 | `{"before":{"role":"organization_core_admin"}}` |
| SuperAdmin 初始化 | `{"method":"cli"/"config","email":"admin@x.com"}`（不记密码明文，DMC-002 修复） |
| 登录审计 | `{"login_method":"password"/"sms_code"/"email_code"/"wechat"}`（DMC-001 修复，亦可由独立 `login_method` 列查询） |

> **ARCH-003 修复**：`event_id` 字段为审计事件唯一标识（UUID），在 Stream 消息中携带。Consumer 落库时使用 `INSERT ... ON CONFLICT (event_id) DO NOTHING` 实现幂等去重，避免 at-least-once 语义下的重复 INSERT。
>
> **ARCH-008 修复（MIN-008）**：新增 `trace_id` 字段，与可观测性方案的 OpenTelemetry trace_id 关联，实现审计日志与链路追踪的联动查询。
