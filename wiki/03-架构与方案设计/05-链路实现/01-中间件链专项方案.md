## 一、中间件执行顺序

```mermaid
graph LR
    Req["请求"] --> RL["① RateLimiter<br/>登录/刷新频控"]
    RL --> JWT["② JWTValidator<br/>Access 签名+有效期"]
    JWT --> BL["③ TokenBlacklist<br/>jti 黑名单"]
    BL --> MV["④ MembershipValidator<br/>X-* 上下文 + 层级回溯"]
    MV --> PC["⑤ PermissionChecker<br/>权限点集合校验"]
    PC --> AU["⑥ Audit<br/>异步审计 + 业务 Handler"]
```

> 公开接口（`auth.register/login/reset_password`、发送验证码、`admin.init`）在 ② 之前短路放行，不参与 ④⑤；但仍经 ①（频控）与 ⑥（登录审计）。

## 二、各中间件实现要点

### ① RateLimiter（登录/刷新限流）
- **依据**：NFR-SEC-002、PRD 登录限流（IP 5min 5 次 → 锁 15min）、刷新频控 104291。
- **实现**：Redis 计数器（`rl:ip:{ip}` 窗口 300s 阈值 5；`rl:acct:{account_id}` 叠加）。命中阈值返回 `429`（104290/104291）。
- **范围**：仅作用于认证/刷新接口；全局业务接口不触发账号锁定。

### ② JWTValidator（Access 校验）
- **依据**：NFR-SEC-004/005、[P2 JWT §二/四](../../03-核心域/03-JWT鉴权链与Token方案.md)。
- **实现**：校验签名（`kid` 区分 HS256/RS256 轮换）、`exp`；失败返回 `401`（101002）。不信任客户端任何权限声明。
- **输出**：写入 `context.account_id` / `org_ids` / `roles` / `jti`。

### ③ TokenBlacklist（黑名单）
- **依据**：ADR-003、[P2 JWT §五](../../03-核心域/03-JWT鉴权链与Token方案.md)。
- **实现**：查 Redis `bl:{jti}`，命中返回 `401`（101003）。TTL=剩余有效期。
- **成本**：每次 +1 次 Redis 读（<1ms，ADR-003 后果已评估）。

### ④ MembershipValidator（上下文 + 层级回溯）
- **依据**：[P2 多租户隔离 §四/五](../../03-核心域/01-多租户隔离方案.md)、FR-PERM-003。
- **实现**：
  1. 解析 `X-Organization-ID`/`X-Team-ID`/`X-Group-ID`，校验父子自洽，否则 `400`。
  2. 多组织用户必填，否则 `400`（上下文缺失）；单组织用户由 `org_ids` 推断。
  3. 确定生效角色：目标 scope 直接成员 → 直接角色；否则向上回溯 `group→team→org→global`，命中上级成员则按二维继承取同层级权限；全程无成员返回 `403`（非成员）。
  4. SuperAdmin 跳过回溯，直接全局。
- **结果**：`context.scope` + `context.effective_role` 供 ⑤ 使用。

### ⑤ PermissionChecker（权限点校验）
- **依据**：FR-PERM-002、[P2 RBAC §六](../../03-核心域/02-RBAC权限引擎方案.md)。
- **实现**：
  1. 由 `context.effective_role` 取该角色有效权限点集合（Redis `perm:account:{id}:org:{org_id}`，TTL 24h；未命中查 `role_permissions` 预填充并回填）。
  2. 与接口所需权限点比对（接口→权限点映射由路由元数据声明）。
  3. 不含 → `403`（600001 类）。
  4. 缓存不可用回退 DB 查询（不阻断，仅性能下降）。

### ⑥ Audit（审计）
- **依据**：ADR-011、[P4 审计日志方案](./03-审计日志方案.md)。
- **实现**：在 Handler 成功后（或登录/注册等入口处）将审计事件推入 Redis Stream，由 consumer group 异步落 `audit_logs`。不阻塞主流程（NFR-PERF-006）。

## 三、短路与错误处理

| 阶段失败 | HTTP | code | 行为 |
|----------|------|------|------|
| 限流命中 | 429 | 104290/104291 | 直接返回 |
| Token 无效/过期 | 401 | 101002 | 短路 |
| Token 已撤销 | 401 | 101003 | 短路 |
| 上下文缺失/非法 | 400 | 100001 | 短路 |
| 非成员 | 403 | 600001 | 短路 |
| 权限不足 | 403 | 60xxx | 短路 |

> 统一响应格式 `{code, message, detail, data}`（ADR-013）。
