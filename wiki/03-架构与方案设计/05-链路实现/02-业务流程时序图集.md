## 一、注册流程（手机号 + 验证码）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API(Gin)
    participant MW as 中间件链
    participant SVC as Account/Auth SVC
    participant RD as Redis(验证码/黑名单)
    participant PG as PostgreSQL

    C->>API: POST /auth/register {phone, code, password}
    API->>MW: ①RateLimiter + ②JWT跳过(公开)
    MW->>RD: 校验验证码 vc:{phone}:register
    alt 验证码无效
        RD-->>C: 400 110001/110004
    else 通过
        MW->>SVC: 创建账号
        SVC->>PG: INSERT accounts (默认 active)
        SVC->>PG: INSERT sessions (Refresh)
        SVC->>RD: 写黑名单? (无) 生成 Token 对
        SVC->>RD: XADD audit_logs Stream (register)
        SVC-->>C: 201 {access, refresh}
    end
```

## 二、登录流程（密码）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as 中间件链
    participant SVC as Auth SVC
    participant RD as Redis
    participant PG as PG

    C->>API: POST /auth/login {identifier, password}
    API->>MW: ①RateLimiter(IP/账号维度)
    MW->>SVC: 识别标识 + bcrypt 校验
    SVC->>PG: 查 accounts(按 phone/email/username/id)
    alt 失败/锁定
        SVC->>RD: 失败计数 + 锁定
        SVC-->>C: 401 101008 / 429 104290
    else 成功
        SVC->>PG: 更新 last_login_at
        SVC->>PG: INSERT sessions
        SVC->>RD: XADD audit (login success)
        SVC-->>C: 200 {access, refresh}
    end
```

## 三、登出流程（幂等）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant MW as ②③④ 中间件
    participant RD as Redis
    participant PG as PG

    C->>API: POST /auth/logout (Bearer Access + refresh?)
    MW->>RD: ②校验 Access 签名 → ③查 bl:{jti}
    alt 已失效
        MW-->>C: 200 幂等成功
    else 有效
        MW->>RD: SET bl:{jti} (TTL=剩余)
        MW->>PG: UPDATE sessions SET revoked_at (按 refresh)
        MW->>RD: XADD audit (logout)
        MW-->>C: 200 成功
    end
```

## 四、刷新 Token（Rotation）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant PG as PG(sessions)
    participant RD as Redis(黑名单)

    C->>API: POST /auth/refresh {refresh_token}
    API->>PG: 校验会话存在/未撤销/未过期
    alt 已被使用(疑似被盗)
        API->>PG: 该账号全部会话 revoked
        API->>RD: 相关 jti 入黑名单
        API-->>C: 401 101010 全员下线
    else 正常
        API->>PG: 旧会话 revoked + 新会话 INSERT
        API-->>C: 200 {新 access, 新 refresh}
    end
```

## 五、邀请 → 接受（统一邀请流）

```mermaid
sequenceDiagram
    participant I as 邀请人
    participant API as API
    participant MW as ④Membership ⑤Permission
    participant PG as PG(invitations/members)
    participant A as 被邀请人

    I->>API: POST /organizations/{id}/members/invitations
    MW->>MW: ④上下文 + ⑤ org.member.invite 校验
    API->>PG: INSERT invitations (status=pending, token)
    A->>API: POST /invitations/{token}/accept (或登录后确认)
    API->>PG: 校验 token 一次性 + 未过期
    API->>PG: INSERT org_members (role=指定)
    API->>PG: UPDATE invitations status=accepted
    API->>PG: 失效 perm 缓存 perm:account:{id}:org:{org_id}
    API-->>A: 200 加入成功
```

## 六、注销 → 恢复（宽限期）

```mermaid
sequenceDiagram
    participant C as Client
    participant API as API
    participant SVC as Account SVC
    participant PG as PG(accounts)

    C->>API: POST /account/deactivate
    API->>SVC: 置 status=deactivating, deactivated_at=now+30d
    SVC-->>C: 200 进入宽限期
    Note over C,PG: 宽限期内不可业务操作；定时任务扫描到期→匿名化
    C->>API: POST /account/undeactivate {code}
    API->>SVC: 校验验证码 + 未超宽限
    SVC->>PG: status=active (角色权限自动恢复)
    SVC-->>C: 200 恢复成功
```

## 七、组织创建（接棒机制）

```mermaid
sequenceDiagram
    participant S as SuperAdmin
    participant API as API
    participant MW as ④⑤ (org.create)
    participant PG as PG(organizations/members)
    participant RD as Redis(perm 缓存)

    S->>API: POST /organizations {name, initial_admin?}
    MW->>MW: ⑤ org.create 仅 SuperAdmin
    API->>PG: INSERT organizations (owner=创建者)
    API->>PG: INSERT org_members (创建者=organization_core_admin)
    alt 指定 initial_admin
        API->>PG: INSERT invitations (role=organization_core_admin, pending)
    end
    API->>RD: XADD audit (org.create)
    API-->>S: 201 {org_id}
```

## 八、典型权限校验（访问团队接口）

```mermaid
sequenceDiagram
    participant C as Client
    participant MW as 中间件链
    participant RD as Redis(perm)
    participant PG as PG(role_permissions)

    C->>MW: GET /teams/{id} + X-Org/Team-ID
    MW->>MW: ① ② ③ 通过
    MW->>MW: ④ 解析 scope + 层级回溯定角色
    MW->>RD: GET perm:account:{id}:org:{org_id}
    alt 命中
        RD-->>MW: 权限点集合
    else 未命中
        MW->>PG: SELECT permission_key FROM role_permissions WHERE role_key=生效角色
        PG-->>MW: 集合
        MW->>RD: SET 缓存 TTL=24h
    end
    MW->>MW: ⑤ 含 team.read? 是→放行 否→403
    MW->>C: 业务 Handler 处理
```
