## 一、测试分层

```
┌─────────────────────────────────────────────┐
│              E2E 测试                         │  ← 关键用户路径
├─────────────────────────────────────────────┤
│            集成测试 (Integration)             │  ← API + DB + Redis
├─────────────────────────────────────────────┤
│            单元测试 (Unit)                    │  ← 函数/方法级
└─────────────────────────────────────────────┘
```

| 层级 | 目标覆盖率 | 执行频率 | 工具 |
|------|-----------|---------|------|
| 单元测试 | ≥ 80% | 每次提交 | `go test` |
| 集成测试 | ≥ 60% | 每次 PR | `go test -tags=integration` |
| E2E 测试 | 关键路径全覆盖 | CI + 发布前 | Playwright / cypress |

### 2.2 测试命名规范

```go
// 文件名：{被测文件}_test.go
// 函数名：Test{被测函数}_{场景}
func TestAuthService_Login_Success(t *testing.T)

func TestAuthService_Login_AccountLocked(t *testing.T)
```

### 2.3 AAA 模式

```go
func TestPermissionChecker_CacheHit(t *testing.T) {
    // Arrange
    ctx := setupContextWithToken(t, "valid-token")
    setupRedisCache(t, "perm:account:1:org:1", []string{"org.read"})

    // Act
    handler := PermissionChecker("org.read")
    handler(ctx)

    // Assert
    assert.Equal(t, http.StatusOK, ctx.Writer.Status())
}
```

|-------------|----------|
| PostgreSQL | sqlx.NewMock 或 testcontainers |

| Service 依赖 | 接口 + mock 实现 |

## 三、集成测试规范

### 3.1 测试数据库

| 方案 | 说明 |
|------|------|
| Testcontainers | 每次测试启动独立 PostgreSQL 17 + Redis 7 容器 |
| 事务回滚 | 每个测试用例后 `db.Rollback()` 恢复数据 |
| 清理策略 | 测试结束后删除 testcontainers |

```go
func setupTestDB(t *testing.T) (*gorm.DB, func()) {
    dbContainer, err := postgres.Run(t,
        "postgres:17",
        postgres.WithDatabase("xyfamily_test"),
        postgres.WithUsername("test"),
        postgres.WithPassword("test"),
    )
    require.NoError(t, err)

    db, err := gorm.Open(postgres.Open(dbContainer.DSN(nil)))
    require.NoError(t, err)

    // 执行迁移
    autoMigrate(db)

    return db, func() { dbContainer.Terminate(t) }
}
```

### 3.2 测试 Redis

```go
func setupTestRedis(t *testing.T) (*redis.Client, func()) {
    redisContainer, err := generic.Run(t, "redis:7-alpine")
    require.NoError(t, err)

    client := redis.NewClient(&redis.Options{
        Addr:     redisContainer.Endpoint(t),
        Password: "",
        DB:       0,
    })

    return client, func() { redisContainer.Terminate(t) }
}
```

### 3.3 集成测试标签

```go
// 使用 build tag 区分集成测试
//go:build integration

func TestLoginFlow_Integration(t *testing.T) {
    // 测试完整注册 → 登录 → 登出 流程
}
```

执行：`go test -tags=integration ./...`

## 四、E2E 测试

### 4.1 关键用户路径

| 路径 | 步骤 | 优先级 |
|------|------|--------|
| 注册 → 登录 → 修改密码 → 登出 | 1. 发送验证码 2. 注册 3. 登录 4. 修改密码 5. 登出 | P0 |
| 创建组织 → 邀请成员 → 成员确认 | 1. 登录 2. 创建组织 3. 邀请 4. 被邀请方确认 | P0 |
| 组织 → 团队 → 小组 层级管理 | 1. 创建团队 2. 创建小组 3. 各层级成员管理 | P0 |
| 权限校验 → 越权拒绝 | 1. 高权限操作成功 2. 低权限操作返回 403 | P0 |
| Token 刷新 → Refresh Rotation | 1. Access Token 过期 2. 刷新 3. 旧 Token 失效 | P0 |
| 多租户隔离 → 跨 org 拒绝 | 1. org A 用户访问 org B 数据 → 403 | P0 |
| 账号注销 → 宽限期 → 恢复 | 1. 申请注销 2. 宽限期内恢复 3. 验证数据完整 | P1 |

### 4.2 测试工具

| 工具 | 用途 |
|------|------|
| Playwright | API E2E（Go 测试框架更契合） |
| curl + Go test | HTTP 请求级 E2E |
| k6 | 性能/压力测试 |

## 五、测试数据管理

### 5.1 测试数据准备

```go
// 使用 factory pattern 生成测试数据
func createTestAccount(db *gorm.DB, opts ...AccountOption) *Account {
    account := &Account{
        AccountID: uuid.New(),
        Phone:     "13800000001",
        Email:     "test@example.com",
        Nickname:  "测试用户",
        Status:    "active",
    }
    for _, opt := range opts {
        opt(account)
    }
    db.Create(account)
    return account
}

func WithPhone(phone string) AccountOption {
    return func(a *Account) { a.Phone = phone }
}
```

### 5.2 数据隔离

| 策略 | 说明 |
|------|------|
| 独立数据库 | 测试使用 `xyfamily_test` 数据库，与开发/生产隔离 |
| 事务回滚 | 每个测试用例后回滚 |
| 唯一标识 | 测试数据使用固定标识（如 `test_` 前缀） |

## 六、CI 集成

### 6.1 GitHub Actions 流水线

```yaml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:17
        env:
          POSTGRES_DB: xyfamily_test
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
        ports:
          - 5432:5432
      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.22'
      - run: go mod download
      - name: Unit Tests
        run: go test -v -cover ./...
      - name: Integration Tests
        run: go test -tags=integration -v ./...
        env:
          DB_HOST: localhost
          REDIS_ADDR: localhost:6379
      - name: Lint
        run: golangci-lint run
      - name: Vulnerability Check
        run: govulncheck ./...
```

### 6.2 质量门禁

| 门禁 | 要求 |
|------|------|
| 单元测试覆盖率 | ≥ 80% |
| 集成测试 | 全部通过 |
| Lint | 零错误 |
| 漏洞扫描 | 零高危 |
