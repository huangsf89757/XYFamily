## 一、测试用例模板

### 1.1 用例基本信息

| 字段 | 说明 |
|------|------|
| 用例 ID | TC-{模块缩写}-{序号}，如 TC-AUTH-001 |
| 用例标题 | 简洁描述被测功能 |
| 所属模块 | 认证 / 账号 / 组织 / 团队 / 小组 / 权限 / 审计 |
| 用例类型 | 功能测试 / 边界测试 / 异常测试 / 安全测试 / 性能测试 |
| 优先级 | P0（阻塞）/ P1（重要）/ P2（一般） |
| 前置条件 | 执行前需要的环境/数据状态 |
| 测试步骤 | 操作步骤 |
| 预期结果 | 预期返回 |
| 实际结果 | 执行后填写 |
| 通过/失败 | 最终判定 |

### 1.2 用例模板示例

```markdown
### TC-AUTH-001：用户注册 - 正常流程

**所属模块**：认证  
**用例类型**：功能测试  
**优先级**：P0  

**前置条件**：
1. 服务正常运行
2. Redis + PostgreSQL 可用
3. 手机号未被注册

**测试步骤**：
1. POST /api/v1/auth/send-code {"target": "13800138000", "type": "register"}
2. 获取返回的验证码（从 Redis 读取）
3. POST /api/v1/auth/register {"phone": "13800138000", "code": "<验证码>", "password": "Str0ng@Pass#2024", "nickname": "测试用户"}

**预期结果**：
- HTTP 201
- { "code": 0, "data": { "account_id": "<uuid>", "nickname": "测试用户" } }
- accounts 表新增记录
- Redis 验证码已消费

**测试数据**：
- 手机号：13800138000
- 密码：Str0ng@Pass#2024（≥12 位，含大小写+数字+特殊字符）
```

| 测试项 | 说明 | 优先级 |
|--------|------|--------|

## 五、Go 测试代码示例

### 5.1 表驱动测试

```go
func TestPasswordValidation(t *testing.T) {
    tests := []struct {
        name      string
        password  string
        wantValid bool
    }{
        {"valid", "Str0ng@Pass#2024", true},
        {"too_short", "Str0ng@P", false},
        {"no_special", "Str0ngPass12345", false},
        {"no_upper", "strong@pass#2024", false},
        {"no_lower", "STRONG@PASS#2024", false},
        {"no_digit", "Strong@Pass####", false},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := ValidatePassword(tt.password)
            assert.Equal(t, tt.wantValid, got)
        })
    }
}
```

func TestPermissionChecker_CacheHit(t *testing.T) {
    // Arrange
    ctx, _ := gin.CreateTestContext(httptest.NewRecorder())
