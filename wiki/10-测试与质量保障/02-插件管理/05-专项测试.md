## 1. 文档定位

本文件是「测试与质量保障」板块在 **Stage 2 插件管理** 下的专项测试分册，承接 [Stage 1 专项测试](../01-多租户底座/05-专项测试.md) 的通用专项，对插件管理 Stage 的专属质量项给出可执行测试方法。

## 2. 插件沙箱隔离专项（最高优先级）

### 2.1 测试目标

- **沙箱越权零泄露**：插件不得越出沙箱读取 / 写入宿主或其他租户数据。
- **资源隔离**：插件的计算 / 存储 / 网络资源严格隔离，逃逸应被拒绝。
- **钩子（Hook）安全**：插件钩子不得篡改宿主关键链路或越权触发其他插件逻辑。

### 2.2 隔离用例矩阵

| 用例类别 | 场景 | 预期结果 |
|----------|------|----------|
| 沙箱越权读 | 插件试图读取沙箱外 / 其他租户资源 | 拒绝（403 / 空结果） |
| 沙箱越权写 | 插件试图写入宿主存储 / 其他租户数据 | 拒绝并告警 |
| 资源隔离逃逸 | 插件突破 CPU / 内存 / 网络隔离 | 被沙箱拦截 |
| 钩子越权 | 插件钩子篡改宿主或其他插件状态 | 拒绝并记录 |
| 上下文透传 | 插件调用携带错误 / 伪造 X-Organization-ID | 回落或拒绝 |

## 3. 插件市场安全专项

### 3.1 测试目标

- **上架审核不可绕过**：插件上架 / 审核流程无法被绕过或伪造签名。
- **供应链安全**：插件依赖（第三方库）经 SCA 扫描，零 Critical/High。
- **恶意插件防护**：含恶意逻辑 / 后门 / 数据外传的插件在安装前被拦截。

### 3.2 安全用例矩阵

| 用例类别 | 场景 | 预期结果 |
|----------|------|----------|
| 审核绕过 | 伪造审核状态 / 跳过上架审核 | 拒绝 |
| 签名伪造 | 篡改插件包签名 | 安装拒绝 |
| 依赖投毒 | 插件携带高危依赖 | SCA 拦截 |
| 数据外传 | 插件运行时外传租户数据 | 沙箱 / 网关拦截 |
