## 1. 文档定位

本文件是「测试与质量保障」板块的专项测试分册，承接《测试策略 - 测试与质量保障》中"专项测试"类型，对以下三类非功能 / 跨切面质量项给出可执行测试方法：

1. 多租户隔离专项（最高优先级，RISK-003）
2. 性能压测专项（G3 卡点）
3. 安全渗透专项（G3 卡点）

## 2. 多租户隔离专项（RISK-003）

### 2.1 风险背景

| 项 | 内容 |
|----|------|
| 风险 ID | RISK-003 跨组织数据泄露 |
| 概率 / 影响 | 低 / 高 |
| 阶段 | Phase2 / 3 |
| 应对措施 | 强制 X-Organization-ID 上下文校验、隔离专项测试 |
| 状态 | 待开始 |

> 隔离机制设计详见《多租户隔离方案 - 核心域》；本文件只定义**测试验证方法**，不重复设计细节。

### 2.2 测试目标

- **跨组织数据零泄露**：任一组织身份不得读取 / 写入 / 推断其他组织的数据。
- **上下文强校验**：所有数据访问路径强制携带并校验 `X-Organization-ID`，缺失 / 伪造 / 越权上下文应被拒绝。
- **审计可观测**：隔离相关的越权访问尝试应进入审计日志（保留 1 年）。

### 2.3 隔离用例矩阵

| 用例类别 | 场景 | 预期结果 | 关联风险 |
|----------|------|----------|----------|
| 横向越权读 | 组织 A 用户以自身令牌请求组织 B 的资源 URI | 拒绝（403 / 空结果），无 B 数据返回 | RISK-003 |
| 上下文伪造 | 篡改 / 缺失 `X-Organization-ID` 发起请求 | 拒绝或回落到自身组织上下文 | RISK-003 |
| 数据交叉校验 | 同库多租户数据批量导出，校验无串租户行 | 导出结果仅含本组织行 | RISK-003 |
| 共享资源隔离 | 跨组织共享 / 协作场景下的可见性边界 | 仅显式授权范围内可见 | RISK-003 |
| 搜索 / 列表越界 | 全局搜索 / 列表接口跨组织检索 | 结果严格限定本组织 | RISK-003 |

### 2.4 数据校验方法

- **专项用例 + 数据校验**：构造 ≥ 2 个组织数据集，执行上述矩阵后，对落库 / 返回结果做字段级租户归属断言。
- **自动化回归**：隔离用例纳入 CI 回归，作为 G3 准出的"多租户隔离验证通过"证据（待完善：自动化接入方式）。

## 3. 性能压测专项（G3）

| 项 | 阈值（引自里程碑 §3） | 检测方式 |
|----|----------------------|----------|
| API 响应 | 95% < 100ms | 压测（k6 / JMeter） |
| 登录响应 | 95% < 200ms | 压测 |
| 吞吐 | ≥ 1000 req/s | 压测 |

- **场景库**（待完善）：核心读 / 写链路、登录链路、多租户混合负载。
- **基线达标**作为 G3 准出项"性能基线达标"的判定依据。

## 4. 安全渗透专项（G3）

| 项 | 标准（引自里程碑 §3） | 检测方式 |
|----|----------------------|----------|
| 安全渗透 | 无高危及以上风险 | 渗透测试 / 漏洞扫描 |

- 与静态扫描（SAST 零高危、依赖扫描零 Critical/High，G2）形成纵深；渗透聚焦运行时 / 业务逻辑漏洞。
- 多租户隔离的越权场景（§2）是渗透重点路径。
