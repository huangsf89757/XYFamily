## 1. 需求背景与目标

在统一身份、权限与多租户隔离基座之上，提供插件化业务工具接入框架，让第三方或内部团队以插件形式快速接入平台，复用 Stage 1 的 RBAC 权限、多租户隔离与审计能力，避免重复建设。

> 本 PRD 仅描述产品需求（What），不含接口定义、数据库表结构等实现细节（How）。详细内容待 Stage 1 上线稳定后细化。

## 2. 预期功能方向（草案，待细化）

| 方向 | 说明 | 复用底座能力 |
|------|------|--------------|
| 插件接入框架 | 插件注册 / 生命周期 / 权限声明 | RBAC、多租户隔离 |
| 插件市场 | 插件发现、安装、版本管理 | 组织 / 团队 / 小组结构 |
| 插件隔离 | 插件数据与主租户隔离 | org_id 隔离、审计 |
| 插件鉴权 | 插件调用统一鉴权 | JWT、权限校验中间件 |

## 3. 与底座衔接点

- 权限：插件权限点纳入 45 权限点矩阵扩展（FR-PERM 体系）。
- 隔离：插件数据随组织上下文隔离（NFR-SEC-006）。
- 审计：插件关键操作纳入审计日志（FR-AUDIT）。

## 4. 预期里程碑拆解（草案，待细化）

> 以下为 Stage 2 的方向性拆解草案，非最终计划；待 Stage 1 生产上线稳定、需求明确后，按 Stage-Gate 模式正式拆解为 Phase 与里程碑文档。

| 预期 Phase | 方向 | 复用底座能力 |
|-----------|------|--------------|
| Phase A 插件接入框架 | 插件注册 / 生命周期管理 / 权限点声明 | RBAC、多租户隔离 |
| Phase B 插件市场 | 插件发现、安装、版本与灰度 | 组织 / 团队 / 小组结构 |
| Phase C 插件隔离与鉴权 | 插件数据隔离、插件调用统一鉴权 | org_id 隔离、JWT、审计 |
