# 超级管理员模块

> 超级管理员（SuperAdmin）是系统最高权限角色，独立于组织架构之外。本模块定义 SuperAdmin 的初始化、系统配置管理、跨组织强制降级与全局审计日志查看的产品需求。本模块为 V1.1.0，在 V1.0.0 基础上细化拆分需求。

## 一、模块定位

超级管理员模块是系统的**最高权限管理入口**，仅 `super_admin` 角色可访问。它解决三类问题：

1. **「鸡生蛋」问题**：系统首次启动、数据库为空时，如何产生第一个拥有创建组织权限的 SuperAdmin。
2. **系统级兜底管理**：当各业务模块（组织 / 团队 / 小组）因「至少保留一名核心管理员」等自保护规则而无法完成某些操作时，由 SuperAdmin 越级强制处理。
3. **全局可观测**：跨组织查看全量审计日志，支撑安全审计与合规检查。

## 二、功能需求

### FR-ADMIN-001：超级管理员初始化（「鸡生蛋」问题）

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 系统首次启动、数据库为空时，产生第一个 SuperAdmin 账号 |
| **验收标准** | 数据库为空时可成功创建首个 SuperAdmin；已存在时拒绝重复创建 |

**详细规则：**

- **初始化触发条件**：仅当系统中不存在任何 SuperAdmin 账号时允许初始化；已存在则拒绝（幂等保护）。
- **生产环境方式**：通过初始化命令交互式创建，密码通过交互式输入，**不进入命令历史、不以命令行参数传递**，避免泄露。
- **开发 / 测试环境方式**：通过初始化配置文件快速创建，仅当数据库为空时执行一次；初始化完成后应清除该配置，避免留存被滥用。
- **强密码策略**：初始密码 ≥ 12 位，且同时包含大写字母、小写字母、数字、特殊字符。
- **强制修改**：创建后首次登录必须修改默认密码，否则仅允许执行「修改密码」动作，无法进入其它功能。
- **初始化审计**：初始化操作本身需记录审计日志（即便此时尚无可归属的业务主体），用于追溯首个账号来源。
- **多 SuperAdmin（延后 P2）**：首版仅支持单一 SuperAdmin；后续版本考虑支持多个 SuperAdmin 并明确相互制约关系。

### FR-ADMIN-002：超级管理员游离态与权限边界

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 明确 SuperAdmin 不隶属任何组织架构，权限直达、全局可用 |
| **验收标准** | SuperAdmin 在成员关系表中无记录，但可执行全部跨组织操作 |

**详细规则：**

- SuperAdmin **不属于**任何 组织 / 团队 / 小组，在成员关系表中没有成员记录。
- 其权限直接来源于 `super_admin` 角色（最高层级），**不依赖** membership 关系。
- 可跨组织执行所有操作，**无需先加入**目标组织。
- **边界例外**：创建组织时若未指定 `initial_admin`，默认将创建者（SuperAdmin）加入该组织并分配 `organization_core_admin` 角色，以避免组织创建后无人管理。此例外不改变 SuperAdmin 在其它组织中的游离态。
- SuperAdmin 不可被组织 / 团队 / 小组层级的管理员操作（如被移除、被降级）。

---

### FR-ADMIN-003：系统配置查看

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 查看系统级配置项及其当前取值 |
| **验收标准** | SuperAdmin 可查看全部系统配置项与当前值 |

**详细规则：**

- 仅 SuperAdmin 可访问系统配置查看。
- 可查看的配置项（详见 FR-ADMIN-004）：
  - 登录自动注册开关（【延后 P2】首版暂不启用）
  - Token 有效期（Access / Refresh）
  - 登录限流策略（失败阈值、统计窗口、锁定时长）
  - 验证码有效期
  - 审计日志保留期
  - 账号注销宽限期
- 查看操作本身不强制记录审计日志（修改才记录）。

---

### FR-ADMIN-004：系统配置修改

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 修改系统级配置项并即时生效 |
| **验收标准** | 修改后配置即时生效，且记录审计日志 |

**通用规则：**

- 仅 SuperAdmin 可修改。
- 修改**即时生效**，无需重启。
- 每次修改记录审计日志（`admin.config.update`），`details` 含修改前后值快照。
- 修改需通过**高危操作二次确认**（见 FR-ADMIN-007）。
- 输入校验：越界 / 非法取值应被拒绝并提示，不得静默落库。

**各配置项细化：**

| 配置项 | 规则与边界 |
|--------|------------|
| 登录自动注册开关 | 布尔值；首版写死为关闭且不对外暴露修改入口（P2 开放） |
| Token 有效期 | Access Token 默认 30 分钟、Refresh Token 默认 7 天（参考 NFR-SEC-004）；允许在合理区间内调整 |
| 登录限流策略 | 可配置失败阈值、统计窗口、锁定时长（参考 NFR-SEC-002） |
| 验证码有效期 | 可配置，默认 5 分钟（参考 NFR-SEC-003） |
| 审计日志保留期 | 可配置，默认 1 年（参考 NFR-SEC-007）；缩短需二次确认 |
| 账号注销宽限期 | 可配置；宽限期内可恢复，到期后执行真正注销 |

---

### FR-ADMIN-005：强制降级

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 将任意组织 / 团队 / 小组的管理员强制降级为普通成员 |
| **验收标准** | 降级后目标角色即时变更为 RegularMember，原范围权限立即回收 |

**详细规则：**

- 仅 SuperAdmin 可执行。
- **可降级目标**（任意一级管理员 → RegularMember）：
  - 组织核心管理员（`organization_core_admin`）
  - 组织普通管理员（`organization_ordinary_admin`）
  - 团队核心管理员 / 团队普通管理员
  - 小组核心管理员 / 小组普通管理员
- **突破自保护规则**：不受各业务模块「至少保留一名核心管理员」限制，可清空某组织 / 团队 / 小组的全部管理员。
- **不可降级对象**：SuperAdmin 自身、RegularMember（无管理员身份可降）。
- 支持**单个降级**与**批量降级**（如一次性清空某组织全部管理员）。
- 降级即时生效，目标对原管理范围的访问权限立即回收。
- 记录审计日志（`admin.force_downgrade`），`details` 含操作前角色快照（`before.role`）。

---

### FR-ADMIN-006：全局审计日志查看

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 跨组织查看全部登录审计与操作审计日志 |
| **验收标准** | 返回全局审计日志，支持筛选与分页 |

**详细规则：**

- 仅 SuperAdmin 可访问。
- 可查看**所有组织**的登录审计日志与操作审计日志。
- 筛选维度：时间范围、账号、操作类型、登录结果、组织维度（按组织过滤）。
- 支持分页与日志详情查看（含 `details` 前后快照）。
- 详见审计日志模块（FR-AUDIT-007 / FR-AUDIT-008）。

---

### FR-ADMIN-007：高危操作二次确认

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 对强制降级、保留期缩短等高危操作执行前二次确认 |
| **验收标准** | 二次确认未通过则操作中止，不产生任何变更 |

**详细规则：**

- 触发范围：强制降级（FR-ADMIN-005）、缩短审计日志保留期、关闭关键安全开关等高危操作。
- 确认方式：需显式确认风险告知（如勾选 + 输入确认），必要时要求再次输入密码。
- 二次确认仅校验本次操作意图，不延长会话有效期。

---

## 三、安全与约束

- **账号保护**：SuperAdmin 不可被组织 / 团队 / 小组层级的管理员移除或降级；首版不支持删除 SuperAdmin。
- **操作可审计**：SuperAdmin 的所有敏感操作（配置修改、强制降级、全局日志查看）均记录审计日志，且日志不可被 SuperAdmin 自身篡改或删除（见审计日志模块 FR-AUDIT-009）。
- **最小暴露**：SuperAdmin 初始化凭证仅用于创建首个账号，不应长期留存或复用。
- **权限直达**：SuperAdmin 的权限不经由任何 membership 派生，避免权限随成员关系变动而丢失。
