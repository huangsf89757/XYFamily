# 权限点校验

> 请求到达受保护接口时，应校验当前账号在目标数据范围内是否拥有该接口所要求的权限点。无权限应被拒绝。

## 一、功能需求

### FR-PERM-002：权限点校验

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 请求到达受保护接口时，应校验当前账号在目标数据范围内是否拥有该接口所要求的权限点 |
| **验收标准** | 通过校验则进入业务逻辑；无权限（角色缺失 / 数据范围不匹配 / 权限点不足）应被拒绝 |

---

## 二、业务规则

### 2.1 接口权限声明

- 每个受保护接口应声明其所需的**一个或一组**权限点（如创建组织接口要求 `org.create`）。
- 公开操作（仅需 `auth.register` / `auth.login` / `auth.reset_password`）不进入权限校验，由 Public 角色默认覆盖。

> 具体接口与权限点的对应关系在接口设计阶段确定，不在本 PRD 中定义。

### 2.2 校验规则

- 系统应确定请求所处的**数据范围**（组织 / 团队 / 小组或全局），并据此找到用户的**生效角色**。
- 系统应校验生效角色所拥有的权限点集合是否包含接口所需的全部权限点。
- 校验所需的成员关系与生效角色的确定，遵循 [数据范围控制](./03-数据范围控制.md) 中的层级回溯规则。
- 校验通过则进入业务逻辑；未通过则应拒绝请求。

### 2.3 校验性能

- 系统应对权限校验结果进行缓存以提升性能，并在角色分配、成员移除、管理员降级、权限矩阵变更等场景下使相关缓存失效。

### 2.4 与业务流程图的对应

完整流程图（身份认证 → 范围上下文确定 → 成员关系回溯 → 角色确定 → 权限判定）见 [权限管理模块](./权限管理模块.md) 与 [多租户底座](../多租户底座.md)。

## 三、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 身份无效 / 缺失 | 拒绝访问 |
| 数据范围上下文缺失或非法 | 拒绝访问 |
| 当前范围无成员关系且上级范围也无 | 拒绝访问 |
| 角色存在但权限点不足 | 拒绝访问 |
| 跨组织越权访问 | 拒绝访问 |
| 公开操作（注册 / 登录 / 重置密码） | 不进入权限校验，直接放行 |
| 角色 / 成员关系变更后缓存未失效 | 强制失效相关缓存，下次请求重新加载 |
