## 一、性能需求

| ID | 需求描述 | 指标 | 关联模块 |
|----|----------|------|----------|
| NFR-PERF-001 | API 响应时间 | 95% < 100ms，99% < 500ms（不含文件上传/下载） | 全局 |
| NFR-PERF-002 | 登录请求响应时间 | 95% < 200ms（含密码校验与 Token 生成，不含验证码发送） | 用户认证 |
| NFR-PERF-003 | 系统吞吐量 | 单实例 ≥1000 req/s，水平扩展后线性提升，支持 3 倍突发 | 全局 |
| NFR-PERF-004 | 并发在线用户 | 支持 ≥10000 用户同时在线 | 全局 |
| NFR-PERF-005 | 批量操作性能 | 单次批量邀请 100 人 < 3s；批量操作应支持异步与进度反馈 | 组织/团队/小组 |
| NFR-PERF-006 | 审计日志写入 | 异步写入，不阻塞主流程，写入可见延迟 P99 < 1s | 审计 |
| NFR-PERF-007 | 页面/资源加载 | 关键页面首屏 < 2s（弱网除外） | 全局 |
| NFR-PERF-008 | 高频读取 | 组织信息、角色权限等热点数据应提供缓存以缓解高频读取（命中率目标 ≥ 95%） | 权限/组织 |

## 二、安全需求

| ID | 需求描述 | 指标 | 关联模块 |
|----|----------|------|----------|
| NFR-SEC-001 | 密码存储 | bcrypt，cost factor 12；明文不落库不落日志 | 认证/账号 |
| NFR-SEC-002 | 登录限流 | 同一 IP 5 分钟内 5 次失败锁定 15 分钟；基于 IP + 账号双重维度 | 认证 |
| NFR-SEC-003 | 验证码安全 | 6 位数字，有效期 5 分钟，单码单场景绑定，限频限次 | 认证 |
| NFR-SEC-004 | Token 安全 | Access 30min / Refresh 7 天；不含敏感信息；登出/改密立即失效 | 认证 |
| NFR-SEC-005 | Token 签名 | 默认 HS256，预留 RS256；密钥建议 90 天轮换 | 认证 |
| NFR-SEC-006 | 数据隔离 | 组织间完全隔离；查询强制带组织上下文；跨组织仅 SuperAdmin | 全局 |
| NFR-SEC-007 | 审计日志保留 | 保留 1 年，账号注销后匿名化，仅追加不可篡改 | 审计 |
| NFR-SEC-008 | 传输安全 | 全站 HTTPS，HTTP 重定向，TLS ≥ 1.2，启用 HSTS | 全局 |
| NFR-SEC-009 | 敏感字段存储 | 手机号/邮箱等 PII 加密存储；与密码同样不落明文日志 | 账号 |
| NFR-SEC-010 | 防重放攻击 | 关键请求防重放（如时间戳/随机数校验），避免重复提交 | 认证/全局 |
| NFR-SEC-011 | 会话安全 | 支持多端登录管理；闲置超时自动登出；异地/新设备登录提醒 | 认证 |
| NFR-SEC-012 | 最小权限 | 默认无权限，显式授权；权限变更需经审计 | 权限 |
| NFR-SEC-013 | 安全响应头 | 启用 CSP、X-Content-Type-Options、X-Frame-Options 等防护头 | 全局 |

## 三、兼容性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-COMP-001 | API 版本 | 前缀 `/api/v1/`；升级保留旧版本 ≥ 6 个月过渡期 |
| NFR-COMP-002 | JSON 格式 | 请求/响应均为 JSON，`Content-Type: application/json`，UTF-8 |
| NFR-COMP-003 | 跨域支持 | 支持 CORS，来源可配置，允许携带凭证 |
| NFR-COMP-004 | 浏览器兼容 | Chrome / Edge / Firefox / Safari 最新两个大版本 |
| NFR-COMP-005 | 时区 | 服务端统一 UTC 存储，前端按用户时区展示 |
| NFR-COMP-006 | 移动端适配 | 管理后台响应式，移动端可用核心功能 |

## 四、可扩展性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-SCAL-001 | 水平扩展 | 服务无状态化，支持水平扩展与负载均衡，实例间无共享状态 |
| NFR-SCAL-002 | 数据库扩展 | 预留读写分离、分库分表等扩展规划 |
| NFR-SCAL-003 | 第三方认证扩展 | 预留 OAuth2.0 扩展点，支持快速接入新第三方登录 |
| NFR-SCAL-004 | 权限/审计扩展 | 权限点、审计操作类型可扩展注册，不修改核心逻辑 |
| NFR-SCAL-005 | 多租户规模 | 支持持续增长的租户数与用户数，容量可水平扩容 |

## 五、可用性 / 可靠性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-AVAIL-001 | 服务可用性 | 核心服务 SLA ≥ 99.9% |
| NFR-AVAIL-002 | 无单点故障 | 关键组件冗余部署，单节点故障不影响整体 |
| NFR-AVAIL-003 | 优雅降级 | 非核心依赖（如通知）故障时，核心认证/管理功能仍可用 |
| NFR-AVAIL-004 | 限流与熔断 | 对异常流量限流、对故障依赖熔断，保护系统稳定 |

## 七、数据备份与容灾需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-BACKUP-001 | 数据备份 | 定期全量 + 增量备份，备份保留 ≥ 30 天，可恢复 |
| NFR-BACKUP-002 | RTO / RPO | 恢复时间目标 RTO < 4h，数据丢失目标 RPO < 15min |
| NFR-BACKUP-003 | 异地容灾 | 关键数据异地备份，重大故障可切换恢复 |
