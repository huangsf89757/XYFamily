## 四、覆盖范围（移动端 · 底部 4 Tab + 详情 + 登录）

移动端定位为**轻量客户端**，面向普通成员，聚焦查看与轻操作（管理功能引导至 Web / 桌面端）。

**主框架**
- 底部 Tab Bar（4 Tab，49pt 含安全区）：首页 / 组织 / 消息（含未读红点 Badge）/ 我的
- 顶部导航栏：首页大标题模式（34pt） / 其余页面标准导航栏（返回 + 操作）
- 安全区适配：状态栏（50px）、底部 Tab Bar（83px 含 18px Home Indicator 区）

**页面**
1. 首页 `#/home` — 当前组织卡、数据概览（组织/成员/待办）、4 宫格快捷入口、待办列表
2. 组织 `#/org` — 我的组织（Inset Grouped 列表，头像 + 角色标签）+ 团队列表
3. 组织详情 `#/org-detail` — 头部信息卡、数据统计、信息组、操作组（邀请 / 退出）
4. 消息 `#/message` — 消息列表（头像 + 标题 + 摘要 + 未读圆点 / 数字 Badge）
5. 消息详情 `#/message-detail` — 消息正文 + 操作按钮
6. 我的 `#/profile` — 用户信息卡、账号组、显示组（深色模式开关）、退出登录

**全局态**
- 登录页（演示）：账号密码 + 第三方（Apple / 微信 / 邮箱），`#login` 浮层
- 明暗双主题：我的页「深色模式」开关实时切换（`[data-theme]` 变量重映射，OLED 纯黑背景）
- 底部 Action Sheet：首页「安全」、组织详情「更多 / 退出组织」演示原生半模态弹层

## 五、设计对齐

- **设计语言**：Pinguo（苹果风），单一主色 `#007AFF`
- **移动端规范**：安全区、44pt 最小触控目标、Inset Grouped 列表、12pt 圆角、卡片阴影
- **角色标签色板**：超级管理员紫 `#5856D6`、组织管理员蓝 `#007AFF`、团队管理员绿 `#34C759`、普通成员灰 `#AEAEB2`、访客浅灰 —— 与全局规范一致
- **状态标签**：活跃（绿）/ 停用（灰）/ 待审核（橙）—— 状态色仅用于真实语义
- **克制**：无装饰性多色，列表与卡片依赖留白与阴影建立层级

| 维度 | 结论 |

| 哲学 | PASS · 内容优先、克制用色 |

## 七、已知简化（演示边界）

- 管理类操作（成员编辑、权限配置、审计日志）按规范仅在 Web / 桌面端提供，移动端以提示引导；
- 登录、深色模式、Action Sheet、退出登录为前端交互演示，无真实后端；
- 下拉刷新、上拉加载、滑动操作（Swipe）、边缘返回手势为规范示意，未做完整手势实现。
