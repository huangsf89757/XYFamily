## 一、设计概述

### 1.1 Web 端定位

XYFamily Web 端定位为**管理后台**，面向组织管理员、团队管理员及超级管理员，提供组织管理、成员管理、权限配置、审计日志、系统配置等全功能管理能力。Web 端是 XYFamily 多端体系中功能最完整的管理入口，不面向终端消费者用户。

| 维度 | 说明 |
|------|------|
| 产品形态 | SaaS 多租户管理后台（B 端管理工具） |
| 目标用户 | SuperAdmin、OrgAdmin、TeamAdmin、GroupAdmin |
| 核心场景 | 组织/团队/小组管理、成员管理、角色权限配置、审计日志查询、系统全局配置 |
| 设计风格 | Pinguo 设计语言（Apple 风格，克制、优雅、内容优先） |
| 设计基准 | 桌面端优先（Desktop First），最小支持宽度 1024px |

### 1.2 技术栈

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 框架 | React | 19+ |
| 构建工具 | Vite | 6+ |
| 样式方案 | Tailwind CSS | 4+ |
| 组件库 | shadcn/ui | latest |
| 图标库 | Lucide Icons | 1.8.0+ |
| 字体 | DM Sans（sans）、JetBrains Mono（mono） | CDN 加载 |
| 状态管理 | Zustand | 5+ |
| 路由 | React Router | 7+ |
| 请求库 | Axios / TanStack Query | latest |
| 表单 | React Hook Form + Zod | latest |

### 1.3 浏览器兼容性

| 浏览器 | 最低支持版本 | 说明 |
|--------|-------------|------|
| Chrome | 最新两个大版本 | 主力测试浏览器 |
| Edge | 最新两个大版本 | 基于 Chromium，兼容性与 Chrome 一致 |
| Firefox | 最新两个大版本 | 需验证 CSS 兼容性 |
| Safari | 最新两个大版本 | 需验证字体渲染、日期组件等差异 |

> **注意**：不支持 IE 浏览器及上述浏览器的过旧版本。最低屏幕分辨率要求为 1280×720，推荐 1440×900 及以上。

### 1.4 设计原则

| 原则 | 说明 |
|------|------|
| 内容优先 | 界面为内容服务，减少装饰性元素，让管理员聚焦核心管理操作 |
| 信息密度适中 | 管理后台需在信息密度与视觉舒适度间取得平衡，避免过度留白导致频繁滚动 |
| 操作可逆 | 关键操作提供撤销或二次确认，降低误操作风险 |
| 状态可见 | 所有异步操作需有明确的加载、成功、失败反馈 |
| 权限感知 | 界面元素根据当前用户角色动态显示/隐藏，无权限的入口不展示 |

## 二、响应式断点

### 2.1 断点定义

Web 端采用 Tailwind CSS 默认断点体系：

| 断点 | 宽度阈值 | Tailwind 前缀 | 典型设备 |
|------|----------|--------------|----------|
| 默认 | < 640px | 无 | 手机竖屏（仅适配，非主要场景） |
| sm | ≥ 640px | `sm:` | 手机横屏 / 小平板 |
| md | ≥ 768px | `md:` | 平板竖屏 |
| lg | ≥ 1024px | `lg:` | 平板横屏 / 小笔记本 |
| xl | ≥ 1280px | `xl:` | 标准桌面显示器 |
| 2xl | ≥ 1536px | `2xl:` | 大屏显示器 |

### 2.2 各断点布局策略

| 断点 | 侧边栏 | 内容区 | 表格 | 顶部栏 |
|------|--------|--------|------|--------|
| < sm（< 640px） | 隐藏，通过汉堡菜单抽屉展开 | 全宽，内边距 16px | 水平滚动 | 简化，仅显示汉堡菜单 + 标题 |
| sm ~ md（640~767px） | 隐藏，通过汉堡菜单抽屉展开 | 全宽，内边距 24px | 水平滚动 | 显示汉堡菜单 + 面包屑 |
| md ~ lg（768~1023px） | 收起为图标模式（64px） | 内容区左移，内边距 24px | 水平滚动 | 显示面包屑 + 用户菜单 |
| lg ~ xl（1024~1279px） | 展开模式（256px） | max-width 1408px，内边距 32px | 正常显示 | 完整顶部栏 |
| xl ~ 2xl（1280~1535px） | 展开模式（256px） | max-width 1408px，内边距 32px | 正常显示 | 完整顶部栏 |
| ≥ 2xl（≥ 1536px） | 展开模式（256px） | max-width 1408px，居中 | 正常显示 | 完整顶部栏 |

### 2.3 断点使用规则

| 规则 | 说明 |
|------|------|
| Desktop First | 样式默认按桌面端编写，通过 `md:` / `lg:` 前缀向下适配 |
| 最小宽度限制 | `lg`（1024px）为功能完整显示的最低断点，低于此宽度进入适配模式 |
| 组件级响应式 | 复杂组件（表格、表单）在组件内部通过断点前缀调整布局，而非整页切换 |
| 不使用容器查询 | 统一使用视口断点，保持与 Tailwind 默认体系一致 |

## 三、布局系统

### 3.1 整体布局结构

Web 管理后台采用**左侧边栏 + 右侧内容区**的经典管理后台布局，参考现有 `admin-shell.html` 原型：

```
+----------------------------------------------------------+
|  侧边栏  |              顶部栏（sticky）                   |
|         +-------------------------------------------------+
|  256px  |                                                 |
|  或     |              内容区（main）                      |
|  64px   |              max-width: 1408px                  |
|         |              padding: 32px                      |
|         |                                                 |
|         |                                                 |
+---------+-------------------------------------------------+
```

| 区域 | 说明 |
|------|------|
| 侧边栏 | 固定定位（`position: fixed`），宽度 256px（展开）/ 64px（收起），左侧贴边 |
| 顶部栏 | sticky 定位，高度 56px（h-14），横跨内容区宽度 |
| 内容区 | `margin-left` 随侧边栏宽度变化，`max-width: 1408px`，左右内边距 32px |
| 页面背景 | 内容区背景色 `--background-200`（#F2F2F7） |

### 3.2 侧边栏

#### 3.2.1 尺寸规格

| 属性 | 展开模式 | 收起模式 |
|------|----------|----------|
| 宽度 | 256px | 64px |
| 背景色 | `--sidebar`（Light: #F2F2F7 / Dark: #1C1C1E） | 同展开模式 |
| 右边框 | 1px solid `--sidebar-border` | 同展开模式 |
| 定位 | `position: fixed; top: 0; left: 0; bottom: 0` | 同展开模式 |
| z-index | 50 | 50 |
| 过渡动画 | 宽度过渡 400ms `ease-in-out` | 同展开模式 |

#### 3.2.2 侧边栏结构

| 区域 | 高度 | 说明 |
|------|------|------|
| 品牌区 | 64px | Logo 图标 32×32px（`--brand-500` 背景，圆角 `--brand-radius-sm`）+ "XYFamily" 文字（18px bold） |
| 导航区 | flex-1，可滚动 | 多级菜单分组，详见[导航体系](#四导航体系) |
| 用户区 | 自适应 | 底部固定，头像 + 用户名 + 角色信息，上边框分隔 |

#### 3.2.3 折叠交互

| 交互 | 说明 |
|------|------|
| 触发方式 | 顶部栏左侧折叠按钮（ChevronLeft/ChevronRight 图标） |
| 折叠时 | 仅显示导航图标，隐藏文字标签和分组标题 |
| Hover 展开 | 收起模式下，鼠标悬停在侧边栏时可临时展开显示文字（可选） |
| 持久化 | 折叠状态存储到 localStorage，刷新后保持 |
| 移动端 | 低于 `lg` 断点时自动收起为图标模式；低于 `md` 断点时隐藏，通过汉堡菜单抽屉展开 |

### 3.3 内容区

| 属性 | 值 |
|------|-----|
| max-width | 1408px |
| 左右内边距 | 32px（`px-8`） |
| 上下内边距 | 32px（`py-8`） |
| 背景色 | `--background-200` |
| margin-left | 256px（侧边栏展开）/ 64px（侧边栏收起） |
| 过渡动画 | margin-left 过渡 400ms `ease-in-out` |

### 3.4 认证页面布局

登录、注册、重置密码等认证页面不使用侧边栏布局，采用**居中卡片布局**：

| 属性 | 值 |
|------|-----|
| 页面背景 | `--background-200` |
| 布局方式 | `display: flex; align-items: center; justify-content: center; min-height: 100vh` |
| 卡片宽度 | 420px（登录）/ 480px（注册、重置密码） |
| 卡片内边距 | 40px（`p-10`） |
| 卡片圆角 | 20px |
| 卡片阴影 | `--shadow-xl` |
| 卡片背景 | `--background-50`（#FFFFFF） |
| 水平外边距 | 左右各 16px（`px-4`），防止小屏溢出 |

### 3.5 栅格系统

内容区内部使用 12 列栅格（Tailwind CSS Grid），间距通过 `gap` 控制：

| 场景 | 栅格配置 | 间距 |
|------|----------|------|
| 统计卡片行 | `grid-cols-4`（4 列等宽） | 24px（`gap-6`） |
| 双栏表单 | `grid-cols-2` | 24px（`gap-6`） |
| 详情+操作 | `grid-cols-3`（2:1 或 1:2） | 24px（`gap-6`） |
| 卡片列表 | `grid-cols-3` 或 `grid-cols-2` | 24px（`gap-6`） |

> 响应式栅格在 `lg` 以下断点自动降列：`grid-cols-4` → `md:grid-cols-2` → `grid-cols-1`。

## 四、导航体系

### 4.1 侧边栏导航

#### 4.1.1 导航分组结构

侧边栏导航按功能域分组，每组以大写标题分隔：

| 分组 | 导航项 | 说明 |
|------|--------|------|
| 个人设置 | 个人信息、密码与安全、账号设置 | 当前用户个人相关 |
| 组织管理 | 组织详情、成员管理、角色权限 | 当前组织相关 |
| 当前团队 | 团队详情、成员管理、小组列表（动态显示） | 进入团队上下文后显示 |
| 系统管理 | 权限管理、审计日志 | 系统级管理 |
| 超级管理员 | 系统配置、强制降级、全局审计 | 仅 SuperAdmin 可见 |

#### 4.1.2 导航项规格

| 属性 | 默认态 | 选中态 | Hover 态 |
|------|--------|--------|----------|
| 高度 | 40px（`h-10`） | 40px | 40px |
| 左内边距 | 20px（`px-5`） | 20px | 20px |
| 字号 | 14px（`text-sm`） | 14px | 14px |
| 字重 | 400（normal） | 600（semibold） | 400 |
| 文字色 | `--text-500`（#6E6E73） | `--brand-500`（#007AFF） | `--text-500` |
| 背景色 | 透明 | `--brand-50`（#E8F2FF） | `rgba(0,0,0,0.04)` |
| 左边框 | 3px transparent | 3px solid `--brand-500` | 3px transparent |
| 图标尺寸 | 16px（`w-4 h-4`） | 16px | 16px |
| 图标透明度 | 0.7 | 1.0（不透明） | 0.7 |
| 图标右边距 | 12px（`mr-3`） | 12px | 12px |
| 过渡 | 150ms `ease-default` | — | 150ms `ease-default` |

#### 4.1.3 分组标题规格

| 属性 | 值 |
|------|-----|
| 字号 | 12px（`text-xs`） |
| 字重 | 600（semibold） |
| 文字色 | `--text-400`（#8E8E93） |
| 字母大写 | `text-transform: uppercase` |
| 字间距 | `tracking-wide` |
| 内边距 | 左 20px，上 16px，下 4px |

#### 4.1.4 多级菜单

| 层级 | 缩进 | 说明 |
|------|------|------|
| 一级 | 0 | 标准导航项 |
| 二级 | 32px（`pl-8`） | 父级上下文路径显示（如组织名 > 团队名 > 导航项） |
| 上下文路径 | 32px（`pl-8`） | 非链接文字，仅显示层级关系，字号 13px，色 `--text-400` |

#### 4.1.5 用户区

| 属性 | 值 |
|------|-----|
| 位置 | 侧边栏底部，上边框分隔 |
| 内边距 | 20px 水平，16px 垂直 |
| 头像 | 36×36px（`w-9 h-9`），圆形，背景 `--brand-50`，文字 `--brand-500` |
| 用户名 | 14px，medium，`--text-800` |
| 角色信息 | 12px，`--text-400` |
| 布局 | 头像 + 信息纵向排列，间距 12px |

### 4.2 顶部栏

#### 4.2.1 顶部栏规格

| 属性 | 值 |
|------|-----|
| 高度 | 56px（`h-14`） |
| 背景色 | `--background-50`（#FFFFFF） |
| 下边框 | 1px solid `--background-300` |
| 内边距 | 左右 32px（`px-8`） |

#### 4.2.2 顶部栏内容

| 区域 | 位置 | 内容 |

### 4.3 面包屑

#### 4.3.1 面包屑规格

| 属性 | 值 |
|------|-----|
| 字号 | 14px（`text-sm`） |
| 分隔符 | `>` 或 ChevronRight 图标，色 `--text-400` |
| 间距 | 元素间 8px（`gap-2`） |
| 链接色 | `--brand-500`，hover 下划线 |
| 当前页色 | `--text-800`，font-medium，不可点击 |

#### 4.3.2 面包屑规则

| 规则 | 说明 |
|------|------|
| 起始项 | 固定为"首页"，链接到 `/` |
| 中间项 | 可点击链接，对应上级页面路由 |
| 末项 | 当前页面标题，纯文字不可点击 |
| 最大层级 | 不超过 4 级，超出时中间用 `...` 折叠 |
| 动态内容 | 组织名、团队名等动态内容直接显示实际名称 |

**面包屑示例**：

```
首页 > 组织管理 > 示例科技有限公司
首页 > 组织管理 > 成员管理
首页 > 组织管理 > 示例科技有限公司 > 产品研发部 > 团队详情
首页 > 超级管理员 > 系统配置
```

## 五、组件规范

> 所有组件仅消费全局设计令牌（语义令牌），禁止硬编码色值。色彩、字体、间距、圆角、阴影等令牌定义详见《全局设计规范》。

### 5.1 按钮（Button）

#### 5.1.1 按钮变体

| 变体 | 背景色 | 文字色 | 边框 | 用途 |
|------|--------|--------|------|------|
| Primary | `--primary`（#007AFF） | `--primary-foreground`（#FFFFFF） | 无 | 主操作（提交、保存、创建） |
| Secondary | `--secondary`（#F2F2F7） | `--secondary-foreground`（#1D1D1F） | 无 | 次要操作（取消、返回） |
| Outline | 透明 | `--foreground`（#1D1D1F） | 1px solid `--border` | 次要操作（筛选、导出） |
| Ghost | 透明 | `--foreground` | 无 | 工具栏操作、行内操作 |
| Destructive | `--destructive`（#FF3B30） | `--destructive-foreground`（#FFFFFF） | 无 | 危险操作（删除、移除） |
| Link | 透明 | `--primary` | 无 | 文字链接 |

#### 5.1.2 按钮尺寸

| 尺寸 | 高度 | 水平内边距 | 字号 | 字重 | 圆角 | 图标尺寸 |
|------|------|-----------|------|------|------|----------|
| sm | 32px（`h-8`） | 12px（`px-3`） | 13px | 500 | `--brand-radius-sm`（8px） | 14px |
| md | 40px（`h-10`） | 20px（`px-5`） | 14px | 500 | `--brand-radius-sm`（8px）或 full | 16px |
| lg | 48px（`h-12`） | 24px（`px-6`） | 14px | 600 | 12px（`rounded-xl`） | 16px |

#### 5.1.3 按钮状态

| 状态 | 样式变化 | 说明 |
|------|----------|------|
| Default | 基础样式 | 默认状态 |
| Hover | Primary: `--brand-600`；Secondary: `--background-300`；Ghost: `rgba(0,0,0,0.04)` | 背景色加深 |
| Active/Pressed | Primary: `--brand-700` | 背景色再加深 |
| Focused | 添加 `box-shadow: 0 0 0 2px var(--ring)` | 焦点环 |
| Disabled | `opacity: 0.5; cursor: not-allowed` | 禁用态 |
| Loading | 文字替换为 Spinner + "处理中..."，按钮禁用 | 加载态 |

#### 5.1.4 带图标按钮

| 类型 | 布局 | 图标位置 |
|------|------|----------|
| 左图标 | `[Icon] Text` | 图标在文字左侧，间距 8px |
| 右图标 | `Text [Icon]` | 图标在文字右侧，间距 8px |
| 仅图标 | `[Icon]` | 无文字，正方形按钮，需 `aria-label` |

### 5.2 输入框（Input）

#### 5.2.1 输入框类型

| 类型 | HTML type | 用途 |
|------|-----------|------|
| 文本 | `text` | 通用文本输入 |
| 密码 | `password` | 密码输入，带显示/隐藏切换 |
| 邮箱 | `email` | 邮箱输入，带格式校验 |
| 数字 | `number` | 数字输入 |
| 搜索 | `search` | 搜索框，带搜索图标前缀 |
| 选择 | `select` | 下拉选择 |
| 多行文本 | `textarea` | 长文本输入 |

#### 5.2.2 输入框尺寸

## 七、交互规范

### 7.1 加载策略

| 场景 | 加载方式 | 说明 |
|------|----------|------|
| 页面首次加载 | 骨架屏 | 模拟页面结构，减少布局跳动 |
| 表格数据加载 | 骨架屏 | 显示表头 + 骨架行，5~8 行 |
| 卡片内容加载 | 骨架屏 | 卡片形状骨架 + `animate-pulse` |
| 按钮提交 | Spinner + 文字 | 按钮内显示 Spinner + "处理中..."，按钮禁用 |
| 局部刷新 | Spinner | 区域内居中 Spinner，20px |
| 下拉/Popover | 即时 | 预加载数据，无加载态 |
| 全屏切换 | 顶部进度条 | 顶部 2px 进度条动画（路由切换时） |

**优先级**：骨架屏 > Spinner > 文字提示。首屏和主要内容区必须使用骨架屏，避免白屏。

### 7.2 表单提交

| 规则 | 说明 |
|------|------|
| 防重复提交 | 提交后立即禁用按钮 + Loading 态，响应前不可再次点击 |
| Loading 态 | 按钮文字变为"处理中..."，左侧显示 Spinner（16px） |
| 成功反馈 | Toast success 通知 + 跳转或停留在当前页 |
| 失败反馈 | Toast error 通知 + 表单内联错误（如有字段级错误） |
| 数据保持 | 提交失败时表单数据不丢失，用户可修改后重试 |
| 离开确认 | 表单有未保存修改时，离开页面弹出确认对话框 |
| 自动保存 | 长表单（如权限配置）支持自动保存，间隔 30s |

### 7.3 数据删除

| 规则 | 说明 |
|------|------|
| 二次确认 | 所有删除操作必须弹出确认对话框（Dialog sm 尺寸） |
| 确认对话框内容 | 标题 + 危险描述 + 确认按钮（Destructive）+ 取消按钮（Secondary） |
| 确认按钮文字 | "确认删除" / "确认移除"，明确操作语义 |
| 输入确认 | 高危操作（如删除组织）需输入组织名称确认 |
| 软删除 | 数据默认软删除，审计日志保留操作记录 |
| 成功反馈 | Toast success "删除成功" |
| 撤销（可选） | 删除后 Toast 提供"撤销"按钮，5 秒内可恢复 |

### 7.4 批量操作

| 规则 | 说明 |
|------|------|
| 选择方式 | 表格首列 Checkbox，支持全选/反选 |
| 选择栏 | 选中行后顶部显示操作栏，显示选中数量 + 批量操作按钮 |
| 批量操作 | 批量删除、批量启用/禁用、批量角色变更 |
| 二次确认 | 批量删除需二次确认，显示影响数量 |
| 操作栏位置 | 替换或叠加在筛选栏位置，`sticky top` 跟随滚动 |
| 清除选择 | 操作完成后清除选中状态 |
| 跨页选择 | 支持跨页选择，显示"已选择 N 项"，提供"选择全部"操作 |

### 7.5 搜索与筛选

| 类型 | 触发方式 | 防抖 | 适用场景 |
|------|----------|------|----------|
| 实时搜索 | 输入即触发 | 300ms 防抖 | 成员搜索、快速过滤 |
| 提交搜索 | 按回车或点击搜索按钮 | 无 | 审计日志、复杂条件搜索 |
| 筛选下拉 | 选择即触发 | 无 | 角色筛选、状态筛选 |
| 日期范围 | 选择后自动触发 | 无 | 审计日志时间范围 |

| 规则 | 说明 |
|------|------|
| 搜索框 | 左侧带搜索图标（16px），placeholder "搜索姓名、手机号、邮箱" |
| 清除搜索 | 搜索框右侧显示清除按钮（有内容时） |
| 筛选状态 | 已激活的筛选项在筛选栏下方显示 Tag，可单独清除 |
| 重置筛选 | 提供"重置"按钮，清除所有筛选条件 |
| URL 同步 | 搜索和筛选状态同步到 URL Query，支持分享链接和浏览器前进/后退 |
| 空结果 | 搜索无结果时显示空状态 + "清除筛选"按钮 |
