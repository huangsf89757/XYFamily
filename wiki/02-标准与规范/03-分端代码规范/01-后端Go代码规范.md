## 一、项目包结构约定

本项目采用分层架构，包名遵循 Go 官方小写、短小的惯例：

```
internal/
├── handler/        # HTTP 处理层（按模块分子包：handler/auth、handler/org）
├── service/        # 业务逻辑层（按模块分子包：service/org、service/team）
├── repository/     # 数据访问层（按存储分子包：repository/pg、repository/redis）
└── pkg/            # 内部公共库（pkg/errors、pkg/middleware）
```

## 二、数据库操作约定

### 2.1 ORM 选型

本项目使用 **GORM** 操作数据库，禁止直接使用 `database/sql` 原生操作。

### 2.2 事务边界

- 事务边界在 **Service 层**，不在 Repository 层
- 一个业务操作对应一个事务，不跨 Service 层
- 事务内操作尽量简洁

```go
// 正确：Service 层管理事务
func (s *OrgService) InviteMember(ctx context.Context, orgID string, req *InviteMemberReq) error {
    return tx.WithTransaction(ctx, func(txn *gorm.DB) error {
        err := s.repo.CreateInvitation(txn, &invitation)
        if err != nil {
            return err
        }
        return s.auditRepo.Log(txn, &auditLog)
    })
}
```

### 2.3 连接池参数

```go
sqlDB.SetMaxOpenConns(100)       // 最大连接数
sqlDB.SetMaxIdleConns(10)        // 最小空闲连接
sqlDB.SetConnMaxLifetime(time.Hour) // 连接最大存活时间
```

## 三、测试约定

### 3.1 覆盖率要求

- **最小覆盖率：80%**
- Service 层和 Repository 层必须覆盖
- Handler 层覆盖关键路径

### 3.2 测试文件位置

- 测试文件与源代码同目录，命名 `xxx_test.go`
