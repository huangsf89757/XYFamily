## 一、功能需求

### FR-GROUP-001：获取小组信息

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 查看小组的基本信息 |
| **验收标准** | 小组成员可正常查看，非成员禁止访问 |

**详细规则：**
- 小组成员（含 小组核心管理员（`group_core_admin`）、小组普通管理员（`group_ordinary_admin`） 和 RegularMember）均可查看所属小组信息
- 非小组成员尝试查看时禁止访问，确保数据隔离
- 返回信息应包含：小组唯一标识、名称、描述、所属团队、所属组织、当前成员数量、状态、创建时间、更新时间
- 对于已删除状态的小组，仅超级管理员可查看，其他角色视为不存在

---

### FR-GROUP-002：更新小组信息

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 修改小组的名称和描述信息 |
| **验收标准** | 修改成功后返回更新后的完整信息 |

**详细规则：**
- 操作权限：仅 小组核心管理员（`group_core_admin`） 或 团队核心管理员（`team_core_admin`） 可执行
- 可修改字段：
  - 小组名称：2-50 个字符，同一团队内名称不可重复
  - 小组描述：0-500 个字符
- 校验规则：
  - 至少提供一个可修改字段
  - 名称修改时需校验团队内唯一性（排除已删除的小组）
  - 名称仅支持中英文、数字、空格、下划线、连字符
- 修改成功后更新时间自动更新
- 修改操作记录审计日志

---

### FR-GROUP-003：删除小组

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 软删除小组，保留历史数据 |
| **验收标准** | 删除后小组状态变更，成员关系解除，数据保留可审计 |

**详细规则：**
- 操作权限：仅 小组核心管理员（`group_core_admin`） 或 团队核心管理员（`team_core_admin`） 可执行
- 删除策略为软删除：
  - 小组状态更新为已删除
  - 记录删除时间
  - 小组数据（名称、描述等）保留，不物理删除
- 删除后级联处理：
  - 该小组的所有成员关系同步解除（软删除）
  - 成员的历史角色信息保留，用于审计追溯
  - 不级联影响上级团队或组织
- 删除后影响：
  - 小组成员无法再查看该小组信息
  - 已删除小组的成员列表不可查询
  - 超级管理员可在全局审计日志中查看已删除小组的历史记录
- 删除操作记录审计日志
- 不可重复删除已删除的小组

## 二、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 非小组成员查看小组信息 | 禁止访问 |
| 小组普通管理员或普通成员尝试修改/删除小组 | 禁止访问 |
| 小组名称在团队内重复 | 提示名称已存在，拒绝修改 |
| 小组名称格式非法 | 提示名称格式要求，拒绝修改 |
| 更新时未提供任何可修改字段 | 提示至少提供一个字段，拒绝提交 |
| 删除已删除的小组 | 视为不存在 |
| 删除不存在的小组 | 视为不存在 |
