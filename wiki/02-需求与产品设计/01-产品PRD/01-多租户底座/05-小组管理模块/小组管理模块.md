## 一、模块定位

小组（Group）是组织架构中的**最底层管理单元**，隶属于团队（Team）。一个团队下可创建多个小组，每个小组可独立管理成员和权限。小组管理模块面向 小组核心管理员（`group_core_admin`）、小组普通管理员（`group_ordinary_admin`） 和 团队核心管理员（`team_core_admin`） 提供管理能力，面向 RegularMember 提供信息查看能力。

### 1.1 组织层级关系

```
组织 (Organization)
  └── 团队 (Team)
       └── 小组 (Group)  ← 本模块
            └── 成员 (Member)
```

### 1.2 管理边界

| 边界 | 说明 |
|------|------|
| 数据隔离 | 小组数据仅对组织内成员可见，跨组织不可访问 |
| 管理权限 | 小组核心管理员（`group_core_admin`） 与 小组普通管理员（`group_ordinary_admin`） 管理本小组（核心管理员另含结构性 / 管理性操作）；团队核心管理员（`team_core_admin`） 继承管理权限；SuperAdmin 可跨组织管理 |
| 成员来源 | 被邀请人需先为组织成员，或同时被邀请加入组织 |
| 角色范围 | 小组内支持 小组核心管理员（`group_core_admin`）、小组普通管理员（`group_ordinary_admin`）、RegularMember 三种角色 |

## 四、业务流程

### 4.1 小组生命周期

```
创建（由 团队核心管理员（`team_core_admin`） 在团队中创建）
    │
    ▼
正常运营（成员邀请、角色分配、信息维护）
    │
    ├── 删除（软删除，保留历史数据）
    │
    └── 归档（如有需要，由团队层面控制）
```

### 4.2 成员邀请-确认流程

```
管理员发起邀请
    │
    ├── 被邀请人已注册 → 发送邀请通知（含邀请链接）
    │                       │
    │                       ├── 接受 → 加入小组，分配指定角色
    │                       │
    │                       └── 拒绝 → 邀请关闭
    │
    └── 被邀请人未注册 → 发送邀请通知
                            │
                            ├── 点击链接注册 → 注册成功后自动加入小组
                            │
                            └── 忽略 → 邀请过期（7天）
```

## 五、小组角色体系

小组内支持三种角色，由 小组核心管理员（`group_core_admin`） 或 团队核心管理员（`team_core_admin`） 分配：

| 角色 | 说明 | 管理范围 |
|------|------|----------|
| 小组核心管理员（`group_core_admin`） | 小组核心管理员 | 管理本小组的信息、成员、角色，含结构性 / 管理性操作（修改 / 删除小组、分配 / 降级角色） |
| 小组普通管理员（`group_ordinary_admin`） | 小组普通管理员 | 负责小组成员管理（邀请 / 移除 / 列表）与资源读取，不可做结构性操作 |
| RegularMember | 普通成员 | 查看小组信息和成员列表 |

### 5.1 角色权限矩阵

| 权限 | 小组核心管理员（`group_core_admin`） | 小组普通管理员（`group_ordinary_admin`） | RegularMember |
|------|:--:|:--:|:--:|
| 查看小组信息 | ✅ | ✅ | ✅ |
| 更新小组信息 | ✅ | ❌ | ❌ |
| 删除小组 | ✅ | ❌ | ❌ |
| 邀请小组成员 | ✅ | ✅ | ❌ |
| 移除小组成员 | ✅ | ✅ | ❌ |
| 分配小组角色 | ✅ | ❌ | ❌ |
| 降级小组管理员 | ✅ | ❌ | ❌ |

> **团队核心管理员（`team_core_admin`） 权限继承**：团队核心管理员（`team_core_admin`） 自动拥有其下属所有小组的 小组核心管理员（`group_core_admin`） 权限，无需显式加入小组。
