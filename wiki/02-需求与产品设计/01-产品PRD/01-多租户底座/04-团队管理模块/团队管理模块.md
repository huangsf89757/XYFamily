## 一、模块定位与边界

### 1.1 模块定位

团队是组织架构中的**中间管理层**，隶属于组织，下辖小组。团队管理模块负责团队全生命周期管理，包括信息维护、成员管理、角色分配、小组创建和团队归档。

**组织层级关系：**

```
组织 (Organization)
  └── 团队 (Team)  ← 本模块
       └── 小组 (Group)
            └── 成员 (Member)
```

### 1.2 模块边界

| 属于本模块 | 不属于本模块（其余模块负责） |
|------------|------------------------------|
| 团队信息维护（查看 / 更新 / 归档） | 团队的创建入口（见 [组织管理模块 - 创建团队](../03-组织管理模块/04-团队创建.md)） |
| 团队成员邀请 / 移除 / 列表 | 组织、小组成员管理（组织管理 / 小组管理模块） |
| 团队角色分配（核心 / 普通管理员）、核心管理员降级 | 细粒度权限点管理（权限管理模块） |
| 团队下创建小组入口与接棒机制 | 小组的完整管理（小组管理模块） |
| 团队关键操作的审计日志 | 审计日志存储与查询（审计日志模块） |

> 团队管理员角色（团队核心 / 普通管理员）由权限管理模块统一承载；小组核心 / 普通管理员角色仅在小组层面分配（见 [小组管理模块](../05-小组管理模块/小组管理模块.md)）。

## 四、团队角色体系

团队层面支持三种角色；小组角色仅在小组层面分配（见 [小组管理模块](../05-小组管理模块/小组管理模块.md)）。

| 角色 | 层级 | 说明 | 权限范围 |
|------|------|------|----------|
| 团队核心管理员（`team_core_admin`） | L5 | 团队核心管理员 | 管理本团队及下属小组，含结构性 / 管理性操作 |
| 团队普通管理员（`team_ordinary_admin`） | L4 | 团队普通管理员 | 负责团队成员管理（邀请 / 移除 / 列表）与资源读取，不可做结构性操作 |
| RegularMember | L1 | 普通成员 | 团队内基础权限 |
| 组织核心管理员（`organization_core_admin`） | L7 | 组织核心管理员 | 通过权限继承可操作团队内所有资源 |

> 完整角色体系（含 Public L0、SuperAdmin L8）与权限点矩阵见 [权限管理模块](../06-权限管理模块/权限管理模块.md)。
