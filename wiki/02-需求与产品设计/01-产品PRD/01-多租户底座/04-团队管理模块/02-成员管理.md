## 一、功能需求

### FR-TEAM-003：邀请成员

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 团队管理员邀请成员加入团队 |
| **所需权限点** | `team.member.invite` |
| **验收标准** | 发起邀请，被邀请人确认后加入团队 |

**详细规则：**
- 团队核心管理员（`team_core_admin`）或团队普通管理员（`team_ordinary_admin`）可邀请成员（组织核心管理员 `organization_core_admin` 通过权限继承也可邀请）
- 无论被邀请人是否已有账号，**均需被邀请人确认后加入**（统一邀请流）
- 邀请方式：通过手机号 / 邮箱（对方已有账号则可用 account_id）
- **必须指定角色**（默认 RegularMember），被邀请人可看到将被分配的角色
- 可分配角色仅限：团队核心管理员（`team_core_admin`）、团队普通管理员（`team_ordinary_admin`）、RegularMember（**不含小组角色**，小组角色仅在小组层面分配）
- **前置校验**：被邀请人必须已是本组织成员，否则需先加入组织（见 [组织成员管理](../03-组织管理模块/02-成员管理.md)）
- 同一用户不能重复加入同一团队
- 邀请有效期 7 天，过期自动失效
- 邀请创建记录，状态：`pending`（待确认）→ `accepted`（已接受）/ `rejected`（已拒绝）/ `expired`（已过期）
- 邀请操作记录审计日志（操作类型：`team.member.invite`）

**邀请链接机制（业务）：**
- 链接携带一次性校验码，有效期与邀请有效期一致（7 天）
- 一次性语义：链接被成功使用（接受 / 拒绝）后立即失效；过期后亦失效
- 防篡改：系统校验时比对校验码有效性、邀请状态与有效期；校验不通过则拒绝访问
- 注：此邀请链接机制同样适用于组织和小组的成员邀请

**被邀请人确认流程：**
1. 被邀请人点击链接 → 系统校验链接有效性（状态与有效期）
2. 如果未注册：跳转注册页面，预填邀请信息（目标团队、将被分配的角色）；注册成功后自动接受邀请
3. 如果已注册：跳转邀请确认页面，展示邀请信息（来源团队、角色）；用户点击「接受」加入团队，点击「拒绝」则状态变为 rejected

### FR-TEAM-004：移除成员

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 团队管理员移除团队成员 |
| **所需权限点** | `team.member.remove` |
| **验收标准** | 移除后成员不再属于该团队 |

**详细规则：**
- 团队核心管理员（`team_core_admin`）或团队普通管理员（`team_ordinary_admin`）可移除普通成员（RegularMember）和下属小组管理员（小组核心管理员 `group_core_admin` / 小组普通管理员 `group_ordinary_admin`）
- 不可移除其他团队管理员（需先降级，见 [FR-TEAM-007](./03-角色与权限.md)）
- 不可移除自己
- 组织核心管理员（`organization_core_admin`）通过权限继承也可执行移除
- 移除后该成员在团队下**所有小组**的成员关系同步级联移除
- 级联移除时，每条成员关系移除都需单独记录审计日志（操作类型分别为 `team.member.remove` 和 `group.member.remove`）
- 级联移除使用与直接移除相同的软删除策略
- 移除顺序：小组 → 团队（从底层向上），避免中间层级悬空
- 移除操作记录审计日志（操作类型：`team.member.remove`）
- 被移除成员的权限上下文失效

### FR-TEAM-005：获取成员列表

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 团队管理员查看团队成员列表 |
| **所需权限点** | `team.member.list` |
| **验收标准** | 返回分页成员列表 |

**详细规则：**
- 团队核心管理员（`team_core_admin`）或团队普通管理员（`team_ordinary_admin`）可查看成员列表
- 组织核心管理员（`organization_core_admin`）也可查看（权限继承）
- 支持分页查询（默认第 1 页，每页 20 条，最大 100 条）
- 支持按角色筛选（团队核心管理员 / 团队普通管理员 / RegularMember）
- 支持按用户名搜索（模糊匹配）
- 返回成员基本信息（account_id、用户名、头像、角色、加入时间）

## 二、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 邀请非组织成员 | 拒绝，提示需先加入组织 |
| 邀请已存在成员 | 拒绝，提示已在团队中 |
| 邀请过期 | 拒绝，提示邀请已过期，需重新发起 |
| 邀请已接受 / 已拒绝 | 拒绝，提示邀请状态已处理 |
| 邀请分配小组角色 | 拒绝，小组角色仅在小组层分配 |
| 移除团队管理员 | 拒绝，需先降级 |
| 移除自己 | 拒绝 |
| 非管理员操作 | 拒绝 |
