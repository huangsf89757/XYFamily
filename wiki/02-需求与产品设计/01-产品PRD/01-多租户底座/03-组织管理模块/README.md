## 一、产品目标

组织管理模块是 XYFamily 多租户底座的核心治理模块，目标是：

1. **构建清晰的层级治理模型**：以「组织 → 团队 → 小组 → 成员」四级结构承载企业级多租户数据隔离与权限治理。
2. **提供安全可控的成员生命周期管理**：通过统一邀请流、对方确认、邀请-注册联动、最后管理员保护等机制，保障组织成员关系的安全与可追溯。
3. **支撑组织自治**：允许组织管理员在平台规则内自治（创建团队、邀请成员、分配角色），同时由超级管理员保留全局兜底能力。

## 二、模块定位与边界

### 2.1 模块边界

| 属于本模块 | 不属于本模块（其余模块负责） |
|------------|------------------------------|
| 组织创建、组织信息维护 | 团队 / 小组的完整管理（成员、信息维护，见团队管理 / 小组管理模块） |
| 组织成员邀请 / 移除 / 列表 | 团队、小组成员管理（团队管理 / 小组管理模块） |
| 组织角色分配（核心 / 普通管理员）、核心管理员降级 | 细粒度权限点管理（权限管理模块） |
| 创建团队入口与接棒机制 | 团队的成员管理、小组管理（团队管理模块） |
| 组织创建时的初始管理员指定 | 超级管理员全局兜底操作（超级管理员模块） |
| 组织关键操作的审计日志 | 审计日志存储与查询（审计日志模块） |

组织是**多租户架构中最顶层的管理单元**，所有团队、小组与成员最终都归属于某一个组织。组织间数据完全隔离，确保企业级安全合规。

**组织层级关系：**

```
组织 (Organization)        ← 最高层级管理单元，数据隔离边界
  └── 团队 (Team)          ← 组织下属的中层协作单元
       └── 小组 (Group)    ← 团队下属的基层执行单元
            └── 成员 (Member) ← 归属于组织/团队/小组的用户
```

**与其他模块的关系：**
- 上游依赖：用户认证模块、账号管理模块（提供用户身份）、超级管理员模块（创建组织的入口）。
- 协同模块：团队管理模块、小组管理模块复用本模块的邀请 / 角色 / 最后管理员保护机制；权限管理模块承载细粒度权限；审计日志模块记录本模块所有关键操作。

## 三、术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 组织 | Organization | 多租户架构下最高层级管理单元，数据隔离边界 |
| 团队 | Team | 组织下属的中层协作单元，可包含多个小组 |
| 小组 | Group | 团队下属的基层执行单元 |
| 成员 | Member | 归属于组织 / 团队 / 小组的用户 |
| 超级管理员 | SuperAdmin | 平台级全局管理员，可跨组织操作 |
| 组织管理员 | 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`） | 组织级别管理员，可管理本组织及下属团队 / 小组（核心另含结构性操作） |
| 普通成员 | RegularMember | 组织内仅拥有基础权限的成员 |
| 接棒机制 | Baton Mechanism | 创建组织 / 团队时可通过指定初始管理员（对方确认后生效），而非仅限创建者本人 |
| 统一邀请流 | Unified Invite Flow | 无论被邀请人是否注册，均需对方确认后才加入 |
| 邀请-注册联动 | Invite-Registration Linkage | 未注册用户点击邀请链接完成注册后自动接受邀请并加入 |
| 一次性 Token | One-time Token | 邀请链接的一次性校验码，使用一次或过期后失效，用于防止链接被篡改与滥用 |
| 软删除 | Soft Delete | 逻辑删除，保留数据并标记删除时间，用于审计与恢复 |
| 最后管理员保护 | Last Admin Protection | 组织中最后一个管理员不可被降级、移除或注销 |
| 邀请状态机 | Invitation State Machine | 邀请在 pending（待确认）→ accepted（已接受）/ rejected（已拒绝）/ expired（已过期）之间的状态流转 |

## 六、组织角色体系

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| SuperAdmin | 超级管理员 | 全局所有组织（平台级，可跨组织创建 / 强制降级） |
| 组织核心管理员（`organization_core_admin`） | 组织核心管理员 | 管理本组织及下属团队 / 小组（创建团队、邀请成员、移除成员、分配角色、修改组织信息、查看审计） |
| 组织普通管理员（`organization_ordinary_admin`） | 组织普通管理员 | 负责组织成员管理（邀请 / 移除 / 列表）与资源读取，不可做结构性操作 |
| RegularMember | 普通成员 | 组织内基础权限（查看组织信息、被邀请加入团队 / 小组） |

> 完整角色体系（含 Public L0、SuperAdmin L8）与权限点矩阵见 [权限管理模块](../06-权限管理模块/README.md)。组织核心管理员（`organization_core_admin`） 继承 组织普通管理员（`organization_ordinary_admin`） 全部权限，二者均继承 RegularMember 权限。
