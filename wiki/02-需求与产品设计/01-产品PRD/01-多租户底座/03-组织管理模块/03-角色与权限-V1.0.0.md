## 一、功能需求

### FR-ORG-007：分配角色

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 为组织成员分配 组织核心管理员 / 组织普通管理员 / RegularMember 角色 |
| **验收标准** | 分配后成员角色即时生效，并刷新其权限上下文 |

#### 功能目标
让组织管理员能够在组织内自治地授予 / 调整成员角色，实现管理权责的分配。

#### 用户故事
> 作为组织核心管理员，我希望把某位核心成员提升为组织管理员，以便共同管理组织。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）或 SuperAdmin。
- 目标成员当前属于本组织。

#### 触发场景
- 管理员在成员列表点击「设为组织核心管理员 / 设为组织普通管理员 / 设为普通成员」。

#### 详细业务流程
1. 组织核心管理员提交为成员分配角色请求，指定目标角色。
2. 系统校验操作者权限为组织核心管理员或超级管理员；否则拒绝。
3. 系统校验目标成员存在且属于本组织（排除已逻辑删除）；否则提示资源不存在。
4. 系统校验角色枚举（组织核心管理员 / 组织普通管理员 / 普通成员）。
5. 更新成员角色。
6. 刷新该成员的权限上下文。
7. 记录角色变更的审计日志（记录变更前后角色）。
8. 返回更新后的角色。

#### 角色权限对比

| 权限 | 组织核心管理员 | 组织普通管理员 | RegularMember |
|------|:-:|:-:|:-:|
| 查看组织信息 | ✅ | ✅ | ✅ |
| 创建团队 | ✅ | ❌ | ❌ |
| 邀请组织成员 | ✅ | ✅ | ❌ |
| 移除组织成员 | ✅ | ✅ | ❌ |
| 分配组织角色 | ✅ | ❌ | ❌ |
| 修改组织信息 | ✅ | ❌ | ❌ |
| 降级核心管理员 | ✅ | ❌ | ❌ |
| 查看审计日志 | ✅ | ❌ | ❌ |

> 权限继承：组织核心管理员 继承 组织普通管理员 的全部权限；两者均继承 RegularMember 的权限（详见 [权限管理模块](../06-权限管理模块/权限管理模块-V1.0.0.md) 二维继承模型）。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 成员标识（account_id） | 是 | 目标成员的账号标识 |
| 角色（role） | 是 | 取值为 组织核心管理员 / 组织普通管理员 / 普通成员 |

### FR-ORG-008：降级核心管理员

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 将 组织核心管理员 降级为 组织普通管理员 或 RegularMember（受最后核心管理员保护约束） |
| **验收标准** | 降级后该成员不再拥有对应管理员权限，权限上下文刷新 |

#### 功能目标
在保障组织始终至少保留一名核心管理员的前提下，允许调整管理结构、收回管理权限。

#### 用户故事
> 作为组织核心管理员，我希望移除某位管理员的 admin 权限，以便调整管理结构；但系统必须阻止把组织变成「无核心管理员」状态。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）或 SuperAdmin。
- 目标成员当前为 组织核心管理员 或 组织普通管理员 且属于本组织。

#### 触发场景
- 管理员在成员列表将某位管理员「降级为普通管理员 / 降级为普通成员」。

#### 详细业务流程
1. 组织核心管理员提交将某组织管理员降级的请求，指定目标角色（组织普通管理员或普通成员）。
2. 系统校验操作者权限（超级管理员可降级任意；组织核心管理员可降级其他管理员）。
3. 不可降级自己。
4. 最后核心管理员保护：降级 组织核心管理员 时，若降级后组织核心管理员数量为 0，则提示至少保留一个核心管理员，拒绝降级。
5. 更新成员角色为目标角色。
6. 刷新该成员的权限上下文。
7. 记录角色变更的审计日志（变更前后角色）。
8. 返回更新后的角色。

> **SuperAdmin 强制降级例外**：超级管理员可绕过最后核心管理员保护执行强制降级（详见超级管理员模块），用于组织僵局兜底。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 成员标识（account_id） | 是 | 目标成员的账号标识 |
| 角色（role） | 是 | 本场景固定为 组织普通管理员 或 普通成员 |

## 二、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 非核心管理员分配角色 | 拒绝操作 |
| 分配不存在的角色 | 提示角色枚举范围 |
| 降级自己 | 拒绝操作 |
| 降级最后一个核心管理员 | 提示至少保留一个核心管理员 |
| 被降级成员不存在 | 提示资源不存在 |
| 重复分配相同角色 | 幂等返回，状态不变 |
| 超级管理员强制降级绕过保护 | 仅超级管理员可生效 |
