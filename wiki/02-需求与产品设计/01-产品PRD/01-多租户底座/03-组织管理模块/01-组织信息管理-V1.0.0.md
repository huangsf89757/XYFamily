## 一、功能需求

### FR-ORG-001：创建组织

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 超级管理员（SuperAdmin）创建组织；创建者或指定的人成为组织核心管理员（`organization_core_admin`） |
| **验收标准** | 创建成功后返回组织信息，组织管理员身份即时生效；指定初始管理员时走统一邀请流，对方确认后生效 |

#### 功能目标
为平台建立一个新的多租户组织单元，并明确其首位组织核心管理员（`organization_core_admin`）（Owner），作为后续团队、成员与权限治理的根节点。

#### 用户故事
> 作为超级管理员，我希望创建一个新组织并指定其管理者，以便该企业客户能在平台内独立管理自己的团队与成员。

#### 前置条件
- 操作者已通过认证，且角色为 SuperAdmin。
- 系统运行正常，组织名称全局唯一性校验可用。

#### 触发场景
- 平台开通新企业租户时由 SuperAdmin 主动创建。
- 创建时可选将组织管理权「接棒」给另一名用户（已注册或未注册均可）。

#### 详细业务流程
1. 超级管理员提交创建组织请求，填写组织名称，并可选择性填写 logo、描述，以及可选的初始管理员指定。
2. 系统校验操作者为超级管理员；否则拒绝创建。
3. 系统校验组织名称：必填、长度 2–64 字符、格式合法、全局唯一（大小写不敏感比对）。
4. 若未指定初始管理员：创建者自动成为组织核心管理员（`organization_core_admin`）。
5. 若指定初始管理员：
   - 已注册用户（可通过手机号 / 邮箱 / 账号标识定位）→ 创建待确认邀请，等待对方在邀请中心确认。
   - 未注册用户 → 创建待确认邀请并生成一次性邀请链接，发送邀请通知；对方注册并确认后成为组织核心管理员（`organization_core_admin`）。
6. 系统创建组织记录，组织核心管理员（`organization_core_admin`）身份即时生效。
7. 记录创建组织的审计日志。
8. 创建成功后返回组织信息。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织名称（name） | 是 | 长度 2–64 字符；允许中英文、数字、空格及 `-_&()·`；全局唯一（大小写不敏感）；禁止纯空白或纯特殊字符 |
| logo | 否 | URL 格式（http/https），最大长度 512；为空表示未设置 |
| 描述（description） | 否 | 长度 0–500 字符 |
| 初始管理员（initial_admin） | 否 | 存在时需指定其标识与角色 |
| 初始管理员标识 | 条件必填 | 当指定初始管理员时必填；取值为手机号、邮箱或账号标识 |
| 初始管理员角色 | 否 | 取值为组织核心管理员（`organization_core_admin`），默认组织核心管理员 |

### FR-ORG-002：获取组织信息

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 组织成员查看所属组织的基本信息（不含敏感配置） |
| **验收标准** | 返回组织基本信息（名称、logo、描述、成员数、团队数、管理员数等） |

#### 功能目标
为组织成员提供组织基本信息的只读视图，支撑前端展示与后续操作入口。

#### 用户故事
> 作为组织成员，我希望查看组织名称、logo、成员数等基本信息，以便了解我所在的组织。

#### 前置条件
- 操作者已通过认证，且为组织成员（数据隔离边界）。

#### 触发场景
- 进入组织首页 / 组织设置页时拉取。
- 其他模块（团队、小组）需要展示所属组织上下文时调用。

#### 详细业务流程
1. 组织成员发起查看组织信息请求。
2. 系统校验操作者属于该组织（数据隔离边界）；否则拒绝访问。
3. 系统返回组织基本信息（排除已逻辑删除的记录）。
4. 聚合成员数、团队数、管理员数等统计信息。
5. 返回基本信息（不含敏感配置，如第三方集成密钥等）。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识，用于定位组织 |

### FR-ORG-003：更新组织信息

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 组织核心管理员（`organization_core_admin`）修改组织信息 |
| **验收标准** | 修改成功后返回更新后的信息，并记录审计日志 |

#### 功能目标
允许组织管理员维护组织资料，保持组织信息准确。

#### 用户故事
> 作为组织管理员，我希望修改组织名称、logo、描述，以便保持组织资料准确。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）或 SuperAdmin。

#### 触发场景
- 管理员在设置页编辑并保存组织资料。

#### 详细业务流程
1. 组织管理员提交更新组织信息请求。
2. 系统校验操作者权限为 组织核心管理员（`organization_core_admin`）或超级管理员；否则拒绝修改。
3. 系统校验各字段；组织名称若变更需全局唯一（排除自身）。
4. 系统更新组织信息。
5. 记录更新组织的审计日志。
6. 返回更新后的组织信息。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 组织名称（name） | 否 | 若提供则长度 2–64、全局唯一（排除自身）、格式合法 |
| logo | 否 | URL 格式（http/https），最大长度 512 |
| 描述（description） | 否 | 长度 0–500 字符 |

## 二、最后管理员保护

**核心规则**：组织 / 团队 / 小组中最后一个管理员不可降级、不可移除、不可注销。

| 约束 | 说明 |
|------|------|
| 最后管理员保护 | 当一个组织 / 团队 / 小组只有一个核心管理员时，该核心管理员不可被降级为普通管理员或普通成员 |
| 必须先转让 | 降级 / 移除前必须指定新的管理员（通过邀请流程确认后生效） |
| 不可注销 | 最后一个管理员注销账号时，必须先转让管理员身份 |
| SuperAdmin 强制降级例外 | 超级管理员可强制降级，不受此限制（详见超级管理员模块） |

**适用范围**：组织（组织核心管理员 `organization_core_admin`）、团队（团队核心管理员 `team_core_admin`）、小组（小组核心管理员 `group_core_admin`）三个层级均遵循本规则。

## 三、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 非超级管理员创建组织 | 拒绝创建 |
| 组织名称重复 | 提示名称已存在，拒绝创建 |
| 组织名称长度不合规 | 提示长度需在 2–64 字符 |
| 组织名称含非法字符 | 提示允许字符范围 |
| 非组织成员查看 | 拒绝访问（数据隔离） |
| 非组织核心管理员修改 | 拒绝修改 |
| 组织名称未变更却触发唯一性冲突 | 与自身排除后比对，避免误报 |
| 组织已删除后访问 | 视为不存在 |
| 组织标识格式非法 | 提示标识格式错误 |
