## 一、功能需求

### FR-ORG-004：邀请成员

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 组织管理员通过手机号 / 邮箱邀请成员加入组织；无论对方是否注册，均需确认后加入 |
| **验收标准** | 发起邀请后创建待确认邀请记录并生成一次性邀请链接；被邀请人确认后成为组织成员 |

#### 功能目标
提供统一、安全、可追溯的成员邀请机制，覆盖已注册与未注册两类用户，并保证邀请链接不可篡改、不可滥用。

#### 用户故事
> 作为组织管理员，我希望通过手机号或邮箱邀请同事加入组织，无论对方是否已有账号，对方确认后即可成为成员。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`）或 SuperAdmin。
- 邀请目标未被禁邀（无活跃封禁策略）。

#### 触发场景
- 管理员在「成员管理」点击「邀请成员」，填写手机号 / 邮箱并指定角色。

#### 详细业务流程
1. 组织管理员提交邀请成员请求，填写被邀请人手机号 / 邮箱并指定角色。
2. 系统校验操作者权限为 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`）或超级管理员；否则拒绝。
3. 系统校验被邀请人标识格式（手机号 / 邮箱 / 账号标识）。
4. 解析目标用户：若已注册则取其账号标识；若未注册则以标识暂存。
5. 校验该用户未已是本组织成员（已逻辑删除的除外）；否则提示已在组织中。
6. 校验角色枚举（组织核心管理员 / 组织普通管理员 / 普通成员），默认普通成员。
7. 创建待确认邀请记录：状态为待确认、有效期 7 天、生成一次性邀请链接。
8. 发送邀请通知（短信 / 邮件 / 站内）。
9. 记录邀请成员的审计日志。
10. 返回邀请信息（含邀请链接）。

#### 邀请链接机制（业务）

- 链接有效期：与邀请有效期一致（7 天）。
- 一次性语义：链接被成功使用（接受 / 拒绝）后立即失效；过期后亦失效。
- 防篡改：链接携带一次性校验码，系统在校验时比对校验码有效性、邀请状态与有效期；校验不通过（不一致、已处理或已过期）则拒绝访问。

> 注：此邀请链接机制同样适用于团队和小组的成员邀请。

#### 未注册用户邀请-注册联动流程

1. 被邀请人点击链接 → 系统校验邀请链接有效性（状态与有效期比对）。
2. 如果未注册：
   - 跳转注册页面，预填邀请信息（目标组织 / 团队 / 小组、将被分配的角色）。
   - 用户完成注册（使用同一手机号 / 邮箱）→ 自动接受邀请，加入目标组织 / 团队 / 小组。
   - 注册成功后跳转至组织首页。
3. 如果已注册：
   - 跳转邀请确认页面，展示邀请信息（来源、目标、角色）。
   - 用户点击「接受」 → 加入目标组织 / 团队 / 小组。
   - 用户点击「拒绝」 → 邀请状态变为已拒绝。

#### 邀请状态机

```
        ┌─────────┐
        │ pending │
        └─────────┘
         │ 接受        │ 拒绝        │ 过期（7天）
         ▼            ▼            ▼
   ┌──────────┐ ┌──────────┐ ┌──────────┐
   │ accepted │ │ rejected │ │ expired  │
   └──────────┘ └──────────┘ └──────────┘
```

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 被邀请人标识（identifier） | 是 | 取值为手机号、邮箱或账号标识 |
| 角色（role） | 是 | 取值为组织核心管理员 / 组织普通管理员 / 普通成员，默认普通成员 |

### FR-ORG-005：移除成员

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 组织管理员移除组织成员（含级联移除其在组织下属团队 / 小组的关系） |
| **验收标准** | 移除后成员不再属于该组织，其下属团队 / 小组关系同步逻辑删除，并记录审计日志 |

#### 功能目标
在保障数据一致性与可追溯性的前提下，安全地解除成员与组织及其下属层级的归属关系。

#### 用户故事
> 作为组织管理员，我希望移除某位不再协作的成员，系统自动清理他在各团队 / 小组中的关系，避免遗留悬空数据。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`）或 SuperAdmin。
- 目标成员当前属于本组织（未逻辑删除）。

#### 触发场景
- 管理员在成员列表点击「移除」并确认。

#### 详细业务流程
1. 组织管理员提交移除成员请求。
2. 系统校验操作者权限为 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`）或超级管理员；否则拒绝。
3. 系统校验目标成员存在且属于本组织（排除已逻辑删除）；否则提示资源不存在。
4. 不可移除自己。
5. 不可移除其他组织管理员（需先降级）。
6. 级联移除：自下而上按「小组 → 团队 → 组织」顺序处理，避免中间层级悬空：
   - 移除该成员在所有本组织小组中的关系。
   - 移除该成员在所有本组织团队中的关系。
   - 移除该成员在组织中的关系。
7. 级联移除统一使用软删除策略（逻辑删除、保留数据、可恢复、保留审计）。
8. 记录各层级移除的审计日志。
9. 返回移除结果（含级联统计）。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 成员标识（account_id） | 是 | 目标成员的账号标识 |

### FR-ORG-006：获取成员列表

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 组织管理员查看组织成员列表，支持分页、角色筛选与关键字搜索 |
| **验收标准** | 返回分页成员列表，包含成员基本信息与角色 |

#### 功能目标
为管理员提供可检索、可筛选的成员视图，支撑后续邀请、移除、改角色等操作。

#### 用户故事
> 作为组织管理员，我希望按角色或姓名搜索成员并分页浏览，以便快速定位需要管理的成员。

#### 前置条件
- 操作者已认证，角色为 组织核心管理员（`organization_core_admin`）/ 组织普通管理员（`organization_ordinary_admin`）或 SuperAdmin。

#### 触发场景
- 进入「成员管理」页面时加载首屏列表。
- 输入搜索关键字或切换角色筛选时刷新列表。

#### 详细业务流程
1. 组织管理员提交查看成员列表请求，可携带分页、角色筛选与关键字搜索条件。
2. 系统校验操作者权限；否则拒绝。
3. 系统校验分页与筛选参数合法性（页码≥1、每页数量 1–100、角色枚举、关键字长度）。
4. 仅查询未逻辑删除的成员记录。
5. 按角色过滤、按关键字对姓名 / 标识模糊匹配。
6. 分页返回列表与分页元信息。

#### 字段业务约束

| 字段 | 必填 | 业务约束 |
|------|------|----------|
| 组织标识（org_id） | 是 | 目标组织的唯一标识 |
| 页码（page） | 否 | ≥1，默认 1 |
| 每页数量（page_size） | 否 | 1–100，默认 20 |
| 角色（role） | 否 | 取值为组织核心管理员 / 组织普通管理员 / 普通成员 |
| 关键字（keyword） | 否 | 长度 0–64，模糊匹配姓名 / 标识 |

## 二、边界与异常处理

| 场景 | 处理方式 |
|------|----------|
| 邀请已存在成员 | 提示已在组织中 |
| 邀请不存在用户 | 创建待确认邀请记录，发送邀请通知，等用户确认后加入 |
| 邀请过期 | 提示邀请已过期，需重新发起 |
| 邀请已接受 / 已拒绝 | 提示邀请状态已处理 |
| 邀请链接校验失败 | 链接失效或校验不通过，拒绝访问 |
| 移除其他组织管理员 | 需先降级 |
| 移除自己 | 拒绝操作 |
| 非管理员操作 | 拒绝操作 |
| 级联移除中某层级关系不存在 | 跳过该层级，继续向上，不阻断整体移除 |
| 并发移除同一成员 | 以首次成功为准，后续幂等返回 |
