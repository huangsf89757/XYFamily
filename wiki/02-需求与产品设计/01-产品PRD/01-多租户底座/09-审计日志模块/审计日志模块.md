## 一、模块定位

审计日志模块负责记录系统中的关键操作与登录行为，为安全审计、合规检查和问题排查提供依据。分为**登录审计日志**和**操作审计日志**两类；操作审计又按业务域（组织 / 团队 / 小组 / 账号 / 系统配置）细分。所有日志**仅追加、不可篡改**，并受保留期与脱敏规则约束。

---

## 二、功能需求

### FR-AUDIT-001：登录审计日志

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 记录所有登录尝试（成功 / 失败） |
| **验收标准** | 所有登录操作均记录日志，含失败原因 |

**详细规则：**

- 记录所有登录尝试（包括成功和失败）。
- 记录字段：

| 字段 | 说明 |
|------|------|
| account_id | 账号ID |
| login_method | 登录方式（password / sms_code / email_code / wechat） |
| login_result | 登录结果（success / failed） |
| failure_reason | 失败原因（如密码错误、验证码错误、账号锁定、触发限流） |
| ip_address | IP 地址 |
| user_agent | User-Agent |
| created_at | 时间戳 |

- 登录失败同样记录（密码错误、验证码错误、账号锁定、限流等）。
- 日志**异步写入**，不影响登录性能。
- 日志保留 1 年（参考 NFR-SEC-007）。

---

### FR-AUDIT-002：组织域操作审计

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 记录组织在创建 / 修改 / 成员 / 角色方面的关键操作 |
| **验收标准** | 组织相关关键操作均记录，含前后快照 |

**操作清单：** `org.create`、`org.update`、`org.member.invite`、`org.member.remove`、`org.role.assign`、`org.role.downgrade`

**详细规则：**

- 记录字段：账号ID、操作类型、操作目标、操作详情（含前后快照）、IP、时间戳。
- 角色变更：`before.role` + `after.role`。
- 组织信息修改：`before.name`/`before.description` + `after.name`/`after.description` 等变更字段。
- 成员移除：`before.membership`（含 role、joined_at 等）。
- 异步写入，不影响业务性能。

### FR-AUDIT-005：账号域操作审计

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 记录账号在密码 / 状态 / 第三方绑定方面的关键操作 |
| **验收标准** | 账号相关关键操作均记录，密码不落明文 |

**操作清单：** `account.password.update`、`account.password.reset`、`account.deactivate`、`account.undeactivate`、`account.third_party.bind`、`account.third_party.unbind`

**详细规则：**

- 记录字段：账号ID、操作类型、操作目标、操作详情、IP、时间戳。
- 密码修改 / 重置：记录操作类型，不记录密码明文（记录 `changed: true`）。
- 账号注销 / 恢复：记录操作前后状态。
- 异步写入，不影响业务性能。

### FR-AUDIT-006：系统配置域操作审计

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 记录系统配置修改与强制降级等关键操作 |
| **验收标准** | 配置变更与强制降级均记录，含前后值 |

**操作清单：** `admin.config.update`、`admin.force_downgrade`

**详细规则：**

- 配置变更：`before.config_value` + `after.config_value`。
- 强制降级：`before.role`（操作前角色快照）。
- 异步写入，不影响业务性能。

### FR-AUDIT-007：组织级审计日志查询

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 组织核心管理员查询本组织审计日志 |
| **验收标准** | 返回本组织符合筛选条件的日志列表 |

**详细规则：**

- 组织核心管理员（`organization_core_admin`）可查询**本组织**审计日志。
- 支持按时间范围、账号ID、操作类型、登录结果筛选。
- 支持分页查询。
- 普通成员不可查询审计日志。

---

### FR-AUDIT-008：全局审计日志查询

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | SuperAdmin 跨组织查询全部审计日志 |
| **验收标准** | 返回全局符合筛选条件的日志列表 |

**详细规则：**

- SuperAdmin 可查询**所有组织**的登录审计与操作审计日志。
- 在 FR-AUDIT-007 的筛选维度基础上，额外支持按**组织维度**过滤。
- 支持分页与日志详情查看（含 `details` 前后快照）。
- 详见超级管理员模块 FR-ADMIN-006。

---

### FR-AUDIT-009：审计日志防篡改与保留

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 日志仅追加、不可篡改，并按保留期与匿名化规则管理 |
| **验收标准** | 日志不可修改/删除；到期按策略保留；注销后匿名化 |

**详细规则：**

- 日志**仅追加**，不可修改、不可删除（不可篡改）。
- 保留期 1 年（参考 NFR-SEC-007）。
- 账号注销后，日志中该账号的个人信息匿名化处理，且不逆向还原。
- 即便是 SuperAdmin 也不可篡改或删除日志（见超级管理员模块「安全与约束」）。

---

### FR-AUDIT-010：敏感信息脱敏

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 日志中不记录凭证明文，查询展示按需脱敏 |
| **验收标准** | 密码/凭证不落库；查询展示的 PII 脱敏 |

**详细规则：**

- 密码、Token、验证码等凭证**绝不记录明文**。
- 查询 / 导出展示时，对手机号、邮箱等 PII 按需脱敏。
- 匿名化后信息不可逆向还原。

---

### FR-AUDIT-011：审计日志导出（延后 P2）

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **描述** | 按筛选条件导出审计日志 |
| **验收标准** | 导出文件符合脱敏与权限规则 |

**详细规则：**

- 支持按筛选条件导出（如 CSV / Excel）。
- 导出操作本身记录审计日志。
- 导出内容遵循 FR-AUDIT-010 脱敏规则。
- 仅具备对应查询权限的角色可导出相应范围日志。
