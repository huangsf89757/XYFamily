## 1. 需求背景与目标

生态扩展对应里程碑 Stage 3，是平台能力对外开放与商业化的延展阶段。本 Stage 复用 Stage 1 的 RBAC 权限、多租户隔离与审计能力，向第三方 / 跨组织场景延伸。当前仅建立入口占位。

> 本 PRD 仅描述产品需求（What），不含接口定义、数据库表结构等实现细节（How）。详细内容待 Stage 1 上线稳定后细化。

## 2. 预期功能方向（草案，待细化）

- 开放平台（OpenAPI / 开放应用接入）
- 商业化（计费 / 订阅 / 增值能力）
- 跨组织协作（组织间授权与数据协作）
- 第三方集成统一方案（参见 03-架构与方案设计 / 05-支撑域 / 03-第三方集成统一方案）

## 3. 依赖与约束

- 强依赖 Stage 1 底座稳定的认证、权限与隔离能力
- 需配套数据合规与分级治理（07-合规安全与数据治理）

## 4. 预期里程碑拆解（草案，待细化）

> 以下为 Stage 3 的方向性拆解草案，非最终计划；待 Stage 1 底座能力稳定后正式细化。

| 预期 Phase | 方向 | 复用底座能力 |
|-----------|------|--------------|
| Phase A 开放平台 | OpenAPI / 开放应用接入 / 开发者接入 | RBAC、多租户隔离、审计 |
| Phase B 商业化 | 计费 / 订阅 / 增值能力 | 组织 / 账号体系 |
| Phase C 跨组织协作 | 组织间授权与数据协作 | 多租户隔离、权限委派 |
| Phase D 第三方集成统一 | 统一第三方集成方案 | 认证、权限中间件 |
