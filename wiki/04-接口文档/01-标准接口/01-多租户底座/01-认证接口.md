## 一、发送验证码

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/verification-codes` |
| 认证 | ❌ 公开 |
| HTTP Status | 200 |
| 限流 | 同一手机号/邮箱 1 分钟 1 次；同一 IP 5 分钟 5 次 |

### 1.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target` | `string` | ✅ | 手机号（11 位）或邮箱（RFC 5322） |
| `type` | `string` (枚举) | ✅ | 验证码类型：`register`/`login`/`reset_password` |

**请求示例**：

```json
{
  "target": "13800000000",
  "type": "register"
}
```

### 1.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "expires_in": 300
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `expires_in` | `int` | 验证码有效期（秒），默认 300 秒 |

**错误响应**：

```json
{
  "code": 114290,
  "message": "发送频率超限",
  "detail": "该手机号发送验证码过于频繁，请稍后重试",
  "data": null,
  "pagination": null
}
```

### 1.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 429 | 114290 | 发送频率超限（手机号） |
| 429 | 114291 | 发送频率超限（IP） |
| 429 | 114293 | 发送频率超限（账号级） |

## 二、注册

### 2.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/register` |
| 认证 | ❌ 公开 |
| HTTP Status | 201 |
| 限流 | 无 |

### 2.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` (枚举) | ✅ | 注册类型：`phone`/`email` |
| `target` | `string` | ✅ | 手机号（11 位）或邮箱（RFC 5322） |
| `code` | `string` | ✅ | 6 位数字验证码 |
| `password` | `string` | ✅ | 密码（至少 8 位，含大小写字母和数字） |

**请求示例**：

```json
{
  "type": "phone",
  "target": "13800000000",
  "code": "123456",
  "password": "Str0ng@Pass12"
}
```

### 2.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174000",
    "nickname": "用户1234",
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 1800
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `account_id` | `string` (UUID) | 用户 ID |
| `nickname` | `string` | 默认昵称 |
| `access_token` | `string` | 访问令牌 |
| `refresh_token` | `string` | 刷新令牌 |
| `expires_in` | `int` | Access Token 有效期（秒） |

**错误响应**：

```json
{
  "code": 200002,
  "message": "手机号已被注册",
  "detail": "该手机号已注册账号，请直接登录或使用其他手机号",
  "data": null,
  "pagination": null
}
```

### 2.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 400 | 110001 | 验证码错误 |
| 400 | 110002 | 验证码已过期 |
| 400 | 110003 | 验证码已被使用 |
| 400 | 110004 | 验证码尝试次数超限 |
| 400 | 200001 | 密码强度不足 |
| 400 | 200002 | 手机号已被注册 |
| 400 | 200003 | 邮箱已被注册 |

## 三、登录

### 3.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/login` |
| 认证 | ❌ 公开 |
| HTTP Status | 200 |
| 限流 | 同一 IP 5 分钟 5 次失败，锁定 15 分钟 |

### 3.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `identifier` | `string` | ✅ | 账号标识（手机号/邮箱/用户名/account_id） |
| `identifier_type` | `string` (枚举) | ✅ | 标识类型：`phone`/`email`/`username`/`account_id` |
| `credential_type` | `string` (枚举) | ✅ | 凭证类型：`password`/`code` |
| `password` | `string` | ⚠️ | 密码（credential_type 为 password 时必填） |
| `code` | `string` | ⚠️ | 验证码（credential_type 为 code 时必填） |

**请求示例（密码登录）**：

```json
{
  "identifier": "13800000000",
  "identifier_type": "phone",
  "credential_type": "password",
  "password": "Str0ng@Pass12"
}
```

**请求示例（验证码登录）**：

```json
{
  "identifier": "13800000000",
  "identifier_type": "phone",
  "credential_type": "code",
  "code": "123456"
}
```

### 3.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174000",
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 1800,
    "status": "active"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `account_id` | `string` (UUID) | 用户 ID |
| `access_token` | `string` | 访问令牌 |
| `refresh_token` | `string` | 刷新令牌 |
| `expires_in` | `int` | Access Token 有效期（秒） |
| `status` | `string` | 账号状态：`active`/`deactivating` |

**业务说明**：`deactivating` 状态的账号允许登录，但仅允许执行「恢复账号」操作，其他业务操作返回 403 + 200007。

**错误响应**：

```json
{
  "code": 101008,
  "message": "密码错误",
  "detail": "密码输入错误，请检查后重试",
  "data": null,
  "pagination": null
}
```

### 3.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 401 | 101007 | 账号不存在 |
| 401 | 101008 | 密码错误 |
| 401 | 101009 | 账号已注销 |
| 400 | 110001 | 验证码错误 |
| 400 | 110002 | 验证码已过期 |
| 400 | 110003 | 验证码已被使用 |
| 400 | 110004 | 验证码尝试次数超限 |
| 429 | 104290 | 登录失败次数过多，已锁定 |

## 四、刷新 Token

### 4.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/refresh` |
| 认证 | ✅ 需 Refresh Token |
| HTTP Status | 200 |
| 限流 | 每账号每分钟 5 次 |

### 4.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `refresh_token` | `string` | ✅ | Refresh Token |

**请求示例**：

```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### 4.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 1800
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `access_token` | `string` | 新的访问令牌 |
| `refresh_token` | `string` | 新的刷新令牌（Rotation：旧 Refresh 立即作废） |
| `expires_in` | `int` | Access Token 有效期（秒） |

**错误响应**：

```json
{
  "code": 101006,
  "message": "Refresh Token 已过期",
  "detail": "Refresh Token 已超出有效期，请重新登录",
  "data": null,
  "pagination": null
}
```

### 4.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100003 | 必填字段缺失 |
| 401 | 101004 | Refresh Token 无效 |
| 401 | 101005 | Refresh Token 已撤销 |
| 401 | 101006 | Refresh Token 已过期 |
| 401 | 101010 | Refresh Token 被盗用，全员下线 |
| 429 | 104291 | 刷新频控 |

## 五、登出

### 5.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/logout` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 无 |

### 5.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `refresh_token` | `string` | 否 | Refresh Token，带则单设备撤销；不带则仅黑名单当前 Access |

**请求示例**：

```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### 5.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": null,
  "pagination": null
}
```

**业务说明**：**幂等性**——Token 已失效仍返回成功。

### 5.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 六、重置密码

### 6.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/auth/reset-password` |
| 认证 | ❌ 公开 |
| HTTP Status | 200 |
| 限流 | 每账号每小时 3 次 |

### 6.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | `string` (枚举) | ✅ | 类型：`phone`/`email` |
| `target` | `string` | ✅ | 手机号（11 位）或邮箱（RFC 5322） |
| `code` | `string` | ✅ | 6 位数字验证码 |
| `new_password` | `string` | ✅ | 新密码（至少 8 位，含大小写字母和数字） |

**请求示例**：

```json
{
  "type": "phone",
  "target": "13800000000",
  "code": "123456",
  "new_password": "New@Pass34"
}
```

### 6.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": null,
  "pagination": null
}
```

**业务说明**：**防枚举**——账号不存在也返回成功，避免账号枚举攻击。

**错误响应**：

```json
{
  "code": 200001,
  "message": "密码强度不足",
  "detail": "密码至少需要 8 位，包含大小写字母和数字",
  "data": null,
  "pagination": null
}
```

### 6.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 400 | 110001 | 验证码错误 |
| 400 | 110002 | 验证码已过期 |
| 400 | 110003 | 验证码已被使用 |
| 400 | 110004 | 验证码尝试次数超限 |
| 400 | 200001 | 密码强度不足 |
| 400 | 200005 | 新密码与当前密码相同 |
| 429 | 104292 | 重置频控 |
