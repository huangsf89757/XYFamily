## 一、系统初始化

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/superadmin/init` |
| 认证 | ❌ 公开 |
| HTTP Status | 201 |
| 限流 | 仅允许调用一次 |

### 1.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `nickname` | `string` | ✅ | 超级管理员昵称 |
| `phone` | `string` | ⚠️ | 手机号（11 位，phone/email 至少一个） |
| `email` | `string` | ⚠️ | 邮箱（RFC 5322，phone/email 至少一个） |
| `password` | `string` | ✅ | 密码（至少 8 位，含大小写字母和数字） |

**请求示例**：

```json
{
  "nickname": "系统管理员",
  "phone": "13800000000",
  "password": "Super@Admin123"
}
```

### 1.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174000",
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 1800
  },
  "pagination": null
}
```

**业务说明**：系统仅允许初始化一次，已初始化后调用返回 400。

**错误响应**：

```json
{
  "code": 800004,
  "message": "系统已初始化",
  "detail": "系统已完成初始化，不允许重复初始化",
  "data": null,
  "pagination": null
}
```

### 1.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 400 | 200001 | 密码强度不足 |
| 400 | 800004 | 系统已初始化 |

## 三、更新系统配置

### 3.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `PATCH` |
| 路径 | `/api/v1/superadmin/config` |
| 认证 | ✅ 需 Token（超级管理员） |
| HTTP Status | 200 |
| 限流 | 无 |

### 3.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `max_org_per_account` | `int` | 否 | 单账号最大组织数（1-10） |
| `max_team_per_org` | `int` | 否 | 单组织最大团队数（1-500） |
| `max_group_per_team` | `int` | 否 | 单团队最大小组数（1-200） |
| `token_expire_minutes` | `int` | 否 | Token 过期时间（分钟，15-1440） |
| `deactivate_grace_days` | `int` | 否 | 账号注销宽限期（天，7-90） |

**请求示例**：

```json
{
  "max_org_per_account": 5,
  "token_expire_minutes": 60
}
```

### 3.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "system_name": "XYFamily 多租户底座",
    "system_version": "1.0.0",
    "max_org_per_account": 5,
    "max_team_per_org": 100,
    "max_group_per_team": 50,
    "token_expire_minutes": 60,
    "deactivate_grace_days": 15,
    "init_completed": true,
    "updated_at": "2026-07-16T10:00:00Z"
  },
  "pagination": null
}
```

**错误响应**：

```json
{
  "code": 100002,
  "message": "参数值不合法",
  "detail": "max_org_per_account 必须在 1-10 之间",
  "data": null,
  "pagination": null
}
```

### 3.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100002 | 参数值不合法 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 403 | 600001 | 权限不足（非超级管理员） |

## 四、强制降级账号

### 4.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/superadmin/accounts/:account_id/downgrade` |
| 认证 | ✅ 需 Token（超级管理员） |
| HTTP Status | 200 |
| 限流 | 无 |

### 4.2 请求体

无请求体。

### 4.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174001",
    "previous_roles": ["organization_core_admin"],
    "action": "downgrade"
  },
  "pagination": null
}
```

**业务说明**：
- 超级管理员可强制降级任何账号在任何层级的角色
- 降级后该账号在所有组织/团队/小组的角色全部移除
- 操作记录审计日志

### 4.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100002 | 参数值不合法 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 403 | 600001 | 权限不足（非超级管理员） |
| 404 | 200404 | 账号不存在 |
