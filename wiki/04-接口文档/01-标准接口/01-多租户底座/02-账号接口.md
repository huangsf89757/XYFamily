## 一、获取个人信息

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `GET` |
| 路径 | `/api/v1/account/profile` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 无 |

### 1.2 请求体

无请求体。

### 1.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174000",
    "nickname": "张三",
    "avatar": null,
    "phone": "138****1234",
    "email": "z***@example.com",
    "status": "active",
    "created_at": "2026-07-13T10:00:00Z",
    "updated_at": "2026-07-14T15:30:00Z"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `account_id` | `string` (UUID) | 用户 ID |
| `nickname` | `string` | 用户昵称 |
| `avatar` | `string`/`null` | 头像 URL |
| `phone` | `string` | 手机号（脱敏） |
| `email` | `string` | 邮箱（脱敏） |
| `status` | `string` | 账号状态：`active`/`deactivating`/`deleted` |
| `created_at` | `string` | 创建时间（ISO 8601 UTC） |
| `updated_at` | `string` | 更新时间（ISO 8601 UTC） |

### 1.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 二、更新个人信息

### 2.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `PATCH` |
| 路径 | `/api/v1/account/profile` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 无 |

### 2.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `nickname` | `string` | 否 | 昵称（1-50 字符） |
| `avatar` | `string` | 否 | 头像 URL |

**请求示例**：

```json
{
  "nickname": "张三",
  "avatar": "https://example.com/avatar.jpg"
}
```

### 2.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "account_id": "123e4567-e89b-12d3-a456-426614174000",
    "nickname": "张三",
    "avatar": "https://example.com/avatar.jpg",
    "phone": "138****1234",
    "email": "z***@example.com",
    "status": "active",
    "created_at": "2026-07-13T10:00:00Z",
    "updated_at": "2026-07-16T09:00:00Z"
  },
  "pagination": null
}
```

**错误响应**：

```json
{
  "code": 100002,
  "message": "参数值不合法",
  "detail": "昵称长度必须在 1-50 字符之间",
  "data": null,
  "pagination": null
}
```

### 2.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100002 | 参数值不合法 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 三、修改密码

### 3.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/account/change-password` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 每账号每小时 3 次 |

### 3.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `old_password` | `string` | ✅ | 当前密码 |
| `new_password` | `string` | ✅ | 新密码（至少 8 位，含大小写字母和数字） |

**请求示例**：

```json
{
  "old_password": "Old@Pass12",
  "new_password": "New@Pass34"
}
```

### 3.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": null,
  "pagination": null
}
```

**业务说明**：修改成功后，**当前 Access Token 立即失效**（写入 Redis 黑名单），需重新登录。

**错误响应**：

```json
{
  "code": 200006,
  "message": "旧密码错误",
  "detail": "输入的旧密码不正确，请检查后重试",
  "data": null,
  "pagination": null
}
```

### 3.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100003 | 必填字段缺失 |
| 400 | 200001 | 密码强度不足 |
| 400 | 200005 | 新密码与当前密码相同 |
| 400 | 200006 | 旧密码错误 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
| 429 | 104294 | 修改密码频控 |

## 四、注销账号

### 4.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/account/deactivate` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 无 |

### 4.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `password` | `string` | ✅ | 当前密码（二次确认） |

**请求示例**：

```json
{
  "password": "Str0ng@Pass12"
}
```

### 4.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {
    "deactivated_at": "2026-07-16T09:00:00Z",
    "grace_period_end": "2026-07-31T09:00:00Z"
  },
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `deactivated_at` | `string` | 注销时间（ISO 8601 UTC） |
| `grace_period_end` | `string` | 宽限期结束时间（ISO 8601 UTC），默认 15 天 |

**业务说明**：
- 注销后账号状态变为 `deactivating`
- **宽限期 15 天**内可恢复，期间仅允许登录并执行恢复操作
- 宽限期结束后自动硬删除，**不可恢复**

**错误响应**：

```json
{
  "code": 200006,
  "message": "密码错误",
  "detail": "输入的密码不正确，请检查后重试",
  "data": null,
  "pagination": null
}
```

### 4.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100003 | 必填字段缺失 |
| 400 | 200006 | 密码错误 |
| 400 | 200008 | 当前已处于注销状态 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |

## 五、恢复账号

### 5.1 基本信息

| 项目 | 内容 |
|------|------|
| 方法 | `POST` |
| 路径 | `/api/v1/account/recover` |
| 认证 | ✅ 需 Token |
| HTTP Status | 200 |
| 限流 | 无 |

### 5.2 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `password` | `string` | ✅ | 当前密码（二次确认） |

**请求示例**：

```json
{
  "password": "Str0ng@Pass12"
}
```

### 5.3 响应体

**成功响应**：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": null,
  "pagination": null
}
```

**业务说明**：仅 `deactivating` 状态的账号可调用此接口。

```json
{
  "code": 200009,
  "message": "账号状态不允许恢复",
  "detail": "账号未处于注销宽限期内，无法恢复",
  "data": null,
  "pagination": null
}
```

### 5.4 错误码

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100003 | 必填字段缺失 |
| 400 | 200006 | 密码错误 |
| 400 | 200009 | 账号状态不允许恢复 |
| 401 | 101001 | 缺少 Token |
| 401 | 101002 | Token 无效或已过期 |
