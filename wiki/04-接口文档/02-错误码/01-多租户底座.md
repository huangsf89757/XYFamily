## 一、错误码体系概述

业务错误码采用 **5 位数字分段体系**，前 2~3 位表示模块，后 2~3 位表示具体错误。

**HTTP Status 与业务 Code 分离原则**：
- HTTP Status 表示请求层级语义（401/403/404/429/500）
- 业务 Code 表示具体错误原因
- 两者独立，一个 HTTP Status 可对应多个业务 Code

## 二、通用参数类（10xxxx）— HTTP 400

| HTTP | 业务 Code | 说明 |
|------|----------|------|
| 400 | 100001 | 参数类型不合法 |
| 400 | 100002 | 参数值不合法 / 请求体格式错误（JSON 解析失败等） |
| 400 | 100003 | 必填字段缺失 |
| 400 | 100004 | 枚举值不合法 |

## 三、认证类（101xxx）— HTTP 401

| HTTP | 业务 Code | 说明 | 处理建议 |
|------|----------|------|----------|
| 401 | 101001 | 缺少 Token | 提示用户登录 |
| 401 | 101002 | Token 无效或已过期 | 自动跳转到登录页 |
| 401 | 101003 | Token 已被撤销（黑名单） | 强制重新登录 |
| 401 | 101004 | Refresh Token 无效 | 清除本地存储，跳转到登录 |
| 401 | 101005 | Refresh Token 已撤销 | 清除本地存储，跳转到登录 |
| 401 | 101006 | Refresh Token 已过期 | 清除本地存储，跳转到登录 |
| 401 | 101007 | 账号不存在 | 提示用户检查输入 |
| 401 | 101008 | 密码错误 | 提示重新输入，显示剩余尝试次数 |
| 401 | 101009 | 账号已注销（deactivated） | 提示用户账号已注销 |
| 401 | 101010 | Refresh Token 被盗用，全员下线 | 强制所有设备重新登录 |
