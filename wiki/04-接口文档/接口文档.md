## 一、通用约定

### 1.1 Base URL

| 环境 | URL |
|------|-----|
| 开发 | `http://localhost:8080/api/v1` |
| 测试 | `https://test.xfamily.com/api/v1` |
| 生产 | `https://api.xfamily.com/api/v1` |

### 1.2 统一响应格式

所有接口**必须**使用统一的 JSON 响应信封：

```json
{
  "code": 0,
  "message": "success",
  "detail": null,
  "data": {},
  "pagination": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 业务状态码。`0`=成功；非0=失败（5 位分段体系） |
| `message` | string | 错误摘要（中文） |
| `detail` | string/null | 详细错误信息（成功时为 `null`） |
| `data` | any/null | 成功时返回业务数据；失败时为 `null` |
| `pagination` | object/null | 分页信息（非分页接口为 `null`） |

### 1.3 HTTP 状态码

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 200 | 成功 | 请求成功 |
| 201 | 创建成功 | 资源创建成功 |
| 400 | 请求参数错误 | 参数类型/值不合法、必填字段缺失 |
| 401 | 认证失败 | Token 缺失/无效/过期 |
| 403 | 权限不足 | 无权限访问或操作 |
| 404 | 资源不存在 | 请求的资源未找到 |
| 429 | 请求频率超限 | 触发限流策略 |
| 500 | 系统内部错误 | 服务端异常 |

### 1.4 认证方式

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| 公开接口 | 注册、登录、重置密码、发送验证码 | 无需 Token |
| Bearer Token | 所有其他接口 | `Authorization: Bearer <access_token>` |

### 1.5 版本管理

| 版本 | 状态 | 说明 |
|------|------|------|
| v1 | 当前 | 第一版 API |

### 1.6 CORS 支持

| 项目 | 配置 |
|------|------|
| 允许来源 | `*`（生产环境限制具体域名） |
| 允许方法 | `GET, POST, PUT, PATCH, DELETE, OPTIONS` |
| 允许 Header | `Content-Type, Authorization, X-Organization-ID, X-Team-ID, X-Group-ID` |

## 三、错误码速查

| 模块段 | Code 范围 | 说明 |
|--------|----------|------|
| 101xxx | 认证 | Token 无效/过期、账号不存在、密码错误 |
| 104xxx | 认证限流 | 登录失败过多已锁定 |
| 110xxx | 验证码 | 验证码错误/过期/已使用/尝试超限 |
| 114xxx | 验证码限流 | 发送频率超限 |
| 20xxx | 账号 | 密码强度、账号已注册/已注销、宽限期 |
| 21xxx | 邀请 | 邀请不存在/过期、状态不允许 |
| 30xxx | 组织 | 组织不存在、名称重复、最后管理员约束 |
| 40xxx | 团队 | 团队不存在、名称重复、最后管理员约束 |
| 50xxx | 小组 | 小组不存在、名称重复、角色不合法 |
| 60xxx | 权限 | 不在当前 scope、权限不足、Header 参数错误 |
| 80xxx | 系统 | 内部错误、Redis/数据库连接失败、服务不可用 |

## 四、依赖关系

```
认证接口 (01)
    └── 登录后获取 Token，用于其他接口认证

账号接口 (02)
    └── 需要认证接口获取的 Token

组织接口 (03)
    ├── 需要认证接口获取的 Token
    └── 创建组织后自动成为 organization_core_admin

团队接口 (04)
    ├── 需要认证接口获取的 Token
    └── 需要组织接口创建的 org_id

小组接口 (05)
    ├── 需要认证接口获取的 Token
    ├── 需要组织接口创建的 org_id
    └── 需要团队接口创建的 team_id

权限接口 (06)
    └── 需要认证接口获取的 Token

超级管理员接口 (07)
    ├── init: 公开接口
    └── 其他: 需要超级管理员 Token

审计接口 (08)
    ├── 需要认证接口获取的 Token
    └── 需要组织接口创建的 org_id
```
