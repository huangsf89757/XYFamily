## 侧边栏 UI 精修（17:25 用户反馈）
用户截图反馈 3 个问题，调度 prototype-builder 针对性修复 `index.html`：
1. **收起态布局**：收起态 nav padding 收紧、item 高度锁 40px、logo 锁 32px（L450–454），图标栏更专业。
2. **通知红点漂移**：根因 `.icon-btn` 缺 `position:relative`，红点 absolute 参考系上飘。已加 `position:relative;overflow:visible` 并改为贴铃铛右上角 8px（L508–509、L1362）。
3. **展开态只有图标**：核查后 CSS 本就正确（`--muted-foreground` 已定义、分组/文字/选中态样式齐全），用户截图"只有图标"是窗口 ≤1280px 触发 `@media(max-width:1280px)` 响应式收起（L461–469，设计规范预期）。≥1281px 展开态有完整分组+文字+选中态。

### 判断结论
- 问题 1、2 为真实 bug，已修复并落盘。
- 问题 3 非 CSS 缺陷，是响应式断点预期行为；若用户常用 ≤1280px 窗口且希望保留文字，需调整断点（如降到 1024px 才收起，或窄屏也显示文字）——待用户决定。
